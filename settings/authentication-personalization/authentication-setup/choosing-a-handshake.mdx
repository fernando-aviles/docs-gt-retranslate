<Info>
  This is the documentation for **Authentication** Handshake methods. See [Personalization Handshake methods](/settings/authentication-personalization/personalization-setup/choosing-a-handshake) for personalization options.
</Info>

Mintlify supports four Authentication Handshake methods for user access:

1. **Password**: Global passwords for documentation access
2. **JWT**: Custom login flow with JWT authentication
3. **OAuth 2.0**: Standard Authorization Code flow integration
4. **Mintlify Dashboard**: Built-in access for dashboard users

## Method Comparison

<Tabs>
  <Tab title="Password">
    **Prerequisites**: Security policy allows password sharing
    
    ✅ Simple setup, easy sharing
    ❌ Limited access control, no user differentiation
  </Tab>
  <Tab title="JWT">
    **Prerequisites**: Existing login flow that can create JWTs
    
    ✅ Secure, zero CORS setup
    ❌ Requires login flow modification
  </Tab>
  <Tab title="OAuth 2.0">
    **Prerequisites**: OAuth server with Authorization Code flow
    
    ✅ High security standard
    ❌ Complex setup if no existing OAuth
  </Tab>
  <Tab title="Mintlify Dashboard">
    **Prerequisites**: Users have Mintlify dashboard access
    
    ✅ Zero configuration
    ❌ Limited to dashboard users
  </Tab>
</Tabs>
