---
title: 'Shared Session Handshake'
description: 'Share user sessions between your dashboard and docs'
---

<Info>
  This feature is only available for Personalization, not Authentication.
</Info>

Uses existing session authentication in the user's browser to personalize docs.

## Implementation

<Steps>
  <Step title="Create your Info API">
    Create an API endpoint that uses session auth to identify users and returns JSON in the [User](../sending-data) format.

    If your API domain differs from docs domain:
      - Add docs domain to `Access-Control-Allow-Origin` header (not `*`)
      - Set `Access-Control-Allow-Credentials: true`
      <Warning>
        Only enable these CORS settings on the user info endpoint.
      </Warning>
  </Step>
  <Step title="Configure Personalization">
    Add your API URL and Login URL in your [Mintlify dashboard settings](https://dashboard.mintlify.com/products/authentication).
  </Step>
</Steps>

## Examples

### Dashboard and docs on subdomains

**Setup:** Dashboard on `dash.foo.com`, docs on `docs.foo.com`
1. Create endpoint: `dash.foo.com/api/docs/user-info`
2. Enable CORS for `https://docs.foo.com`
3. Set API URL in Mintlify dashboard

### Dashboard on subdomain, docs on root

**Setup:** Dashboard on `dash.foo.com`, docs on `foo.com/docs`
1. Create endpoint: `dash.foo.com/api/docs/user-info`
2. Enable CORS for `https://foo.com`
3. Set API URL in Mintlify dashboard

### Both on root domain

**Setup:** Dashboard on `foo.com/dashboard`, docs on `foo.com/docs`
1. Create endpoint: `foo.com/api/docs/user-info`
2. Set API URL in Mintlify dashboard
