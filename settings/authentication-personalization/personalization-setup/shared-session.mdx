<Info>
  This is the documentation for the Shared Session **Personalization** Handshake. The Shared Session Handshake is not available for Authentication.
</Info>

This method uses existing session authentication in your user's browser for seamless documentation access.

## Implementation

<Steps>
  <Step title="Create your Info API">
    Create an API endpoint that uses session authentication to identify users and returns a JSON payload in the [UserInfo](../sending-data) format.

    For API domains that don't match the docs domain:
    - Add docs domain to `Access-Control-Allow-Origin` header (not `*`)
    - Set `Access-Control-Allow-Credentials` header to `true`
    <Warning>
      Enable these CORS options only on the user information endpoint.
    </Warning>
  </Step>
  <Step title="Configure your Personalization settings">
    Add your API URL and Login URL to your Personalization settings in the [Mintlify dashboard](https://dashboard.mintlify.com/products/authentication).
  </Step>
</Steps>

## Example Setup

### For Different Domain Configurations

1. **Dashboard and Docs on Subdomains** (e.g., dash.foo.com & docs.foo.com)
   - Create endpoint: `dash.foo.com/api/docs/user-info`
   - Set CORS headers for docs.foo.com

2. **Dashboard on Subdomain, Docs on Root** (e.g., dash.foo.com & foo.com/docs)
   - Create endpoint: `dash.foo.com/api/docs/user-info`
   - Set CORS headers for foo.com

3. **Both on Root** (e.g., foo.com/dashboard & foo.com/docs)
   - Create endpoint: `foo.com/api/docs/user-info`
   - No special CORS configuration needed
