---
title: 'Shared Session Handshake'
description: 'Share user sessions between your dashboard and docs'
---

<Info>
  The Shared Session Handshake is only available for Personalization, not for Authentication.
</Info>

The Shared Session Handshake uses existing session authentication from your user's browser for a seamless docs experience.

## Setup Steps

<Steps>
  <Step title="Set up your User Info API">
    Create an API endpoint that:
    - Uses your session authentication to identify users
    - Returns user data in the [User](../sending-data) JSON format

    If your API and docs are on different domains:
    - Add your docs domain to `Access-Control-Allow-Origin`
    - Set `Access-Control-Allow-Credentials` to `true`
    
    <Warning>
      Only enable these CORS settings on your user info endpoint, not your entire API.
    </Warning>
  </Step>
  <Step title="Update Mintlify Settings">
    Add your API URL and Login URL in your [Mintlify dashboard](https://dashboard.mintlify.com/products/authentication) under Personalization settings.
  </Step>
</Steps>

## Configuration Examples

### Docs and Dashboard on Subdomains
**Setup:** Dashboard at `dash.foo.com`, docs at `docs.foo.com`
1. Create endpoint: `dash.foo.com/api/docs/user-info`
2. Enable CORS for `https://docs.foo.com`
3. Set API URL to `https://dash.foo.com/api/docs/user-info`

### Dashboard on Subdomain, Docs on Root
**Setup:** Dashboard at `dash.foo.com`, docs at `foo.com/docs`
1. Create endpoint: `dash.foo.com/api/docs/user-info`
2. Enable CORS for `https://foo.com`
3. Set API URL to `https://dash.foo.com/api/docs/user-info`

### Dashboard and Docs on Root
**Setup:** Dashboard at `foo.com/dashboard`, docs at `foo.com/docs`
1. Create endpoint: `foo.com/api/docs/user-info`
2. Set API URL to `https://foo.com/api/docs/user-info`
