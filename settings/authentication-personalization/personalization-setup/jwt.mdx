<Info>
  This is the documentation for the JWT **Personalization** Handshake. For JWT **Authentication** Handshake setup, see [here](/settings/authentication-personalization/authentication-setup/jwt).
</Info>

Use your own login flow to send user info to your docs via a JWT in the URL when you want to keep your dashboard and docs separate.

## Implementation

<Steps>
  <Step title="Generate a private key">
    Generate a private key in your [Mintlify dashboard settings](https://dashboard.mintlify.com/products/authentication). Store this key securely for backend access.
  </Step>
  <Step title="Create a login flow">
    Create a login flow that:
    - Authenticates the user
    - Creates a JWT with user info in the [UserInfo](../sending-data) format
    - Signs the JWT with the secret key (EdDSA algorithm)
    - Redirects to your docs with the JWT as the hash
  </Step>
  <Step title="Configure your Personalization settings">
    Add your login URL to the Personalization settings in your [Mintlify dashboard](https://dashboard.mintlify.com/products/authentication).
  </Step>
</Steps>

## Example Implementation

```ts
import * as jose from 'jose';
import { Request, Response } from 'express';

const TWO_WEEKS_IN_MS = 1000 * 60 * 60 * 24 * 7 * 2;

const signingKey = await jose.importPKCS8(process.env.MINTLIFY_PRIVATE_KEY, 'EdDSA');

export async function handleRequest(req: Request, res: Response) {
  const userInfo = {
    expiresAt: Math.floor((Date.now() + TWO_WEEKS_IN_MS) / 1000),
    groups: res.locals.user.groups,
    content: {
      firstName: res.locals.user.firstName,
      lastName: res.locals.user.lastName,
    },
  };

  const jwt = await new jose.SignJWT(userInfo)
    .setProtectedHeader({ alg: 'EdDSA' })
    .setExpirationTime('10 s')
    .sign(signingKey);

  return res.redirect(`https://docs.foo.com#${jwt}`);
}
```

## Preserving Anchors

To redirect to a specific anchor post-login, use: `https://docs.foo.com/page#jwt={SIGNED_JWT}&anchor={ANCHOR}`

Example: `https://docs.foo.com/quickstart#jwt={SIGNED_JWT}&anchor=step-one`
