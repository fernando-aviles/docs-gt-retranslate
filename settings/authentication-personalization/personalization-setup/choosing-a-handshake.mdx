<Info>
  This is the documentation for **Personalization** Handshake methods. Authentication offers a [different set of Handshake methods](/settings/authentication-personalization/authentication-setup/choosing-a-handshake).
</Info>

Mintlify supports three Personalization Handshake methods for authenticating users to access personalized content:

1. **Shared Session**: Uses your dashboard's session token for authentication
2. **JWT**: Implements authentication via JWT in the URL using your login flow
3. **OAuth 2.0**: Integrates with your OAuth server using PKCE flow

## Method Comparison

<Tabs>
  <Tab title="Shared Session">
    **Prerequisites:**
    - Dashboard hosted at your domain with cookie-based sessions
    - API endpoint at same origin/subdomain as dashboard
    - Docs hosted at same domain as dashboard

    **Benefits:**
    - Automatic login sync with dashboard
    - Persistent sessions
    - Simple setup

    **Limitations:**
    - Requires backend requests
    - Needs session authentication
    - CORS configuration required
  </Tab>
  <Tab title="JWT">
    **Prerequisites:**
    - Existing login flow
    - Ability to add JWT creation and redirect step

    **Benefits:**
    - Reduced API abuse risk
    - No CORS needed
    - Flexible API URLs

    **Limitations:**
    - Requires login flow modification
    - Separate dashboard/docs authentication
    - Manual data refresh requires re-login
  </Tab>
  <Tab title="OAuth 2.0">
    **Prerequisites:**
    - OAuth server with PKCE support
    - Token-accessible API endpoint

    **Benefits:**
    - Enhanced security
    - Flexible API URLs

    **Limitations:**
    - Complex initial setup
    - Separate dashboard/docs authentication
    - May be excessive for simple needs
  </Tab>
</Tabs>
