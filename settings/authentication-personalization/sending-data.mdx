---
title: 'Sending Data'
description: 'User data structure for docs personalization'
---

Your API will return either a JSON object or signed JWT with the following structure:

```tsx
type UserInfo = {
  expiresAt?: number;
  groups?: string[];
  content?: Record<string, any>;
  apiPlaygroundInputs?: {
    header?: Record<string, any>;
    query?: Record<string, any>;
    cookie?: Record<string, any>;
    server?: Record<string, string>;
  };
};
```

<ParamField
  path="expiresAt"
  type="number"
>
  Time in seconds since epoch when data expires. For JWT Handshakes: This differs from the JWT `exp` claim - set `exp` to 10 seconds or less, while `expiresAt` can span days to weeks.
</ParamField>
<ParamField
  path="groups"
  type="string[]"
>
  User group list determining page access. Pages are shown if they match any group in their metadata.
</ParamField>
<ParamField
  path="content"
  type="object"
>
  Values accessible in MDX via `userContext`. Example: `{ firstName: 'Ronan' }` enables `{userContext.firstName}` in MDX.
</ParamField>
<ParamField
  path="apiPlaygroundInputs"
  type="object"
>
  Pre-populated values for API playground. Only works for fields in your security scheme configuration.
  
  <Note>Fields must be part of your [security scheme](https://swagger.io/docs/specification/authentication/) to be prefilled.</Note>
</ParamField>
