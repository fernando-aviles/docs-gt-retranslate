## Overview

Every Mintlify site needs a `mint.json` file with the core configuration settings. This file controls everything from your site's appearance to its navigation structure and integrations.

## Basic Properties

Let's start with the essential configuration options that every site needs.

### Site Identity

<ResponseField name="name" type="string" required>
  Name of your company or project. This is used for the global title that appears in browsers and search results.
</ResponseField>

### Logo Configuration

<ResponseField name="logo" type="string or Logo">
  Configure your site's logo for both light and dark modes. SVG format is recommended for crisp display at any size.

  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Path to your light mode logo. Example: `/logo/light.svg`
    </ResponseField>

    <ResponseField name="dark" type="string">
      Path to your dark mode logo. Example: `/logo/dark.svg`
    </ResponseField>

    <ResponseField name="href" type="string" default="/">
      URL that users go to when clicking your logo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Path to your site's favicon image. Example: `/favicon.svg`
</ResponseField>

## Visual Styling

Configure the look and feel of your documentation site.

### Color Scheme

<ResponseField name="colors" type="Colors">
  Define the hex color codes for your global theme.

  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      Main color used for highlights, headers, and accents in light mode
    </ResponseField>

    <ResponseField name="light" type="string">
      Primary color used in dark mode for highlights and accents
    </ResponseField>

    <ResponseField name="dark" type="string">
      Color used for important buttons and elements
    </ResponseField>

    <ResponseField name="background" type="object">
      Background colors for light and dark modes

      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          Background color in light mode
        </ResponseField>

        <ResponseField name="dark" type="string" required>
          Background color in dark mode
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Theme and Layout

<ResponseField name="theme" type={'"venus" | "quill" | "prism"'}>
  Choose from our preset theme configurations. Examples:
  - [Venus](https://starter-venus.mintlify.app)
  - [Quill](https://starter-quill.mintlify.app)
  - [Prism](https://starter-prism.mintlify.app)
</ResponseField>

<ResponseField name="layout" type={'"topnav" | "sidenav" | "solidSidenav"'} default="topnav">
  Set the global layout style of your documentation.
</ResponseField>

### Background Elements

<ResponseField name="background" type="Background">
  Add decorative background elements to your site.

  <Expandable title="Background">
    <ResponseField name="style" type={'"gradient" | "grid" | "windows"'}>
      Choose a decorative background style
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Set a custom background image that appears behind all pages
</ResponseField>

### Typography

<ResponseField name="font" type={
  "FontDetailsType | { headings?: FontDetailsType, body?: FontDetailsType }"
}>
  Customize your site's fonts globally or set different fonts for headings and body text.

Example:
```json
"font": {
  "headings": {
    "family": "Roboto"
  },
  "body": {
    "family": "Oswald"
  }
}
```

  <Expandable title="FontDetailsType">
    <ResponseField name="family" type="string" required>
      Font family name. Supports both [Google Fonts](https://fonts.google.com/) and custom fonts.
    </ResponseField>

    <ResponseField name="weight" type="number">
      Font weight (supports variable fonts with precise values)
    </ResponseField>

    <ResponseField name="url" type="string">
      URL to your custom font file
    </ResponseField>

    <ResponseField name="format" type="'woff' | 'woff2'">
      Required when using a custom font via URL
    </ResponseField>
  </Expandable>
</ResponseField>

## Navigation Structure

Control how users navigate your documentation.

### Primary Navigation

<ResponseField name="navigation" type="Navigation[]" required>
  Define the structure of your sidebar navigation.

  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      Section heading in the sidebar
    </ResponseField>

    <ResponseField name="pages" type="string[]">
      List of paths to your documentation files
    </ResponseField>

    <ResponseField name="icon" type="string">
      [Fontawesome](https://fontawesome.com/icons) icon for sub-folders
    </ResponseField>

    <ResponseField name="iconType" type="string">
      Icon style: brands, duotone, light, sharp-solid, solid, or thin
    </ResponseField>
  </Expandable>
</ResponseField>

### Additional Navigation Elements

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Add links to your topbar navigation.

  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      Link text
    </ResponseField>

    <ResponseField name="url" type="string">
      Destination URL
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  Configure the main call-to-action button in your topbar.

  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={"link or github"} default="link">
      Choose between a standard link or GitHub stats display
    </ResponseField>

    <ResponseField name="url" type="string">
      Button destination URL
    </ResponseField>

    <ResponseField name="name" type="string">
      Button text (for link type)
    </ResponseField>

    <ResponseField name="style" type={'"pill" | "roundedRectangle"'}>
      Button shape
    </ResponseField>

    <ResponseField name="arrow" type="boolean">
      Show/hide the arrow icon
    </ResponseField>
  </Expandable>
</ResponseField>

## Advanced Features

### Versioning and Localization

<ResponseField name="versions" type="string[]">
  Enable documentation versioning or localization. [Learn more](/settings/versioning)

Example:

<CodeGroup>
  ```json Versions
  "versions": ["v1.0", "v1.1"]
  ```

  ```json Localization
  "versions": [
    {
      "name": "English",
      "locale": "en",
    },
    {
      "name": "Espa√±ol",
      "locale": "es"
    }
  ]
  ```
</CodeGroup>

  <Expandable title="Version">
    <ResponseField name="name" type="string" required>
      Version name or language display name
    </ResponseField>

    <ResponseField name="locale" type="string">
      Language code for localization
    </ResponseField>

    <ResponseField name="default" type="true">
      Set as default version
    </ResponseField>
  </Expandable>
</ResponseField>

### API Integration

<ResponseField name="api" type="API">
  Configure API documentation settings. [Learn more](/api-playground)

  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      Base URL for API endpoints
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type="'show' | 'simple' | 'hide'">
          Control API playground visibility
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

### Integrations

<ResponseField name="integrations" type="Integrations">
  Add third-party integrations

  <Expandable title="Integrations">
    <ResponseField name="intercom" type="string">
      Intercom App ID
    </ResponseField>

    <ResponseField name="frontchat" type="string">
      Frontchat App ID
    </ResponseField>
  </Expandable>
</ResponseField>

### Analytics and SEO

<ResponseField name="analytics" type="Analytics">
  Configure analytics integrations. See [supported platforms](/integrations/analytics/overview).
</ResponseField>

<ResponseField name="seo" type="SEO">
  Search engine optimization settings.

Example:
```json
"seo": {
  "indexHiddenPages": true
}
```

  <Expandable title="SEO">
    <ResponseField name="indexHiddenPages" type="boolean" default="false">
      Allow indexing of pages not in navigation
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Configuration

Want to see it all together? Check out this complete example:

<Accordion title="View Complete mint.json Example">
```json
{
  "name": "Mintlify",
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg"
  },
  "favicon": "/favicon.svg",
  "colors": {
    "primary": "#16A34A",
    "light": "#4ADE80",
    "dark": "#166534"
  },
  "topbarLinks": [
    {
      "name": "Contact Us",
      "url": "mailto:support@mintlify.com"
    }
  ],
  "topbarCtaButton": {
    "name": "Get Started",
    "url": "https://mintlify.com/start"
  },
  "anchors": [
    {
      "name": "API Components",
      "icon": "rectangle-terminal",
      "color": "#f59f0b",
      "url": "api-components"
    }
  ],
  "navigation": [
    {
      "group": "Getting Started",
      "pages": ["introduction", "quickstart"]
    }
  ],
  "footerSocials": {
    "github": "https://github.com/mintlify",
    "x": "https://x.com/mintlify"
  }
}
```
</Accordion>
