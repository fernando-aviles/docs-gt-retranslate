# Global Documentation Configuration

Every documentation site requires a `docs.json` file that contains the core configuration settings. This file is the central control point for everything from styling and navigation to integrations and analytics.

<ConfigUpgrade />

<Note>
  Make sure to place your `docs.json` file in the root directory of your documentation project.
</Note>

## Core Properties

These fundamental settings define your documentation's identity and basic configuration.

### Project Information

<ResponseField name="name" type="string" required>
  The name of your project, organization, or product. This appears in various places throughout your documentation.
  
  Minimum length: 1 character
</ResponseField>

<ResponseField name="description" type="string">
  A brief description of your documentation site. This is used for SEO purposes and LLM indexing to help users find your content.
</ResponseField>

### Theme Selection

<ResponseField name="theme" type='"mint" | "maple" | "palm"' required>
  Choose the layout theme for your project. Each theme provides a unique look and feel:
  
  - **Mint**: Our default modern theme
  - **[Maple](https://maple.mintlify.app/)**: Clean and minimalist
  - **[Palm](https://palm.mintlify.app/)**: Bold and engaging
</ResponseField>

## Visual Customization

Configure the visual elements of your documentation to match your brand.

### Brand Colors

<ResponseField name="colors" type="object" required>
  Define your documentation's color scheme. At minimum, specify the primary color:

  ```json
  {
    "colors": {
      "primary": "#ff0000"
    }
  }
  ```

  <Expandable title="Color Configuration Options">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>
      The primary brand color used throughout the theme
    </ResponseField>
    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      A lighter variant for dark mode contrast
    </ResponseField>
    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">
      A darker variant for light mode contrast
    </ResponseField>
  </Expandable>
</ResponseField>

### Logo Configuration

<ResponseField name="logo" type="string or object">
  Set your documentation's logo. You can either use a single logo or specify different versions for light and dark modes.

  <Expandable title="Logo Options">
    <ResponseField name="light" type="string" required>
      Path to the logo for light mode (e.g., `/logo.svg`)
    </ResponseField>
    <ResponseField name="dark" type="string" required>
      Path to the logo for dark mode (e.g., `/logo-dark.svg`)
    </ResponseField>
    <ResponseField name="href" type="string (uri)">
      Custom URL for logo clicks (defaults to homepage)
    </ResponseField>
  </Expandable>
</ResponseField>

### Favicon Settings

<ResponseField name="favicon" type="string or object">
  Configure your site's favicon. The file will be automatically resized to appropriate favicon sizes.

  <Expandable title="Favicon Options">
    <ResponseField name="light" type="string" required>
      Light mode favicon path
    </ResponseField>
    <ResponseField name="dark" type="string" required>
      Dark mode favicon path
    </ResponseField>
  </Expandable>
</ResponseField>

## Typography & Fonts

<ResponseField name="fonts" type="object">
  Customize the typography of your documentation with custom fonts.

  <Expandable title="Font Configuration">
    <ResponseField name="family" type="string" required>
      The font family name (e.g., "Open Sans", "Playfair Display")
    </ResponseField>
    <ResponseField name="weight" type="number">
      Font weight (e.g., 400, 700). Variable fonts support precise weights.
    </ResponseField>
    <ResponseField name="source" type="string (uri)">
      Font file URL for custom fonts
    </ResponseField>
    <ResponseField name="format" type="'woff' | 'woff2'">
      Font format for custom sources
    </ResponseField>
  </Expandable>
</ResponseField>

## Navigation Structure

### Main Navigation

<ResponseField name="navigation" type="object" required>
  Define your documentation's navigation structure.

  <Expandable title="Navigation Options">
    <ResponseField name="global" type="object">
      Add external links that appear across all sections
    </ResponseField>
    <ResponseField name="groups" type="array of object">
      Organize content into logical groups
    </ResponseField>
    <ResponseField name="pages" type="array of string or object">
      Define individual documentation pages
    </ResponseField>
  </Expandable>
</ResponseField>

### Header & Footer

<ResponseField name="navbar" type="object">

  <Expandable title="Navbar Options">
    <ResponseField name="links" type="array of object">

    </ResponseField>
    <ResponseField name="primary" type="object">
      Primary call-to-action button
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="footer" type="object">

  <Expandable title="Footer Options">

    </ResponseField>
    <ResponseField name="links" type="array of object">
      Footer navigation links
    </ResponseField>
  </Expandable>
</ResponseField>

## Best Practices

When configuring your `docs.json` file:

1. Keep related settings grouped together
2. Use meaningful names for navigation items
3. Provide complete paths for all assets
4. Test configurations in both light and dark modes
5. Verify all external links
6. Maintain appropriate color contrast
7. Configure SEO settings properly

<Tip>
  Use JSON schema validation to ensure your configuration is correct. Add this to your `docs.json`:
  ```json
  {
    "$schema": "https://mintlify.com/docs.json"
  }
  ```
</Tip>
