## Habilitando la Autenticación

Puede agregar un método de autenticación a su docs.json para habilitarlo en todas las páginas o puede configurarlo página por página.

El método de autenticación de la página anulará docs.json si ambos están configurados.

### Token Bearer

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "bearer"
        }
      }
  }
  ```

  ```md Page Metadata
  ---
  title: "Your page title"
  authMethod: "bearer"
  ---
  ```
</CodeGroup>

### Autenticación Básica

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "basic"
        }
      }
  }
  ```

  ```md Page Metadata
  ---
  title: "Your page title"
  authMethod: "basic"
  ---
  ```
</CodeGroup>

### Clave API

<CodeGroup>
  ```json docs.json
  "api": {
      "mdx": {
        "auth": {
          "method": "key",
          "name": "x-api-key"
        }
      }
  }
  ```

  ```md Page Metadata
  ---
  title: "Your page title"
  authMethod: "key"
  ---
  ```
</CodeGroup>

### Ninguna

El método de autenticación "ninguna" es útil para deshabilitar la autenticación en un endpoint específico después de establecer un valor predeterminado en docs.json.

<CodeGroup>
  ```md Page Metadata
  ---
  title: "Your page title"
  authMethod: "none"
  ---
  ```
</CodeGroup>
