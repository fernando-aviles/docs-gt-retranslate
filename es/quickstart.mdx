## Primeros pasos

¡Bienvenido! Siga las instrucciones a continuación para aprender cómo implementar, actualizar y
potenciar su documentación con Mintlify.

### Creando el Repositorio

Los documentos de Mintlify se generan a partir de archivos MDX y configuraciones definidas en nuestro
[kit de inicio](https://github.com/mintlify/starter). Usamos GitHub para integrar
su documentación con su código y hacer que el control de versiones sea sin esfuerzo. Incorpórese a través del [panel de control](https://dashboard.mintlify.com) o clone nuestro [kit de inicio](https://github.com/mintlify/starter) para comenzar.

<AccordionGroup>
  <Accordion icon="message-bot" title="Install the GitHub app">
    Instale nuestra aplicación de GitHub para asegurarse de que sus actualizaciones se implementen automáticamente cuando envíe cambios. Puede encontrar el enlace de instalación en el [panel de control](https://dashboard.mintlify.com/settings), en la página de Configuración. Tras una instalación exitosa, aparecerá una marca de verificación junto al hash del commit del repositorio.

    <Frame>
      ![GitHub Bot Verification](https://mintlify.s3-us-west-1.amazonaws.com/mintlify/images/github-bot-verification.png)
    </Frame>
  </Accordion>

  <Accordion icon="code-branch" title="Monorepo Setup">
    Si desea que sus documentos vivan junto a su código como una configuración de monorepo,
    puede: 1. Mover el contenido de sus documentos a su monorepo. 2. Especificar la ruta a su
    `docs.json` en el[panel de control](https://dashboard.mintlify.com/settings/deployment/git-settings)

### Actualizando el Contenido

Mintlify le permite personalizar fácilmente el estilo, la estructura y el contenido de
su documentación.

<AccordionGroup>
  <Accordion icon="rectangle-terminal" title="Editing locally (recommended for developers)">
    1. Instale [git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git).
    2. Una vez que git esté instalado, clone su repositorio de documentación usando `git clone <your-repo>`. Si aún no lo ha configurado, ahora sería un buen momento para hacerlo con estas [claves SSH](https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent).
    3. Use su IDE favorito para abrir el repositorio.
    4. Instale nuestro CLI de Mintlify para previsualizar cambios con `npm i -g mintlify`.

    Aprenda más sobre esto en nuestra [guía de desarrollo local](/development).
  </Accordion>

  <Accordion icon="pen-to-square" title="Editing with the web editor">
    Aprenda más sobre cómo usar el editor web en nuestra [guía](/web-editor).
  </Accordion>

  <Accordion icon="palette" title="Customizing to your brand">
    Personalice fácilmente colores, logos y botones entre otras configuraciones en nuestro `docs.json` archivo. Comience con estas configuraciones básicas:

    ```json
    "name": "Your Company"
    "logo": {
      "light": "/logo/light.svg",
      "dark": "/logo/dark.svg",
      "href": "https://yourcompany.com"
    },
    "favicon": "/favicon.svg",
    "colors": {
      "primary": "#2AB673",
      "light": "#55D799",
      "dark": "#117866",
    },
    ```

    Puede encontrar una lista completa de configuraciones compatibles en [configuración global](/settings/global).
  </Accordion>

  <Accordion icon="pencil" title="Adding content">
    Agregue contenido con archivos MDX simples. Inicie sus páginas con esta plantilla:

    ```md
    ---
    title: "Page Title"
    sidebarTitle: "Sidebar title (optional - if different from page title)"
    description: "Subtitle (optional)"
    ---
    ```

    Aprenda más sobre cómo agregar imágenes, tablas, listas y más usando la [sintaxis MDX](/text). También ofrecemos una [amplia gama de componentes](/content/components).
  </Accordion>

  <Accordion icon="rocket" title="Pushing changes">
    Cuando esté listo, confirme y envíe sus cambios para actualizar su sitio de documentación. Aquí hay una [guía](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push) sobre cómo hacerlo. Si la aplicación de GitHub no puede implementar sus cambios con éxito, puede actualizar manualmente sus documentos a través de nuestro [panel de control](https://dashboard.mintlify.com).

    <Frame>
      ![Manual
      Update](https://mintlify.s3-us-west-1.amazonaws.com/mintlify/images/manual-update.png)
    </Frame>
  </Accordion>

  <Accordion icon="code-branch" title="Generating API references">
    Puede configurar fácilmente sus referencias de API usando un documento OpenAPI.

    1. Agregue su `openapi.yaml` o `openapi.json` archivo en su repositorio de documentos o defina el `openapi` campo en `docs.json` con una URL.

    ```json
    "openapi": "link-to-your-openapi-file"
    ```

    2. Use nuestro [scraper](/api-playground/openapi/setup#autogenerate-files-recommended) para autogenerar sus archivos de endpoints OpenAPI como:

    ```bash
    npx @mintlify/scraping@latest openapi-file <path-or-url-to-openapi-file>
    ```

    3. Finalmente, incluya los archivos MDX de endpoints generados en su `docs.json` bajo `navigation`.

    Para una guía completa sobre el uso de Mintlify con OpenAPI, consulte [esta guía](/api-playground/openapi/setup). [Esta guía](/api-playground/openapi/writing-openapi) explica cómo configurar sus métodos de autenticación de API. Para la definición manual de referencias de API, explore [nuestra sintaxis](/api-playground/overview).
  </Accordion>

  <Accordion icon="chart-simple" title="Adding analytics">
    Nuestras analíticas internas le brindan información sobre vistas de página, análisis de búsqueda, grabaciones de sesión y más. Acceda a estas en su [panel de control](https://dashboard.mintlify.com/analytics).

    También admitimos integraciones con una variedad de proveedores de análisis. Puede encontrar la lista de proveedores [aquí](/integrations/analytics/overview).
  </Accordion>
</AccordionGroup>
