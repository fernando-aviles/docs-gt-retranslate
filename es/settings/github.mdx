Mintlify se integra con la API de GitHub, permitiendo la sincronización entre tu
documentación y tu repositorio de GitHub. Esta integración es posible mediante
el uso de[Aplicaciones de GitHub](https://docs.github.com/en/developers/apps/getting-started-with-apps/about-apps#about-github-apps).

## Instalación

Para comenzar, necesitas instalar la Aplicación de GitHub de Mintlify en la cuenta
de GitHub donde reside tu repositorio de documentación. Instalar una Aplicación de GitHub
requiere ser propietario de la organización o tener permisos de administrador en un
repositorio. Si no tienes los permisos necesarios, el propietario del repositorio
deberá aprobar la solicitud. Puedes acceder a la página de instalación iniciando sesión en el[panel de control de Mintlify](https://dashboard.mintlify.com).

Para las Aplicaciones de GitHub, puedes elegir dar permisos solo a un único repositorio.
Recomendamos encarecidamente que lo hagas así ya que solo necesitamos acceso al
repositorio donde está alojada tu documentación.

## Permisos

Durante la instalación de nuestra aplicación de GitHub, se te pedirá que otorgues
ciertos permisos:

* Acceso de lectura y escritura a `checks`, `contents`, `deployments`, `pull requests`,
  y `workflows`

Estos permisos se aprovechan para proporcionar una experiencia fluida al gestionar
tu documentación.

Cuando realizas un commit en la rama que configuraste como tu rama de implementación
de documentación, obtenemos el contenido de los archivos modificados para actualizar
tu documentación. Para lograr esto, necesitamos acceso de lectura a tu `contents`.

Cuando se crean solicitudes de extracción (pull requests), creamos una verificación y un despliegue de vista previa, por lo que necesitamos acceso de escritura a`checks` y `deployments`.

Dentro del editor web de Mintlify, Mintlify trabaja en tu nombre para crear ramas y solicitudes de extracción, por lo que necesitamos acceso de escritura a`pull requests` y
`workflows`.

<Info>
  Si te preocupan los permisos de escritura, la aplicación de GitHub solo tendrá acceso a los repositorios a los que le des acceso*y* si tienes protecciones de rama activadas*no puede* hacer push directamente a tus ramas - cumple con tus reglas de protección de ramas.
</Info>

## Instalaciones de Repositorios

Al instalar nuestra aplicación de GitHub, se te pedirá que selecciones todos los repositorios o un subconjunto de ellos. Esta selección se puede cambiar en cualquier momento yendo a la
[configuración de la aplicación de GitHub](https://github.com/apps/mintlify/installations/new).
