Cada sitio de documentación requiere un`docs.json` archivo que contiene la configuración central. Este archivo controla todo, desde el estilo y la navegación hasta las integraciones y análisis.

## Propiedades

### Personalización

<ResponseField name="theme" type="&#x22;mint&#x22; | &#x22;maple&#x22; | &#x22;palm&#x22;" required>
  El tema de diseño del proyecto. Ejemplos:
  [Maple](https://maple.mintlify.app/), [Palm](https://palm.mintlify.app/),
</ResponseField>

<ResponseField name="name" type="string" required>
  El nombre del proyecto, organización o producto. Longitud mínima: 1
</ResponseField>

<ResponseField name="description" type="string">
  Descripción opcional utilizada para SEO e indexación LLM
</ResponseField>

## Mejores Prácticas

Al configurar tu `docs.json` archivo, considera estas mejores prácticas:

1. Mantén la configuración organizada agrupando las configuraciones relacionadas
2. Usa nombres significativos para grupos y páginas en tu estructura de navegación
3. Proporciona rutas completas para todos los activos (logos, favicons, etc.)
4. Prueba tu configuración tanto en modo claro como oscuro
5. Verifica que todos los enlaces externos e integraciones estén correctamente configurados
6. Usa contrastes de color apropiados para la accesibilidad
7. Configura los ajustes SEO para mejor visibilidad en motores de búsqueda

## Validación

El `docs.json` archivo se valida contra un esquema JSON para asegurar una configuración correcta. Puedes hacer referencia al esquema incluyendo:

```json
{
  "$schema": "https://mintlify.com/docs.json"
}
```
