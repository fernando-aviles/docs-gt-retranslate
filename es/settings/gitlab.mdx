Utilizamos una combinación de tokens de acceso y Webhooks para autenticar y sincronizar
cambios entre GitLab y Mintlify.

* Utilizamos tokens de acceso para obtener información de GitLab.
* Utilizamos Webhooks para que GitLab pueda notificar a Mintlify cuando se realizan cambios.
  * Esto permite a Mintlify crear despliegues de vista previa cuando se crea un MR.

## Configurar la conexión

<Steps>
  <Step title="Find your project ID">
    Dentro de tu proyecto de GitLab, navega a `Settings` > `General` y encuentra el `Project ID`.

    <Frame>
      <img src="/images/gitlab/gitlab-project-id.png" />
    </Frame>
  </Step>

  <Step title="Generate an access token">
    a. Navega a `Settings` > `Access Tokens`.

    b. Selecciona `Add new token`.

    1. Nombra el token "Mintlify".
    2. Si tienes un repositorio privado, debes establecer el rol como `Maintainer`.
    3. Elige `api` y `read_api` para los alcances.

    c. Finalmente haz clic en `Create project access token` y copia el token.

    <Frame>
      <img src="/images/gitlab/gitlab-project-access-token.png" />
    </Frame>
  </Step>

  <Step title="Set up the connection">
    Dentro del [panel de Mintlify](https://dashboard.mintlify.com/settings/deployment/git-settings), agrega el ID del proyecto y el token de acceso de los pasos anteriores junto con las otras configuraciones. Haz clic en "Guardar Cambios" cuando hayas terminado.

    <Frame>
      <img src="/images/gitlab/gitlab-config.png" />
    </Frame>
  </Step>
</Steps>

## Crear el webhook

Los Webhooks nos permiten recibir eventos cuando se realizan cambios para que podamos
activar despliegues automáticamente.

<Steps>
  <Step title="Navigate to Settings > Webhooks and click 'Add new Webhook'">
    <Frame>
      <img src="/images/gitlab/gitlab-webhook.png" />
    </Frame>
  </Step>

  <Step title="Set up URL and webhook">
    En el campo "URL", ingresa el endpoint `https://leaves.mintlify.com/gitlab-webhook` y nombra el webhook "Mintlify".
  </Step>

  <Step title="Paste token">
    Pega el token del Webhook generado después de configurar la conexión.

    <Frame>
      <img src="/images/gitlab/gitlab-show-webtoken.png" />
    </Frame>
  </Step>

  <Step title="Select events">
    Selecciona los eventos que quieres que activen el webhook:

    * Eventos de push (Todas las ramas)
    * Eventos de solicitudes de fusión

    Cuando hayas terminado, debería verse así:

    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken.png" />
    </Frame>
  </Step>

  <Step title="Test the Webhook">
    Después de crear el Webhook, haz clic en el menú desplegable "Probar" y selecciona "Eventos de push" para enviar una carga útil de prueba y asegurarte de que esté configurado correctamente. Dirá "Hook ejecutado exitosamente: HTTP 200" si está configurado correctamente.

    Esto te ayudará a verificar que todo está funcionando correctamente y que tu documentación se sincronizará adecuadamente con tu repositorio de GitLab.

    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken-test.png" />
    </Frame>
  </Step>
</Steps>
