<Info>
  Esta es la documentación para el Protocolo de **Autenticación** por Contraseña. El Protocolo de Contraseña no está disponible para Personalización.
</Info>

Si no tienes requisitos estrictos de seguridad, o no quieres administrar una base de datos de lectores de documentación, puedes usar un conjunto de contraseñas compartidas para proteger tus documentos.

<Note>
  La autenticación está disponible solo por solicitud. Por favor{" "}

  <a href="mailto:sales@mintlify.com">contacta a nuestro equipo de ventas</a> para obtener acceso. Después de que hayamos habilitado el acceso, puedes seguir las instrucciones de implementación.
</Note>

## Implementación

<Steps>
  <Step title="Add a password">
    Ve a tu [panel de control
    ajustes](https://dashboard.mintlify.com/products/authentication) y crea
    una contraseña.
  </Step>

  <Step title="Share your password">
    Comparte de forma segura la contraseña con los lectores de tu documentación. ¡Eso es todo!
  </Step>
</Steps>

## Ejemplo

Quiero configurar la autenticación para mis documentos alojados en `docs.foo.com`. No quiero
tener que hacer un seguimiento de quién puede y no puede acceder a los documentos. Mi caso de uso principal para
la autenticación es evitar que los competidores husmeen - compartir contraseña es lo suficientemente
seguro para mi equipo.

Para configurar la autenticación con Mintlify, voy a mi panel de control de Mintlify y agrego al
menos una contraseña. Luego comparto esa contraseña, junto con la URL de los documentos privados,
con los clientes potenciales.
