Web Editor es la forma preferida de editar documentos directamente sin tener que abrir tu IDE o ejecutar `mintlify dev`.

El editor incluye algunas características clave para integrarse directamente en tu flujo de trabajo git existente,
como crear ramas, solicitudes de extracción, commits y diferencias para tus cambios actuales.

También incluye una experiencia completamente editable para cambiar y agregar contenido directamente,
incluso con componentes personalizados.

Si ya entiendes los flujos de trabajo de git y nuestras integraciones, puedes saltar a [aquí](#editing-content).

## Git y flujos de trabajo de actualización

### Conceptos básicos de Git

Aunque Web Editor significa que no necesitas ir a GitHub o tu línea de comandos para hacer
cambios, sigue siendo útil conocer los conceptos básicos de git.

Terminología de Git:

* **Repositorio**: La carpeta en la que vive tu código. Puede ser local (en tu computadora) o remoto (como GitHub).

* **Commit**: Una instantánea de los cambios realizados en los archivos del repositorio.

* **Rama**: Una línea de desarrollo separada. Es una copia de trabajo del código que te permite trabajar en cambios sin afectar la versión principal.

* **Solicitud de extracción:** Una solicitud para fusionar cambios de una rama de trabajo a la rama principal. Esto se usa para revisar contenido antes de hacer cambios en vivo.

Para realizar actualizaciones en tus documentos, incluimos algunos botones específicamente para

## Modos del editor

Para ofrecer la máxima flexibilidad, el editor tiene tres modos:

  <img className="block dark:hidden" src="/images/editor/mode-toggle-light.png" />

  <img className="hidden dark:block" src="/images/editor/mode-toggle-dark.png" />
</Frame>

### Editor Visual

El **"Editor Visual"** es el modo WYSIWYG típico que verías en algo como Notion. Te ofrece una vista de tus documentos de una manera totalmente editable que refleja cómo se vería la página final en tu sitio de documentación principal.

### Editor de Código Fuente

El **"Editor de Código Fuente"** te ofrece una manera de editar tus archivos MDX en código, de la misma forma que lo harías en tu IDE. Esto ofrece menos flexibilidad, ya que nuestros componentes no están disponibles para autocompletar, pero tiene dos ventajas únicas.

1. Te permite editar las propiedades de los componentes (ver nuestras [limitaciones a continuación](#current-limitations)) que actualmente no está disponible en el modo **"Editor Visual"** todavía.

2. Te permite corregir errores de sintaxis que podrían haber aparecido en tu repositorio que podrían no ser compatibles con el modo **"Editor Visual"** hasta que los hayas corregido.

### Vista de Diferencias

La **"Vista de Diferencias"** es una forma de ver los cambios realizados en un documento específico antes de enviarlo a tu repositorio.

Esto te ayudará a ver los cambios que has realizado junto con los cambios de formato hechos por el editor.

## Limitaciones actuales

Tenemos algunas limitaciones actuales en el editor en las que estamos trabajando para resolver.

<AccordionGroup>
  <Accordion title="WYSIWYG navigation tree">
    Actualmente no puedes actualizar en tiempo real tu navegación basada en archivos añadidos/editados. Todavía tienes que editar manualmente los archivos renombrados, añadidos y eliminados en tu`docs.json`
  </Accordion>

  <Accordion title="Snippet previews">
    Actualmente no mostramos ninguna vista previa para fragmentos personalizados. Esto está en nuestra hoja de ruta para ser compatible como componentes totalmente editables.
  </Accordion>

  <Accordion title="OpenAPI previews">
    Actualmente no mostramos ninguna vista previa para especificaciones OpenAPI. Esto está en nuestra hoja de ruta para ser compatible como una vista previa de solo lectura.
  </Accordion>
</AccordionGroup>
