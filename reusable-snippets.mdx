Keep your docs DRY (Don't Repeat Yourself) by using custom snippets for repeated content.

## Creating Snippets

All snippets must be created in the `snippets` directory. Files in this directory won't render as standalone pages unless imported into another file.

### Basic Snippets

Create a snippet:
```typescript snippets/my-snippet.mdx
Hello world! This is my reusable content.
```

Use the snippet:
```typescript destination-file.mdx
import MySnippet from '/snippets/my-snippet.mdx';

<MySnippet/>
```

### Snippets with Variables

Create a snippet with a variable:
```typescript snippets/my-snippet.mdx
My keyword of the day is {word}.
```

Use it with a value:
```typescript destination-file.mdx
import MySnippet from '/snippets/my-snippet.mdx';

<MySnippet word="bananas" />
```

### Reusable Variables

Export variables:
```typescript snippets/variables.mdx
export const myName = 'my name';
export const myObject = { fruit: 'strawberries' };
```

Use them:
```typescript destination-file.mdx
import { myName, myObject } from '/snippets/variables.mdx';

Hello, my name is {myName} and I like {myObject.fruit}.
```

### Components

Create a component:
```typescript snippets/custom-component.mdx
export const MyComponent = ({ title }) => (
  <div>
    <h1>{title}</h1>
    <p>... snippet content ...</p>
  </div>
);
```

<Warning>
  Use HTML syntax in arrow functions - MDX won't compile here.
</Warning>

Use it:
```typescript destination-file.mdx
import { MyComponent } from '/snippets/custom-component.mdx';

<MyComponent title={'Custom title'} />
```

### Client-Side Content

For client-side rendering, check for `document` availability:

```typescript snippets/client-component.mdx
export const ClientComponent = () => {
  if (typeof document === "undefined") {
    return null;
  }
  
  setTimeout(() => {
    const el = document.getElementById("client-component");
    if (el) {
      el.innerHTML = "Hello, client-side component!";
    }
  }, 1);

  return <div id="client-component"></div>
}
