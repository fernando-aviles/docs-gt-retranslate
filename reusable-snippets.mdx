Following the DRY (Don't Repeat Yourself) principle, reusable snippets help keep your documentation content synchronized across multiple locations.

## Creating Custom Snippets

All snippets must be created in the `snippets` directory. These files won't be rendered as standalone pages unless imported into another file as a component.

### Default Export

Create your snippet file with reusable content:

```typescript snippets/my-snippet.mdx
Hello world! This is my content I want to reuse across pages. My keyword of the
day is {word}.
```

Import and use in your destination file:

```typescript destination-file.mdx
import MySnippet from '/snippets/path/to/my-snippet.mdx';

<MySnippet word="bananas" />
```

### Variables and Components

Export variables:
```typescript snippets/custom-variables.mdx
export const myName = 'my name';
export const myObject = { fruit: 'strawberries' };
```

Create reusable components:
```typescript snippets/custom-component.mdx
export const MyComponent = ({ title }) => (
  <div>
    <h1>{title}</h1>
    <p>... snippet content ...</p>
  </div>
);
```

<Warning>
  Use HTML syntax within arrow functions as MDX won't compile in this context.
</Warning>

### Client-Side Content

For client-side rendering, check for document availability:

```typescript snippets/client-component.mdx
export const ClientComponent = () => {
  if (typeof document === "undefined") {
    return null;
  }
  setTimeout(() => {
    const clientComponent = document.getElementById("client-component");
    if (clientComponent) {
      clientComponent.innerHTML = "Hello, client-side component!";
    }
  }, 1);
  return <div id="client-component"></div>;
}
```
