Chaque page est un fichier MDX, qui combine du contenu Markdown avec des composants React pour vous permettre de créer une documentation riche et interactive.

<div id="page-metadata">
  ## Métadonnées de page
</div>

Chaque page commence par un front matter, les métadonnées YAML encadrées par `---` au début d’un fichier. Ces métadonnées définissent l’apparence et le comportement de votre page.

Utilisez les métadonnées pour gérer :

* Les titres et descriptions des pages
* Les titres, icônes et tags de la barre latérale

<ResponseField name="title" type="string" required>
  Le titre de votre page qui apparaît dans la navigation et les onglets du navigateur.
</ResponseField>

<ResponseField name="description" type="string">
  Brève description du contenu de cette page. S’affiche sous le titre et améliore le SEO.
</ResponseField>

<ResponseField name="sidebarTitle" type="string">
  Un titre court affiché dans la navigation de la barre latérale.
</ResponseField>

<ResponseField name="icon" type="string">
  L’icône à afficher.

  Options :

  * Nom d’icône [Font Awesome](https://fontawesome.com/icons)
  * Nom d’icône [Lucide](https://lucide.dev/icons)
  * URL vers une icône hébergée en externe
  * Chemin vers un fichier d’icône dans votre projet
</ResponseField>

<ResponseField name="iconType" type="string">
  Le style d’icône [Font Awesome](https://fontawesome.com/icons). Utilisé uniquement avec les icônes Font Awesome.

  Options : `regular`, `solid`, `light`, `thin`, `sharp-solid`, `duotone`, `brands`.
</ResponseField>

<ResponseField name="tag" type="string">
  Un tag qui s’affiche à côté du titre de votre page dans la barre latérale.
</ResponseField>

```yaml Exemple d’encapsulation de frontmatter YAML
---
title: "À propos du front matter"
description: "Le front matter correspond aux métadonnées qui déterminent l’apparence et le comportement de votre page"
sidebarTitle: "Front matter"
icon: "book"
tag: "NOUVEAU"
---
```

<div id="page-mode">
  ## Mode de page
</div>

Contrôlez l’affichage de votre page avec le paramètre `mode`.

<div id="default">
  ### Par défaut
</div>

Si aucun mode n’est défini, on utilise une mise en page standard avec une barre latérale de navigation et une table des matières.

```yaml
---
title: "Titre de la page par défaut"
---
```

<div id="wide">
  ### Large
</div>

Le mode large masque la table des matières. Il est utile pour les pages qui n’ont aucun intertitre ou si vous préférez profiter de l’espace horizontal supplémentaire. Le mode large est disponible avec tous les thèmes.

```yaml
---
title: "Titre de page en largeur"
mode: "large"
---
```

<div id="custom">
  ### Personnalisé
</div>

Le mode personnalisé propose une mise en page minimaliste qui supprime tous les éléments, à l’exception de la barre de navigation supérieure. C’est une toile vierge pour créer des pages de destination ou toute autre mise en page unique nécessitant une navigation minimale. Le mode personnalisé est disponible avec tous les thèmes.

```yaml
---
title: "Titre personnalisé de la page"
mode: "custom"
---
```

<div id="frame">
  ### Cadre
</div>

Le mode Cadre propose une mise en page similaire au mode personnalisé, tout en conservant la navigation dans la barre latérale. Ce mode de page autorise le HTML personnalisé et les composants tout en maintenant l’expérience de navigation par défaut. Le mode Cadre est uniquement disponible avec les thèmes Aspen et Almond.

```yaml
---
title: "Titre de la page en mode cadre"
mode: "frame"
---
```

<div id="center">
  ### Centré
</div>

Le mode centré supprime la barre latérale et la table des matières, et centre le contenu. Il est utile pour les journaux des modifications (changelogs) ou d&#39;autres pages où vous souhaitez mettre l&#39;accent sur le contenu. Le mode centré est disponible pour les thèmes Mint et Linden.

```yaml
---
title: "Centrer le titre de la page"
mode: "center"
---
```

<div id="api-pages">
  ## Pages d’API
</div>

Créez des terrains de jeu API interactifs en ajoutant une spécification d’API dans votre frontmatter, `api` ou `openapi`.

```yaml
---
openapi: "GET /endpoint"
---
```

En savoir plus sur la création de [documentation d’API](/fr/api-playground/overview).

<div id="external-links">
  ## Liens externes
</div>

Liez des sites externes directement depuis votre navigation à l’aide de la métadonnée `url`.

```yaml
---
title: "Package npm"
url: "https://www.npmjs.com/package/mint"
---
```

<div id="search-engine-optimization">
  ## Optimisation pour les moteurs de recherche
</div>

La plupart des balises méta SEO sont générées automatiquement. Vous pouvez aussi les définir manuellement pour améliorer la visibilité de votre site, le partage sur les réseaux sociaux et la compatibilité des navigateurs.

<Note>
  Les balises méta contenant des deux-points doivent être entourées de guillemets.
</Note>

```yaml
---
"twitter:image": "/images/social-preview.jpg"
---
```

Voir [SEO](/fr/settings/seo) pour l’ensemble des options de métadonnées SEO.

<div id="internal-search-keywords">
  ## Mots-clés de recherche internes
</div>

Améliorez la découvrabilité d’une page spécifique dans la recherche intégrée en ajoutant des `keywords` dans vos métadonnées. Ces mots-clés n’apparaîtront pas dans le contenu de la page ni dans les résultats de recherche, mais les utilisateurs qui les saisissent verront cette page s’afficher parmi les résultats.

```yaml
---
keywords: ['configuration', 'mise en place', 'prise en main']
---
```
