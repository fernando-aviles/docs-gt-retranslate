L’onglet Commentaires affiche le nombre de votes positifs et négatifs reçus par vos docs, ainsi que tout retour qualitatif fourni par les utilisateurs. Utilisez ces informations pour évaluer la qualité de vos docs et apporter des améliorations.

Accédez à l’onglet Commentaires en vous rendant sur la page **Analytics** de votre [tableau de bord](https://dashboard.mintlify.com/products/analytics).

<div id="managing-feedback">
  ## Gestion des retours
</div>

Pour les retours contextuels et sur des extraits de code, vous pouvez définir le statut d’un retour et ajouter des notes internes pour suivre l’avancement de la résolution des retours des utilisateurs.

<div id="changing-feedback-status">
  ### Modifier le statut des retours
</div>

Sélectionnez le statut à côté d’un retour pour le marquer comme **En attente**, **En cours**, **Résolu** ou **Ignoré**.

Bonnes pratiques pour définir les statuts des retours :

* **En attente** : Le retour est en attente de revue.
* **En cours** : Le retour a été validé et est en cours de traitement.
* **Résolu** : Le retour a été résolu.
* **Ignoré** : Le retour a été écarté car non exploitable, non pertinent ou inexact.

<div id="filtering-by-status">
  ### Filtrer par statut
</div>

Utilisez le filtre de statut pour contrôler les commentaires affichés. Décochez un statut pour masquer tous les commentaires portant ce statut. Par défaut, tous les commentaires sont affichés.

<div id="adding-internal-notes">
  ### Ajouter des notes internes
</div>

Cliquez sur un commentaire de feedback pour ajouter une note interne. Ces notes ne sont visibles que par les personnes ayant accès à votre tableau de bord.

Utilisez les notes pour ajouter des informations de collaboration, lier des tickets de support ou d’ingénierie pertinents, ou consigner toute autre information utile.

<div id="using-feedback-data">
  ## Utiliser les données de retour d’expérience
</div>

Analysez vos données de retour d’expérience pour :

* **Identifier le contenu qui fonctionne** : Les pages recevant le plus d’avis positifs montrent ce qui marche bien dans votre documentation.
  * **Prioriser les améliorations** : Les pages avec le plus d’avis négatifs indiquent les contenus qui nécessitent votre attention.
  * **Passer à l’action** : Mettez à jour la documentation sur la base des retours directs des utilisateurs.
