La manière dont vos modifications sont publiées dépend de la branche sur laquelle vous travaillez :

* **Branche de déploiement** : Publier met immédiatement à jour votre site en ligne.
* **Autres branches** : Publier crée une pull request, afin que vous puissiez examiner les modifications avant leur déploiement en production.

<div id="pull-requests-and-reviewing-changes">
  ## Pull requests et revue des modifications
</div>

Les pull requests vous permettent de proposer des modifications depuis votre branche afin que d’autres puissent les passer en revue avant de les fusionner dans votre documentation en production. Cela aide à garantir l’exactitude de vos modifications et offre à votre équipe l’occasion de collaborer sur le contenu.

<Tip>
  Même si vous travaillez seul, les pull requests sont utiles pour prévisualiser les modifications et maintenir un historique clair des mises à jour.
</Tip>

<div id="creating-a-pull-request">
  ### Créer une pull request
</div>

<Steps>
  <Step title="Enregistrer votre travail">
    Assurez-vous que toutes vos modifications sont enregistrées sur votre branche en cliquant sur **Save Changes**.
  </Step>
  <Step title="Créer la pull request">
    Sélectionnez **Publish Pull Request** dans l’angle supérieur droit de l’éditeur.
  </Step>
  <Step title="Ajouter un titre et une description">
    Rédigez un titre et une description clairs expliquant :
    - Les modifications que vous avez apportées
    - Pourquoi vous les avez apportées
    - Les aspects spécifiques nécessitant une relecture
  </Step>
  <Step title="Créer et partager">
    Sélectionnez **Publish Pull Request**. L’éditeur fournira un lien pour afficher votre pull request.

<div id="reviewing-pull-requests">
  ## Examiner les pull requests
</div>

Une fois votre pull request créée :

1. **Réviser les changements** : Vous et les membres de votre équipe pouvez passer en revue votre pull request dans votre plateforme Git, comme GitHub ou GitLab.
2. **Laisser des commentaires** : Ajoutez des commentaires ou demandez des modifications.
3. **Apporter des changements supplémentaires** : Effectuez des changements supplémentaires dans l’Éditeur web. Lorsque vous enregistrez, l’éditeur les pousse vers votre pull request.
4. **Approuver** : Approuvez la pull request lorsque vous êtes satisfait des changements.
5. **Fusionner** : Fusionnez la pull request lorsque vous êtes prêt à déployer vos changements en production.
