Mintlify utilise une application GitHub pour synchroniser automatiquement votre documentation avec votre dépôt GitHub.



<div id="installing-the-github-app">
  ## Installation de l'application GitHub
</div>

Installez l’application GitHub Mintlify sur le compte ou l’organisation GitHub qui possède votre dépôt de documentation. Vous devez être propriétaire de l’organisation ou disposer des autorisations d’administrateur sur un dépôt pour installer l’application. Si vous n’avez pas les autorisations nécessaires, le propriétaire du dépôt doit approuver la demande d’installation.

Installez l’application GitHub Mintlify depuis votre [tableau de bord](https://dashboard.mintlify.com/settings/organization/github-app).

<Tip>
Nous recommandons d’accorder l’accès uniquement au dépôt qui héberge votre documentation.
</Tip>

<div id="permissions">
  ## Autorisations
</div>

Lorsque vous installez l’application GitHub, il vous est demandé d’accorder les autorisations suivantes :

Autorisations en lecture :
* `metadata` : Informations de base sur le dépôt

Autorisations en lecture et écriture :
* `checks` : Créer des vérifications d’état sur les pull requests
* `code` : Lire les modifications de fichiers lorsque vous validez des changements sur votre branche de documentation
* `deployments` : Générer des déploiements de prévisualisation pour les pull requests
* `pull requests` : Créer des branches et des pull requests depuis l’Éditeur web
* `workflows` : Gérer GitHub Actions pour les déploiements de la documentation

<Info>
L’application n’accède qu’aux dépôts auxquels vous lui avez explicitement accordé l’accès. Si des règles de protection des branches sont activées, l’application ne peut pas pousser directement vers les branches protégées.
</Info>



<div id="managing-repository-access">
  ## Gestion de l’accès aux dépôts
</div>

Lors de l’installation de notre application GitHub, vous pouvez accorder l’accès à tous vos dépôts ou seulement à certains. Nous recommandons de n’accorder l’accès qu’aux dépôts où se trouve votre documentation. Vous pouvez modifier cette sélection à tout moment dans vos [paramètres de l’application GitHub](https://github.com/apps/mintlify/installations/new).



<div id="configuring-docs-source">
  ## Configurer la source de la documentation
</div>

Modifiez l’organisation, le dépôt ou la branche à partir desquels votre documentation est construite dans la section [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) de votre tableau de bord.



<div id="troubleshooting">
  ## Dépannage
</div>

Si vous rencontrez des problèmes avec l’application GitHub, réinitialiser la connexion permet de résoudre la plupart des cas.

<Steps>
  <Step title="Désinstallez l’application Mintlify via GitHub.">
    1. Dans GitHub, allez aux [installations](https://github.com/settings/installations) et sélectionnez **Configure** à côté de l’application Mintlify. Faites défiler vers le bas et sélectionnez **Uninstall**.
    2. Accédez à [Authorized GitHub Apps](https://github.com/settings/apps/authorizations) et sélectionnez **Revoke** à côté de l’application Mintlify.
  </Step>
  <Step title="Réinstallez l’application Mintlify.">
    1. Dans votre tableau de bord Mintlify, allez dans [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) et installez l’application GitHub.
    2. Autorisez votre compte dans la section [My Profile](https://dashboard.mintlify.com/settings/account) de votre tableau de bord.
  </Step>
</Steps>
