Mintlify applique automatiquement de nombreuses bonnes pratiques SEO, notamment :

* Génération des balises meta
* Génération du sitemap et du fichier `robots.txt`
* Structure HTML sémantique
* Optimisation pour mobile

Vous pouvez personnaliser entièrement les balises meta de votre site en ajoutant le champ `metatags` à votre `docs.json` ou au front matter d’une page.

<div id="global-meta-tags">
  ## Balises meta globales
</div>

Vous pouvez définir des balises meta par défaut en ajoutant le champ `metadata` à votre `docs.json`.

```json
"seo": {
    "metatags": {
        "og:image": "lien vers l’image par défaut de votre balise méta"
    }
}
```

<div id="page-specific-meta-tags">
  ## Balises méta propres à la page
</div>

Ajoutez vos balises méta aux métadonnées de la page comme ceci :

```mdx
---
title: 'Titre de votre page d’exemple'
'og:image': 'lien vers l’image de votre balise meta'
---
```

<Warning>
  Les balises meta contenant des deux-points doivent être entourées de guillemets lorsque vous les ajoutez aux métadonnées de la page.
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## Plans de site et fichiers robots.txt
</div>

Mintlify génère automatiquement un fichier `sitemap.xml` et un fichier `robots.txt`. Vous pouvez consulter votre plan de site en ajoutant `/sitemap.xml` à l’URL de votre site de documentation.

Par défaut, seules les pages référencées dans votre `docs.json` sont incluses. Pour inclure des liens masqués, ajoutez `seo.indexing` à votre `docs.json` :

```json
"seo": {
    "indexation": tout
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### Plans de site personnalisés et fichiers robots.txt
</div>

Pour ajouter un fichier `sitemap.xml` ou `robots.txt` personnalisé, créez un fichier `sitemap.xml` ou `robots.txt` à la racine de votre projet. L’ajout d’un fichier personnalisé remplacera le fichier généré automatiquement portant le même nom. Si vous supprimez un fichier personnalisé, le fichier par défaut sera de nouveau utilisé.

<div id="disabling-indexing">
  ## Désactivation de l’indexation
</div>

Si vous souhaitez empêcher une page d’être indexée par les moteurs de recherche, vous pouvez inclure ce qui suit dans le [frontmatter](fr/pages) de votre page :

```
---
noindex: true
---
```

Vous pouvez également appliquer `noindex` à toutes les pages de votre documentation en définissant le champ `metadata.robots` sur `"noindex"` dans votre `docs.json` :

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## Bonnes pratiques SEO
</div>

<AccordionGroup>
  <Accordion title="Rédigez des titres et des descriptions pertinents">
    * Utilisez des titres de page clairs et descriptifs (50–60 caractères)
    * Rédigez des descriptions accrocheuses (150–160 caractères)
    * Intégrez des mots-clés pertinents
    * Rendez chaque titre et description de page uniques
  </Accordion>

  <Accordion title="Optimisez la structure de votre contenu">
    * Utilisez une hiérarchie de titres adaptée (H1 → H2 → H3)
    * Écrivez d’abord pour les utilisateurs, ensuite pour les moteurs de recherche
    * Intégrez des mots-clés pertinents dans les titres et le contenu
    * Gardez des URL courtes, descriptives et organisées de façon hiérarchique
    * Fractionnez les contenus longs avec des sous-titres et des listes
  </Accordion>

  <Accordion title="Stratégie de maillage interne">
    * Faites des liens vers des pages connexes dans votre documentation
    * Utilisez un texte d’ancrage descriptif plutôt que « cliquez ici »
    * Créez des clusters thématiques en reliant des concepts liés
    * Utilisez les fonctionnalités de référencement croisé automatique
  </Accordion>

  <Accordion title="SEO des images">
    * Utilisez des noms de fichiers descriptifs pour les images
    * Ajoutez toujours un texte alternatif pour l’accessibilité et le SEO
    * Optimisez la taille des fichiers image pour un chargement plus rapide
    * Utilisez des images pertinentes qui renforcent votre contenu
  </Accordion>
</AccordionGroup>
