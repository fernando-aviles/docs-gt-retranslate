Lorsque vous modifiez le chemin d’un fichier dans votre dossier docs, l’URL de la page change également. Cela peut se produire lors de la restructuration de votre documentation ou de la modification du titre de la barre latérale.

<div id="broken-links">
  ## Liens brisés
</div>

Détectez les liens brisés avec notre CLI. [Installez la CLI](/fr/installation) et exécutez la commande :

```bash
mint broken-links
```

La CLI détecte tous les liens relatifs dans votre documentation qui pointent vers des ressources inexistantes.

<div id="redirects">
  ## Redirections
</div>

Configurez des redirections 301 en ajoutant le champ `redirects` à votre fichier `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Cela redirige de manière permanente `/source/path` vers `/destination/path`, afin de ne pas perdre le référencement (SEO) acquis par la page d’origine.

Pour faire correspondre un chemin à l’aide d’un joker, utilisez `*` après un paramètre. Dans cet exemple, `/beta/:slug*` correspond à `/beta/introduction` et le redirige vers `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
