<div id="navigating-your-files">
  ## 浏览文件
</div>

使用侧边栏的文件资源管理器浏览文档。点击任意文件即可在编辑器中打开。

按 <kbd>Command</kbd> + <kbd>P</kbd>（Windows 为 <kbd>Ctrl</kbd> + <kbd>P</kbd>）按名称搜索文件。

<div id="creating-new-pages">
  ## 创建新页面
</div>

<Steps>
  <Step title="创建新文件">
    在文件浏览器侧边栏中选择 **Create a new file** 图标。
    
    <img
      src="/images/editor/files-menu-light.png"
      className="block dark:hidden rounded-2xl border border-gray-100 shadow-lg"
      style={{
        width: '268px',
        height: 'auto',
      }}
      alt="网页编辑器中的文件菜单"/>
    <img
      src="/images/editor/files-menu-dark.png"
      className="hidden dark:block rounded-2xl border border-white/10 shadow-lg"
      style={{
        width: '268px',
        height: 'auto',
      }}
      alt="深色模式下网页编辑器中的文件菜单"/>
  </Step>
  <Step title="输入文件名">
    文件名必须以 `.mdx` 结尾。例如：`api-reference.mdx` 或 `getting-started.mdx`。
  </Step>
  <Step title="创建并打开">
    按 <kbd>Enter</kbd> 创建文件。文件将自动在编辑器中打开，可立即开始编写内容。
  </Step>
</Steps>

<div id="organizing-your-navigation">
  ## 组织站点导航
</div>

通过编辑 `docs.json` 文件，将新页面添加到站点导航中。

<Steps>
  <Step title="打开 docs.json">
    在根目录中使用文件浏览器找到并打开 `docs.json` 文件。
  </Step>
  <Step title="定位导航部分">
    在 `docs.json` 中找到你想添加新页面的位置。
  </Step>
  <Step title="添加页面">
    在对应分组的 `pages` 数组中插入页面文件名（不包含 `.mdx` 扩展名）。
  </Step>
</Steps>

<div id="example-adding-a-themes-page">
  ### 示例：添加主题页面
</div>

```json title="向 Profile 组添加一个 themes 页面" highlight="18"
{
    "navigation": {
        "groups": [
            {
                "group": "快速入门",
                "pages": [
                    "index",
                    "quickstart", 
                    "installation"
                ]
            },
            {
                "group": "个人中心",
                "pages": [
                    "settings",
                    "account-types", 
                    "dashboard",
                    "themes"
                ]
            }
        ]
    }
}
```


<div id="editing-content">
  ## 编辑内容
</div>

在编辑器中使用可视化模式或 Markdown 模式对页面进行修改。

在可视化模式下，按 <kbd>/</kbd> 打开组件菜单。添加内容块、提示框、代码块等组件，个性化你的文档。

在 Markdown 模式下，你可以直接编辑页面的 `MDX`。当你需要以下操作时，这会很有帮助：

- 设置特定的组件属性
- 处理复杂的嵌套组件
- 从其他来源复制并粘贴 `MDX` 内容
