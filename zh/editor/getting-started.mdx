网页编辑器是在浏览器中直接创建、编辑和审阅文档的可视化界面。

* **可视化编辑**：使用所见即所得（WYSIWYG）编辑器修改文档，所见即发布效果。
* **Git 同步**：所有更改会自动同步至 Git 仓库，确保版本控制。
* **团队协作**：多人可同时协作编写与编辑文档。
* **组件集成**：通过斜杠命令快速插入提示、代码块等组件。
* **无需配置**：从控制台即可立即开始写作。

<div id="web-editor-workflow">
  ## 网页编辑器工作流程
</div>

以下是在网页编辑器中的典型工作方式：

<Steps>
  <Step title="选择分支">
    创建一个分支，或直接在部署分支上修改。建议创建分支，以便在更改上线前进行预览。
  </Step>

  <Step title="打开文件">
    在侧边栏中定位到现有文件，或通过文件资源管理器创建新文件。
  </Step>

  <Step title="编辑内容">
    在网页编辑器中进行修改。可通过右上角的开关在可视化模式与 Markdown 模式间切换，选择你更偏好的模式。
  </Step>

  <Step title="预览更改">
    可视化模式会展示你的更改在线上站点的实际呈现。用它来核对是否一切正确。
  </Step>

  <Step title="发布更改">
    如果你在部署分支上工作，可直接在网页编辑器中发布更改；在其他分支上，则需先创建拉取请求进行审核，再进行发布。
  </Step>
</Steps>

<div id="editor-modes">
  ## 编辑器模式
</div>

<div id="visual-mode">
  ### 可视化模式
</div>

可视化模式提供所见即所得（WYSIWYG）体验：你在编辑器中的更改将直接发布到文档站点。该模式非常适合需要实时查看变更效果的场景。

按下 `/` 打开组件菜单，添加样式化内容、提示框、代码片段或其他组件。

<div id="markdown-mode">
  ### Markdown 模式
</div>

Markdown 模式可直接访问文档底层的 `MDX` 代码。此模式适用于需要精确控制组件属性或偏好使用 Markdown 语法编写的场景。
