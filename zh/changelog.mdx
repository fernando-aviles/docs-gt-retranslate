  ## 重大发布

  * **重大增强**：404 页面支持 AI 推荐页面，[当用户访问死链 → AI 代理读取路径 → 推荐语义相似的页面](https://x.com/mintlify/status/1966625627773059495)
  * **重大发布**：文档助手的网页搜索现已可包含外部来源\
    *注意：请联系我们为你的网站启用此功能。*

  ## 文档助手与模型上下文协议

  * 修复了由于时间窗口未正确滑动导致文档助手被错误限流的问题
  * 修复了文档助手工具调用在处理空 `text` 块时的问题
  * 修复了将 MCP 服务器名称与工具调用拼接后有时超过 MCP 客户端强制的 60 字符限制的问题
  * 修复了文档助手菜单高度远超视口并导致无限滚动的问题
  * 修复了仪表盘中文档助手花费数值可能显示超过两位小数的问题

  ## 网页编辑器与部署

  * 增强了编辑器安全性：只有对关联的 Git 托管仓库拥有 `write permissions` 的用户才能进行更改
  * 修复了分支名包含 `=` 时预览部署无法工作的错误
  * 修复了创建预览部署时长分支名会导致模态框内容溢出的问题
  * 体验优化：注册邀请链接中的 email 查询参数将预填输入框
  * 修复了在 Safari 上通过上下文菜单复制页面不起作用的问题

  ## API交互测试台与导航

  * 多个 API交互测试台响应码在聚焦时现在显示为受控样式的下拉选择，而非系统默认的选择控件
  * 现在你可以在 `docs.json` 的导航分组上使用 [`expanded` 字段以默认展开](https://mintlify.com/docs/navigation#default-expanded-state)

  ## SEO 与 UI

  * 修复了搜索引擎中网站图标不显示的问题：现为各站点从与文档站相同的 URL 提供
  * 修复了 YouTube 嵌入在加载时闪烁的问题
  * 修复了将反馈菜单扩展为包含文字回复时会与目录产生布局位移的问题
  * 修复了在 Maple 主题中关闭通知横幅后文本会溢出到顶栏上方的问题
  * 优化 Maple 与 Willow 主题：在侧边栏新增登录/登出按钮以便更易访问

  ## 分析与导出

  * 修复了文档助手分析视图与导出的可靠性问题
  * 文档助手分析导出现在后台执行并通过邮件发送，以提供更可靠的体验
</Update>

<Update label="August 24 - August 30" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "增强的反馈收集、导航改进以及认证相关的错误修复" }}>
  ## 重要发布：增强的反馈收集

  * **重大改进**：读者在选择*点赞/点踩*后，现在可以提供更为详细的反馈，包括选项与文字评论。你还可以对代码块收集反馈，并在仪表盘的分析中查看所有回复。\

  ## 导航与易用性改进

  * 易用性提升：忽略末尾斜杠和双斜杠，因此你无需在 docs.json 中严格处理这些细节
  * 你现在可以在 `h1-6` HTML 标签中添加 `noAnchor` 属性，以避免生成锚点链接
  * Palm 主题现在在左下角提供类似 Stripe 的悬浮语言选择器
  * 在文档配置文件中新增了一个名为 [`drilldown`](https://mintlify.com/docs/navigation#enable-auto-navigation-for-groups) 的字段，用于控制当导航分组展开时，是否自动跳转至该分组的第一页
  * 易用性提升：让嵌套的有序列表在十进制/罗马数字与字母样式间交替显示

  ## 错误修复与可靠性

  * 修复了当页面包含 JS 组件时，滚动位置锚点链接无法正常工作的问题
  * 修复了由于缺少 `x-robots-tag noindex` 头而导致 Google 索引原始 `*.md` 文件的问题
  * 修复了受保护文档上的 OAuth 问题：成功完成流程后未重定向回起始页面
  * 修复了预览受认证保护文档时无法看到完整导航栏的问题
  * 修复了使用新图片 CDN 处理 SVG 的相关问题

  ## 组件与样式增强

  * 为 `SidebarNavGroupDivider` 新增了用于自定义样式的 CSS 选择器
  * 为在 MDX 中定义且包含安全设置的 API 页面新增了回归测试，以提升稳定性

  ## 性能改进

  * 将 KaTeX CSS 从 cdnjs 迁移到我们在 CloudFront 上的自有 CDN，以降低延迟并提升性能
</Update>

  ## 技术改进与可靠性

  * 增强了更新流程的日志系统，加速调试与问题定位
  * 通过从逐文件获取改为仓库克隆，修复了拥有 10+ OpenAPI规范/AsyncAPI规范 的客户在 GitHub 上的速率限制问题
  * 提升了文档助手的可靠性：增加备用 LLM 支持、强化限流错误处理，并改进搜索工具的稳健性

<Update label="8月10日 - 8月16日" tags={["Improvements"]} rss={{ title: "每周更新", description: "性能优化、LLM 集成、网页编辑器优化，以及 API 交互测试台增强" }}>
  ## 性能与构建优化

  * 在未缓存的 NextJS 无服务器环境中，MDX 转译现于部署时进行，而非每次页面加载，从而缩短未缓存页面的首字节时间（TTFB）。
  * 基于内容的哈希在 MDX 未变更时可避免重新转译，将大量页面的更新流程耗时降低约 50%（超过 5 分钟的部署大致可减半）
  * 仪表盘中的预览部署加载更快，得益于后端新增数据库索引与查询并行化
  * 通过移除各页面 `rsc` 负载中重复的 `navigation` 数据来减小页面体积——在页面量大或导航结构复杂时收益最明显
  * 更激进的预取使即时页面加载更为常见

  ## API 交互测试台与 OpenAPI规范 增强

  * 将 OpenAPI 到 MCP 的转换移至后端，使托管的 MCP 服务器能够包含工具（更清晰的文档与配置选项即将推出）
  * 为 API 交互测试台新增 Ruby 支持
  * 我们新增一项功能，现在你可以[仅用 docs.json 指定 API 页面](/zh/api-playground/openapi-setup#auto-populate-api-pages)，无需创建新的 MDX 文件。
  * 在来自 OpenAPI 规范的文档导航中支持[`webhook` 页面](/zh/api-playground/openapi-setup#webhooks)
  * 通过在跳转到 Anthropic、OpenAI 或其他提供商时，从 Markdown 链接中移除锚点参数来优化 AI 模型上下文

  ## 网页编辑器改进

  * 创建/重命名文件时，点击空白处即可保存更改，无需再按 Enter 键
  * 修复了分支导航：将 URL 切换到指定分支时会错误重定向到上次活跃分支的问题
  * 对包含 `/` 的分支标题进行正确的 URL 编码，避免导航异常
  * 修复 monorepo 仪表盘编辑器中的 `Ctrl+K` 链接快捷键会在前面加上 docs 仓库路径并导致链接失效的问题

  ## 分析与 LLM 集成

  * 支持自定义 `llms.txt` 与 `llms-full.txt`——将其添加到 docs 仓库根目录即可分别通过 `/llms.txt` 与 `/llms-full.txt` 端点提供，用于 LLM 定制
  * 新增 [Hightouch 分析集成](/zh/integrations/analytics/hightouch#hightouch)
  * 增强上下文菜单的分析追踪（仪表盘可视化即将上线）
  * 为 `llms.txt` 与 `llms-full.txt` 增加端到端测试，确保正确提供

  ## 组件与样式增强

  * `h{1-4}` 标签支持自定义 className，以应用自定义标题样式
  * 修复在自定义页面模式中 `h{1-4}` 标签被渲染为带有 chips 的 `Heading` 组件的问题
  * 为[面包屑](/zh/navigation#breadcrumbs)新增 CSS 选择器，便于定制化选择
  * 通过分析尺寸在 56px 高度下保持比例，修复被拉伸的 open-graph 图片
  * 在启用的上下文菜单中将 `VSCode` 更正为 `VS Code`
  * 修复自定义组件内的标题与语义标题一同出现在目录中的问题

  ## Bug 修复与可靠性

  * 通过清理会导致生成问题的字符，修复某些页面标题的 PDF 渲染问题
  * 解决遇到空 OpenAPI JSON 文件时的 CLI 错误 `Cannot convert undefined or null to object`
  * 修复自定义 `docs.json` open-graph 元标签被生成标签覆盖的问题
  * 在落到锚点链接时，RSS 链接改用 origin + pathname，修复 RSS 订阅按钮点击问题
  * 通过移除 source map 提升 CLI 下载速度

  ## 技术改进

  * 在 CI 流水线中新增可视化测试，更早发现回归
  * 增强错误处理与调试能力
  * 为新功能与边界场景提供更全面的测试覆盖
</Update>

<Update label="August 3 - August 9" tags={["Improvements"]} rss={{ title: "Weekly Updates", description: "Search API, OAuth polish, asset serving, and MCP tooling" }}>
  ## 认证改进

  * 组级公开访问：通过 `docs.json` 将整个页面分组设为公开，因此无需在每个页面上添加 `public: true`（[了解更多](https://mintlify.com/docs/authentication-personalization/authentication-setup#group-level)）
  * 在 OAuth 配置中支持 [`logoutURL`](https://mintlify.com/docs/authentication-personalization/authentication-setup#implementation-3)，用于删除上游 cookie 并完成登出
  * 发生 OAuth 错误时，用户会被重定向至你指定的 `logoutURL` 以重启认证流程
  * 修复了在 OAuth/JWT 流程回调前短暂闪现 500 错误的问题
  * 在 OAuth/JWT 认证配置中自动去除 URL 中的 `https://`，以防止错误配置

  ## API交互测试台增强

  * 新增 [Search API 端点](https://mintlify.com/docs/api-reference/assistant/search)，可在你的文档之上构建代理与 MCP 服务器
  * 现已在其指定路径下提供 `openapi` 与 `asyncapi` 文件（例如 `https://mydocsurl.extension/{openapi-or-file-name}.json`）
  * 你现在可以在 OpenAPI 规范文件中使用 [`x-mint` 字段](https://mintlify.com/docs/api-playground/openapi-setup#x-mint-extension) 覆盖生成字段、自定义前言内容，或更改代码示例中的端点 URL
  * 在 OpenAPI 配置中，[`x-mcp` 现为 `x-mint.mcp`](https://mintlify.com/docs/api-playground/openapi-setup#mcp)，用于控制哪些路由作为 MCP 工具暴露

  ## 文档助手更新

  * 修复了当新消息流式传入时，较旧消息的操作菜单（包含复制、点赞等选项）会消失的问题
  * 修复了上周[托管 MCP 服务器发布](https://mintlify.com/docs/ai/model-context-protocol#accessing-your-mcp-server)后，嵌套的 `/mcp/...` 页面可访问性问题

  ## 性能与可靠性

  * 你文档仓库中的所有图片和视频资源现已在你的域名下按正确路径提供。例如，如果仓库中有 `/assets/marketing/my-logo.png`，可通过 `https://mydocsurl.extension/assets/marketing/my-logo.png` 访问。
  * Mintlify 控制台登录页的邮箱输入框现已自动聚焦，可立即开始输入（体验优化）
  * 在 Redis 中同时缓存自定义域名与子域名，以提升导航加载性能（约 50ms 时延降低）
  * 为 PDF 导出添加了重试逻辑，以提升可靠性
  * 修复了 Cookie 同意弹窗在接受或关闭后仍会再次出现的问题——现已尊重首次选择
  * 通过在 `navigator.write` 中指定 MIME `type`，修复了在 Safari 上复制页面到剪贴板的问题

  ## 技术改进

  * 修复了 Windows 与 pnpm 下的 CLI 问题，并加入 CI 测试以防回归
  * 改进了错误日志输出——为工程团队调试带来更佳体验
  * 当缺少 `contentDirectory` 文件时，修复了断链 CI 动作的若干小问题
  * 修复了上周认证保护预览修复引发的回归问题：导航 UI 中未能正确设置活动选项卡
  * 修复了主题浅色背景未应用到活动选项卡图标的问题
  * 修复了在控制台更改认证类型会先更新后又回退到先前已保存类型的问题——现在新选择在保存后会持久生效
  * 为拥有自定义 UI 库的企业客户带来内部开发体验改进——我们现在更易在更短时间内纳入你的组件并满足需求
