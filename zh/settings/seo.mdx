Mintlify 会自动处理多项 SEO 最佳实践，包括：

* 自动生成 meta 标签
* 生成站点地图与 `robots.txt` 文件
* 语义化 HTML 结构
* 移动端优化

你可以在 `docs.json` 或页面的 frontmatter 中添加 `metatags` 字段，来自定义站点的 meta 标签。

<div id="global-meta-tags">
  ## 全局元标签
</div>

可在 `docs.json` 中添加 `metadata` 字段来设置默认元标签。

```json
"seo": {
    "metatags": {
        "og:image": "默认元标签图片的链接"
    }
}
```

<div id="page-specific-meta-tags">
  ## 页面特定的元标签
</div>

将元标签按如下方式添加到页面的元数据中：

```mdx
---
title: '示例页面标题'
'og:image': '用于元标签的图片链接'
---
```

<Warning>
  在页面元数据中使用带冒号的 meta 标签时，必须用引号将其括起来。
</Warning>

<div id="sitemaps-and-robotstxt-files">
  ## 站点地图与 robots.txt 文件
</div>

Mintlify 会自动生成 `sitemap.xml` 和 `robots.txt` 文件。你可以在文档站点的 URL 末尾追加 `/sitemap.xml` 来查看站点地图。

默认仅收录 `docs.json` 中包含的页面。若要包含隐藏链接，请在 `docs.json` 中添加 `seo.indexing`：

```json
"seo": {
    "indexing": all
}
```

<div id="custom-sitemaps-and-robotstxt-files">
  ### 自定义站点地图和 robots.txt 文件
</div>

要添加自定义的 `sitemap.xml` 或 `robots.txt` 文件，请在项目根目录创建一个同名文件。添加自定义文件会覆盖系统自动生成的同名文件。删除自定义文件后，将恢复使用默认文件。

<div id="disabling-indexing">
  ## 禁用收录
</div>

如果你想阻止搜索引擎收录某个页面，可以在该页面的 [frontmatter](zh/pages) 中添加以下内容：

```
---
noindex: true
---
```

你也可以在 `docs.json` 中将 `metadata.robots` 字段设置为 `"noindex"`，从而为文档内的所有页面指定 `noindex`：

```json
"seo": {
    "metatags": {
      "robots": "noindex"
    }
  }
```

<div id="seo-best-practices">
  ## SEO 最佳实践
</div>

<AccordionGroup>
  <Accordion title="撰写清晰明确的标题与描述">
    * 使用清晰、具描述性的页面标题（50–60 个字符）
    * 撰写具有吸引力的描述（150–160 个字符）
    * 包含相关关键词
    * 确保每个页面的标题和描述均唯一
  </Accordion>

  <Accordion title="优化内容结构">
    * 使用正确的标题层级（H1 → H2 → H3）
    * 先面向读者写作，再考虑搜索引擎
    * 在标题和正文中包含相关关键词
    * 保持 URL 简洁、具描述性，并按层级组织
    * 用小标题和列表拆分长内容
  </Accordion>

  <Accordion title="内部链接策略">
    * 在文档内链接到相关页面
    * 使用具描述性的锚文本，而非“click here”
    * 通过链接相关概念创建主题集群
    * 使用自动交叉引用功能
  </Accordion>

  <Accordion title="图片 SEO">
    * 为图片使用具描述性的文件名
    * 始终添加 alt 文本以满足无障碍与 SEO 需求
    * 优化图片文件大小以提升加载速度
    * 使用与内容相关且能支撑论述的图片
  </Accordion>
</AccordionGroup>
