title: "自定义 404 页面"

---

您可以自定义当用户访问失效或不存在的链接时显示的 404 错误页面的标题和描述。

在自定义 404 页面时，建议在描述中引导用户前往文档中的相关资源或链接，帮助他们快速找到所需内容。

<div id="configuration">
  ## 配置
</div>

在 `docs.json` 文件的 `errors.404` 配置段中设置你的 404 页面：

```json
"errors": {
  "404": {
    "redirect": false,
    "title": "找不到我了",
    "description": "这个_页面_到底发生了什么？"
  }
}
```


<div id="parameters">
  ## 参数
</div>

<ResponseField name="redirect" type="boolean">
  当页面未找到时，是否自动重定向到首页。
  
  设为 `true` 时将重定向到首页。

  设为 `false` 时将显示自定义 404 页面。
</ResponseField>

<ResponseField name="title" type="string">
  404 错误页面的自定义标题。将替换默认的“Page not found”标题。
</ResponseField>

<ResponseField name="description" type="string">
  404 错误页面的自定义描述。支持 Markdown 格式。
</ResponseField>
