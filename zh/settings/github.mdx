Mintlify 通过 GitHub 应用自动将文档与 GitHub 仓库保持同步。



<div id="installing-the-github-app">
  ## 安装 GitHub 应用
</div>

在拥有你文档仓库的 GitHub 账户或组织中安装 Mintlify GitHub App。你必须具备组织所有者权限或仓库管理员权限才能安装该应用。如果你没有相应权限，则需要由仓库所有者审批安装请求。

可通过你的[控制台](https://dashboard.mintlify.com/settings/organization/github-app)安装 Mintlify GitHub App。

<Tip>
建议仅授予对托管文档的那一个仓库的访问权限。
</Tip>

<div id="permissions">
  ## 权限
</div>

安装 GitHub 应用时，你会被提示授予以下权限：

读取权限：
* `metadata`: 基本仓库信息

读写权限：
* `checks`: 在拉取请求上创建状态检查
* `code`: 在你向文档分支提交时读取文件变更
* `deployments`: 为拉取请求生成预览部署
* `pull requests`: 通过网页编辑器创建分支和拉取请求
* `workflows`: 管理用于文档部署的 GitHub Actions

<Info>
该应用只会访问你明确授予访问权限的仓库。若你启用了分支保护规则，该应用无法直接推送到受保护的分支。
</Info>



<div id="managing-repository-access">
  ## 管理仓库访问权限
</div>

安装我们的 GitHub 应用时，你可以授予对所有仓库或仅特定仓库的访问权限。我们建议只为存放你文档的仓库授予访问权限。你可以随时在 [GitHub 应用设置](https://github.com/apps/mintlify/installations/new) 中修改此选择。



<div id="configuring-docs-source">
  ## 配置文档来源
</div>

在仪表盘的 [Git 设置](https://dashboard.mintlify.com/settings/deployment/git-settings) 中更改用于构建文档的组织、仓库或分支。



<div id="troubleshooting">
  ## 故障排除
</div>

如果你在使用 GitHub 应用时遇到问题，重置连接通常可以解决大多数问题。

<Steps>
  <Step title="通过 GitHub 卸载 Mintlify 应用">
    1. 在 GitHub 中，前往 [installations](https://github.com/settings/installations)，在 Mintlify 应用旁选择 **Configure**，向下滚动并选择 **Uninstall**。
    2. 前往 [Authorized GitHub Apps](https://github.com/settings/apps/authorizations)，在 Mintlify 应用旁选择 **Revoke**。
  </Step>
  <Step title="重新安装 Mintlify 应用">
    1. 在 Mintlify 控制面板中，前往 [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) 并安装 GitHub 应用。
    2. 在控制面板的 [My Profile](https://dashboard.mintlify.com/settings/account) 中为你的账户授权。
  </Step>
</Steps>
