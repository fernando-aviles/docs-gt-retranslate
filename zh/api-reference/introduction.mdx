Mintlify 的 REST API 允许你以编程方式与文档交互、触发更新，并嵌入由 AI 驱动的聊天体验。

<div id="endpoints">
  ## 端点
</div>

* [触发更新](/zh/api-reference/update/trigger): 按需触发你的网站更新。
* [获取更新状态](/zh/api-reference/update/status): 获取某次更新的状态及文档的其他详细信息。
* [生成文档助手消息](/zh/api-reference/assistant/create-assistant-message): 将基于你的文档训练的文档助手嵌入到任意应用中。
* [搜索文档](/zh/api-reference/assistant/search): 搜索你的文档。

<div id="authentication">
  ## 认证
</div>

你可以通过[控制台](https://dashboard.mintlify.com/settings/organization/api-keys)生成 API 密钥。API 密钥与整个组织关联，可用于多个部署。

<div id="admin-api-key">
  ### 管理员 API 密钥
</div>

管理员 API 密钥用于 [Trigger update](/zh/api-reference/update/trigger) 和 [Get update status](/zh/api-reference/update/status) 端点。

管理员 API 密钥以 `mint_` 前缀开头。请妥善保管你的管理员 API 密钥。

<div id="assistant-api-key">
  ### 文档助手 API 密钥
</div>

文档助手 API 密钥用于 [Generate assistant message](/zh/api-reference/assistant/create-assistant-message) 和 [Search documentation](/zh/api-reference/assistant/search) 端点。

文档助手 API 密钥以 `mint_dsc_` 前缀开头。

文档助手 API 的“key”为服务端令牌，应予以保密。

文档助手 API 的“token”为公开令牌，可在你的前端代码中引用。

<Note>
  使用文档助手 API token 的调用可能产生费用：要么消耗你的 AI 文档助手额度，要么产生超额费用。
</Note>
