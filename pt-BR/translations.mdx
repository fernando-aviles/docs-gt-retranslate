## Visão Geral

Você pode traduzir automaticamente sua documentação para vários idiomas mantendo a estrutura, formatação e links com IA. As traduções se integram com o recurso de [languages](/pt-BR/navigation#languages), permitindo que os leitores alternem entre os idiomas disponíveis em sua documentação.

## Pré-requisitos

* Você deve ter um plano Growth, Pro ou Enterprise e complementos de tradução para seus idiomas-alvo.
* Você deve usar o GitHub como seu provedor de repositório para o conteúdo da sua documentação.
* Sua documentação deve usar o inglês como idioma base.
* Você deve ter um arquivo `docs.json` válido em seu repositório.
* Todo o conteúdo a ser traduzido deve usar a formatação `MDX` válida. Qualquer `MDX` inválido fará com que a tradução falhe.

## Começando

Comece com uma tradução completa da sua documentação para cada um dos seus idiomas preferidos. Essa abordagem garante cobertura completa em cada idioma e atualiza automaticamente seu arquivo `docs.json`.

Você pode iniciar uma tradução completa na página de traduções no seu [dashboard](https://dashboard.mintlify.com/settings/organization/translations).

1. Selecione **Iniciar tradução**.
2. Selecione o primeiro idioma para o qual você deseja traduzir seu conteúdo e depois selecione **Iniciar tradução**.
3. O processo de tradução criará uma nova branch e pull request no repositório do seu projeto Mintlify com o conteúdo traduzido. Conjuntos de documentação maiores podem levar mais tempo para processar.
4. Revise o pull request e faça o merge.
5. Repita o processo para cada idioma para o qual você deseja traduzir.

<Info>
  Uma tradução automatizada é armazenada em um diretório nomeado de acordo com o [código do idioma](/pt-BR/navigation#languages) que você selecionou.

  Nomes de diretório personalizados para traduções não são suportados no momento.
</Info>

## Traduções Contínuas

Depois de adicionar uma tradução ao seu arquivo `docs.json`, quaisquer alterações futuras em sua documentação são automaticamente traduzidas, desde que você mantenha uma assinatura do complemento de tradução para seus idiomas configurados.

O processo de tradução contínua segue esta sequência:

1. Um membro da sua organização faz alterações na documentação em seu repositório.
2. Quando as alterações são enviadas ou mescladas em sua branch principal, o trabalho de tradução contínua é acionado.
3. O trabalho de tradução atualiza, remove ou cria os arquivos necessários em todos os idiomas assinados com base no seu arquivo `docs.json`.
4. O trabalho de tradução cria uma branch separada e um pull request contendo as alterações traduzidas para sua revisão.

<Info>
  A Mintlify nunca publicará automaticamente traduções em produção. Você deve revisar e mesclar cada pull request de tradução antes que as alterações estejam ativas.
</Info>

O trabalho de tradução automática não lida bem com alterações manuais nos arquivos localizados ou nas árvores de navegação `docs.json`. Faça todas as alterações em seu conteúdo em inglês e deixe o serviço de tradução gerenciar as versões localizadas.

Recomendamos revisar e mesclar prontamente os pull requests de tradução para que suas localizações sejam mantidas sincronizadas.

## Escrevendo conteúdo para ser traduzido

Para tornar sua documentação mais traduzível e mais fácil de ler em inglês, **evite** estes padrões comuns:

* **Prolixidade**: Frases redundantes são mais difíceis de compreender e traduzir.
  * **Evite**: "Para ser capaz de concluir com sucesso o processo de instalação..."
  * **Melhor**: "Para instalar o software..."
* **Voz passiva**: A voz passiva torna pouco claro quem realiza uma ação.
  * **Evite**: "O arquivo de configuração deve ser atualizado pelo administrador."
  * **Melhor**: "O administrador deve atualizar o arquivo de configuração."
* **Frases negativas**: Diga às pessoas o que fazer em vez do que não fazer.
  * **Evite**: "Não se esqueça de não pular a etapa de configuração."
  * **Melhor**: "Lembre-se de completar a etapa de configuração."
* **Pronomes**: Palavras como "ele", "isto" e "aquilo" podem não ser claras sobre a que se referem, especialmente se uma frase usa muitos pronomes. Use substantivos específicos em vez disso.
  * **Evite**: "Configure a conexão do banco de dados. Depois de fazer isso, ele funcionará corretamente."
  * **Melhor**: "Configure a conexão do banco de dados. Depois de configurar a conexão, o banco de dados funcionará corretamente."
  * **Evite**: "Quando você o configurar, certifique-se de que ele se conecte a eles corretamente para que possam se comunicar entre si."
  * **Melhor**: "Quando você configurar o banco de dados, certifique-se de que o banco de dados se conecte aos servidores corretamente para que os servidores possam se comunicar entre si."
* **Âncoras de link ambíguas**: Descreva com precisão para onde um link leva e evite âncoras de link ambíguas como `Click here`. Quando possível, use o título exato da página ou recurso para o qual um link aponta.
  * **Evite**: "Clique `[here](path/to/getting-started-guide)` para saber mais sobre o processo de configuração."
  * **Melhor**: "Saiba mais sobre o processo de configuração em nosso `[getting started guide](path/to/getting-started-guide)`."
* **Nominalizações**: Verbos ou adjetivos transformados em substantivos, geralmente terminando em -ção, -mento ou -dade. Use verbos ativos em vez disso.
  * **Evite**: "A configuração da sua navegação requer um `docs.json` válido."
  * **Melhor**: "Você deve ter um `docs.json` válido para configurar sua navegação."
* **Modificadores empilhados**: Múltiplos substantivos ou adjetivos agrupados criam ambiguidade sobre as relações entre as palavras.
  * **Evite**: "Sistema de gerenciamento de configuração de endpoint de API."
  * **Melhor**: "Sistema para gerenciar a configuração de endpoints de API."
* **Frases preposicionais**: Longas cadeias de frases preposicionais (frases começando com palavras como "de", "em", "com", "sobre") criam frases complexas.
  * **Evite**: "As seleções das configurações no painel de administração do aplicativo."
  * **Melhor**: "Configurações da aplicação no painel de administração."
* **Verbos auxiliares modais**: Palavras como "pode", "poderia", "deveria" e "deve" que expressam possibilidade ou obrigação podem ser pouco claras. Use comandos diretos em vez disso.
  * **Evite**: "Você pode querer considerar atualizar a configuração, o que poderia potencialmente melhorar o desempenho."
  * **Melhor**: "Atualize a configuração para melhorar o desempenho."
* **Modo não indicativo**: Use declarações diretas em vez de modo subjuntivo ou condicional.
  * **Evite**: "Se você configurasse as configurações, a aplicação funcionaria corretamente."
  * **Melhor**: "Configure as configurações para que a aplicação funcione corretamente."
* **Plurais invisíveis**: Deixe claro se você está se referindo a itens no singular ou no plural.
  * **Evite**: "Dados do usuário" (Pode ser dados de um usuário ou de vários usuários)
  * **Melhor**: "Dados de usuários individuais" ou "Dados de todos os usuários"
* **Palavras específicas de gênero**: Alguns idiomas têm regras de gênero que não se traduzem diretamente.
  * **Evite**: "Quando um usuário faz login, ele deve ver seu painel."
  * **Melhor**: "Após fazer login, os usuários veem o painel."
* **Siglas**: Defina quaisquer siglas no primeiro uso para fornecer contexto.
  * **Evite**: "Use as operações CRUD."
  * **Melhor**: "Use operações CRUD (Criar, Ler, Atualizar, Deletar)."
* **Gírias, piadas ou exemplos culturalmente específicos**: Use linguagem que se traduz bem.
  * **Evite**: "Não se preocupe, nós cuidamos de você."
  * **Melhor**: "Fornecemos suporte para este processo."
  * **Evite**: "Este recurso é um sucesso estrondoso."
  * **Melhor**: "Este recurso fornece resultados excelentes."

## Solução de problemas e FAQs

<AccordionGroup>
  <Accordion title="Qual conteúdo é traduzido?">
    O serviço de tradução traduz todo o conteúdo não-código e não-componente em sua documentação. Isso inclui:

    * Texto
    * Links
    * Títulos e descrições no frontmatter
    * Atributos de componentes
    * Rótulos de navegação

    O serviço não traduz:

    * Blocos de código
    * Imagens ou outras mídias
    * Metadados
  </Accordion>

  <Accordion title="Como atualizo minhas traduções manualmente?">
    Se você precisar modificar suas traduções, faça quaisquer alterações diretamente nos pull requests de tradução criados pelo serviço de tradução.

    Quaisquer alterações manuais podem ser sobrescritas na próxima vez que a versão em inglês da página for atualizada e o trabalho de tradução for executado.
  </Accordion>

  <Accordion title="Por que minha tradução não está funcionando?">
    Verifique estes problemas comuns:

    * Verifique se sua assinatura de tradução está ativa para o idioma.
    * Confirme se seu conteúdo usa`MDX`formatação válida.
    * Confirme se seu arquivo`docs.json`está configurado corretamente.

    Se você ainda estiver enfrentando problemas, por favor[entre em contato com o suporte](mailto:support@mintlify.com).
  </Accordion>

  <Accordion title="Por que os PRs de tradução às vezes incluem seções inalteradas?">
    O sistema de tradução opera no nível da página e tentará traduzir páginas inteiras quando qualquer parte mudar.

    Essa abordagem garante que a tradução seja o mais precisa possível e elimina a necessidade de atualizações manuais em arquivos localizados.
  </Accordion>

  <Accordion title="O idioma que desejo não está disponível.">
    Oferecemos traduções para todos os idiomas suportados pelo recurso[languages](/pt-BR/navigation#languages).

    Se o idioma para o qual você gostaria de traduzir não estiver listado, por favor[entre em contato com o suporte](mailto:support@mintlify.com)e teremos prazer em ajudar.
  </Accordion>

  <Accordion title="Posso definir traduções específicas para palavras ou impedir que termos sejam traduzidos?">
    O serviço de tradução não suporta regras de tradução personalizadas para termos ou frases específicas. Estamos desenvolvendo esse recurso para uma versão futura.
  </Accordion>
</AccordionGroup>
