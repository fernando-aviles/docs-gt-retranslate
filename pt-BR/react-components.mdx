[Componentes React](https://react.dev)são uma maneira poderosa de criar elementos interativos e reutilizáveis em sua documentação.

Você pode usar React diretamente em seus arquivos MDX sem nenhuma configuração adicional.

## Usando Componentes React

Você pode construir componentes diretamente em seus arquivos MDX usando[hooks do React](https://react.dev/reference/react/hooks).

### Exemplo Básico

Aqui está um exemplo básico de um componente contador:

```jsx
export const Counter = () => {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      <p>Current count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        +
      </button>
    </div>
  );
}
```

O componente`Counter`pode então ser usado em seus arquivos MDX assim:

```jsx
<Counter />
```

E o componente será renderizado como um componente React no arquivo MDX.

<Counter />

## Importando Componentes

Assim como no React regular, você pode importar componentes de outros arquivos.

```jsx
import { ColorGenerator } from "/snippets/color-generator.mdx"
```

<Warning>
  Mas diferentemente do React regular, você não pode importar componentes de todos os arquivos MDX. Componentes reutilizáveis só podem ser referenciados a partir de arquivos MDX dentro da pasta`snippets`.
</Warning>

Depois de importar o componente, você pode usá-lo em seus arquivos MDX assim:

```jsx
<ColorGenerator />
```

Saiba mais sobre[snippets reutilizáveis](/pt-BR/reusable-snippets).

### Exemplo Complexo

Você também pode construir componentes muito mais complexos. Aqui está um exemplo de um componente gerador de cores que usa múltiplos hooks do React:

```jsx /snippets/color-generator.mdx [expandable]
export const ColorGenerator = () => {
  const [hue, setHue] = useState(180)
  const [saturation, setSaturation] = useState(50)
  const [lightness, setLightness] = useState(50)
  const [colors, setColors] = useState([])

  useEffect(() => {
    const newColors = []
    for (let i = 0; i < 5; i++) {
      const l = Math.max(10, Math.min(90, lightness - 20 + i * 10))
      newColors.push(`hsl(${hue}, ${saturation}%, ${l}%)`)
    }
    setColors(newColors)
  }, [hue, saturation, lightness])

  const copyToClipboard = (color) => {
    navigator.clipboard
      .writeText(color)
      .then(() => {
        console.log(`Copied ${color} to clipboard!`)
      })
      .catch((err) => {
        console.error("Failed to copy: ", err)
      })
  }

  return (
    <div className="p-4 border dark:border-zinc-950/80 rounded-xl not-prose">
      <div className="space-y-4">
        <div className="space-y-2">
          <label className="block text-sm text-zinc-950/70 dark:text-white/70">
            Hue: {hue}°
            <input
              type="range"
              min="0"
              max="360"
              value={hue}
              onChange={(e) => setHue(Number.parseInt(e.target.value))}
              className="w-full h-2 bg-zinc-950/20 rounded-lg appearance-none cursor-pointer dark:bg-white/20 mt-1"
              style={{
                background: `linear-gradient(to right, 
                  hsl(0, ${saturation}%, ${lightness}%), 
                  hsl(60, ${saturation}%, ${lightness}%), 
                  hsl(120, ${saturation}%, ${lightness}%), 
                  hsl(180, ${saturation}%, ${lightness}%), 
                  hsl(240, ${saturation}%, ${lightness}%), 
                  hsl(300, ${saturation}%, ${lightness}%), 
                  hsl(360, ${saturation}%, ${lightness}%))`,
              }}
            />
          </label>

          <label className="block text-sm text-zinc-950/70 dark:text-white/70">
            Saturation: {saturation}%
            <input
              type="range"
              min="0"
              max="100"
              value={saturation}
              onChange={(e) => setSaturation(Number.parseInt(e.target.value))}
              className="w-full h-2 bg-zinc-950/20 rounded-lg appearance-none cursor-pointer dark:bg-white/20 mt-1"
              style={{
                background: `linear-gradient(to right, 
                  hsl(${hue}, 0%, ${lightness}%), 
                  hsl(${hue}, 50%, ${lightness}%), 
                  hsl(${hue}, 100%, ${lightness}%))`,
              }}
            />
          </label>

          <label className="block text-sm text-zinc-950/70 dark:text-white/70">
            Lightness: {lightness}%
            <input
              type="range"
              min="0"
              max="100"
              value={lightness}
              onChange={(e) => setLightness(Number.parseInt(e.target.value))}
              className="w-full h-2 bg-zinc-950/20 rounded-lg appearance-none cursor-pointer dark:bg-white/20 mt-1"
              style={{
                background: `linear-gradient(to right, 
                  hsl(${hue}, ${saturation}%, 0%), 
                  hsl(${hue}, ${saturation}%, 50%), 
                  hsl(${hue}, ${saturation}%, 100%))`,
              }}
            />
          </label>
        </div>

        <div className="flex space-x-1">
          {colors.map((color, idx) => (
            <div
              key={idx}
              className="h-16 rounded flex-1 cursor-pointer transition-transform hover:scale-105"
              style={{ backgroundColor: color }}
              title={`Click to copy: ${color}`}
              onClick={() => copyToClipboard(color)}
            />
          ))}
        </div>

        <div className="text-sm font-mono text-zinc-950/70 dark:text-white/70">
          <p>
            Base color: hsl({hue}, {saturation}%, {lightness}%)
          </p>
        </div>
      </div>
    </div>
  )
}
```

O componente acima pode então ser usado em seus arquivos MDX assim:

```jsx
<ColorGenerator />
```

E o componente será renderizado como um componente React no arquivo MDX.

<ColorGenerator />

## Considerações

<AccordionGroup>
  <Accordion title="Impacto da Renderização do Lado do Cliente">
    Componentes de hook React renderizam do lado do cliente, o que tem várias implicações:

    * **SEO**: Os mecanismos de busca podem não indexar totalmente o conteúdo dinâmico
    * **Carregamento Inicial**: Os visitantes podem experimentar um flash de conteúdo carregando antes dos componentes renderizarem
    * **Acessibilidade**: Garanta que as mudanças de conteúdo dinâmico sejam anunciadas para leitores de tela
  </Accordion>

  <Accordion title="Melhores Práticas de Desempenho">
    * **Otimize Arrays de Dependência**: Inclua apenas dependências necessárias em seus`useEffect`arrays de dependência
    * **Memoize Cálculos Complexos**: Use`useMemo`ou`useCallback`para operações caras
    * **Reduzir Re-renderizações**: Divida componentes grandes em menores para evitar re-renderizações em cascata
    * **Carregamento Preguiçoso**: Considere o carregamento preguiçoso de componentes complexos para melhorar o tempo de carregamento inicial da página
  </Accordion>
</AccordionGroup>
