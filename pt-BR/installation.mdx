<Info>
  **Pré-requisito**: Por favor, instale o [Node.js](https://nodejs.org/en) (versão 19 ou superior) antes de prosseguir.
</Info>

**Passo 1**: Instale a [CLI](https://www.npmjs.com/package/mint):

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash yarn
  yarn global add mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

**Passo 2**: Navegue até o diretório dos documentos (onde o arquivo `docs.json` está localizado) e execute o seguinte comando:

```bash
mint dev
```

Alternativamente, se você não quiser instalar a CLI globalmente, você pode usar um script de execução única:

<CodeGroup>
  ```bash npm
  npx mint dev
  ```

  ```bash yarn
  yarn dlx mint dev
  ```

  ```bash pnpm
  pnpm dlx mint dev
  ```
</CodeGroup>

Uma prévia local da sua documentação estará disponível em `http://localhost:3000`.

## Atualizações

Cada lançamento da CLI está associado a uma versão específica da plataforma Mintlify. Se sua prévia local não estiver alinhada com a versão de produção, por favor atualize a CLI:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash yarn
  yarn global upgrade mint
  ```

  ```bash pnpm
  pnpm up --global mint
  ```
</CodeGroup>

## Portas Personalizadas

Por padrão, a CLI usa a porta 3000. Você pode personalizar a porta usando a flag `--port`. Para executar a CLI na porta 3333, por exemplo, use este comando:

```bash
mint dev --port 3333
```

Se você tentar executar em uma porta que já está em uso, ela usará a próxima porta disponível:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Comandos Adicionais

Embora `mint dev` seja o comando mais comumente usado, existem outros comandos que você pode usar para gerenciar sua documentação.

### Encontrar Links Quebrados

A CLI pode ajudar a validar links de referência feitos em sua documentação. Para identificar quaisquer links quebrados, use o seguinte comando:

```bash
mint broken-links
```

### Verificar Especificação OpenAPI

Você pode usar a CLI para verificar seu arquivo OpenAPI em busca de erros usando o seguinte comando:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Você pode passar um nome de arquivo (por exemplo, `./openapi.yaml`) ou uma URL (por exemplo, `https://petstore3.swagger.io/api/v3/openapi.json`).

### Renomeando Arquivos

Você pode renomear arquivos usando o seguinte comando:

```bash
mint rename <oldFilename> <newFilename>
```

Isso é uma melhoria em relação à atualização normal do nome do arquivo, pois também atualizará todas as referências ao arquivo, garantindo que não haja links quebrados.

## Formatação

Durante o desenvolvimento local, recomendamos usar extensões em seu IDE para reconhecer e formatar MDX.

Se você usa Cursor, Windsurf ou VSCode, recomendamos a [extensão MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) para realce de sintaxe, e [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) para formatação de código.

Se você usa JetBrains, recomendamos o [plugin MDX IntelliJ IDEA](https://plugins.jetbrains.com/plugin/14944-mdx) para destaque de sintaxe e configuração [Prettier](https://prettier.io/docs/webstorm) para formatação de código.

## Solução de problemas

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    Isso pode ser devido a uma versão desatualizada do node. Tente o seguinte:

    1. Remova a versão atualmente instalada do CLI do mint: `npm remove -g mint`
    2. Atualize para o Node v19 ou superior.
    3. Reinstale o CLI do mint: `npm install -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    Solution: Go to the root of your device and delete the \~/.mintlify folder. Afterwards, run `mint dev` novamente.
  </Accordion>
</AccordionGroup>
