Toda documentação requer um `docs.json` arquivo que contém as configurações principais. Este arquivo controla tudo, desde estilização e navegação até integrações e análises.

## Propriedades

### Personalização

<ResponseField name="theme" type="&#x22;mint&#x22; | &#x22;maple&#x22; | &#x22;palm&#x22;" required>

## Melhores Práticas

Ao configurar seu `docs.json` arquivo, considere estas melhores práticas:

1. Mantenha a configuração organizada agrupando configurações relacionadas
2. Use nomes significativos para grupos e páginas em sua estrutura de navegação
3. Forneça caminhos completos para todos os ativos (logos, favicons, etc.)
4. Teste sua configuração tanto no modo claro quanto no escuro
5. Verifique se todos os links externos e integrações estão configurados corretamente
6. Use contrastes de cores apropriados para acessibilidade
7. Configure as configurações de SEO para melhor visibilidade nos mecanismos de busca

## Validação

O `docs.json` arquivo é validado contra um esquema JSON para garantir a configuração adequada. Você pode referenciar o esquema incluindo:

```json
{
  "$schema": "https://mintlify.com/docs.json"
}
```
