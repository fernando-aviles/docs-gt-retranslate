Quando você altera o caminho de um arquivo na sua pasta docs, isso também mudará o caminho da URL para essa página. Isso pode acontecer ao reestruturar seus documentos ou alterar o título da barra lateral.

## Links Quebrados

Capture links quebrados com nossa CLI. Simplesmente [instale a CLI](/pt-BR/installation) e execute o comando:

```bash
mint broken-links
```

A CLI identificará quaisquer links relativos em seus documentos que não existam.

## Redirecionamentos

Configure redirecionamentos 301 adicionando o campo `redirects` no seu arquivo `docs.json`.

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

Isso redirecionará permanentemente `/source/path` para `/destination/path` para que você não perca nenhum SEO anterior da página original.

Para corresponder a um caminho curinga, use `*` após um parâmetro. Neste exemplo, `/beta/:slug*` corresponderá a `/beta/introduction` e o redireciona para `/v2/introduction`.

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
