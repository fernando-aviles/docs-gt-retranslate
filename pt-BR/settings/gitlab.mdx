Usamos uma combinação de Tokens de Acesso e Webhooks para autenticar e sincronizar
alterações entre o GitLab e o Mintlify.

* Usamos Tokens de Acesso para extrair informações do GitLab.
* Usamos Webhooks para que o GitLab possa notificar o Mintlify quando alterações são feitas.
  * Isso permite que o Mintlify crie implantações de preview quando um MR é criado.

## Configure a conexão

<Steps>
  <Step title="Find your project ID">
    Dentro do seu projeto GitLab, navegue até `Settings` > `General` e encontre o `Project ID`.

    <Frame>
      <img src="/images/gitlab/gitlab-project-id.png" />
    </Frame>
  </Step>

  <Step title="Generate an access token">
    a. Navegue até `Settings` > `Access Tokens`.

    b. Selecione `Add new token`.

    1. Nomeie o token como "Mintlify".
    2. Se você tem um repositório privado, você deve definir a função como `Maintainer`.
    3. Escolha `api` e `read_api` para os escopos.

    c. Finalmente clique em `Create project access token` e copie o token.

    <Frame>
      <img src="/images/gitlab/gitlab-project-access-token.png" />
    </Frame>
  </Step>

  <Step title="Set up the connection">
    Dentro do [painel Mintlify](https://dashboard.mintlify.com/settings/deployment/git-settings), adicione o ID do projeto e o token de acesso das etapas anteriores junto com as outras configurações. Clique em "Salvar Alterações" quando terminar.

    <Frame>
      <img src="/images/gitlab/gitlab-config.png" />
    </Frame>
  </Step>
</Steps>

## Criar o webhook

Webhooks nos permitem receber eventos quando alterações são feitas para que possamos
automaticamente acionar implantações.

<Steps>
  <Step title="Navigate to Settings > Webhooks and click 'Add new Webhook'">
    <Frame>
      <img src="/images/gitlab/gitlab-webhook.png" />
    </Frame>
  </Step>

  <Step title="Set up URL and webhook">
    No campo "URL", insira o endpoint `https://leaves.mintlify.com/gitlab-webhook` e nomeie o webhook como "Mintlify".
  </Step>

  <Step title="Paste token">
    Cole o token do Webhook gerado após configurar a conexão.

    <Frame>
      <img src="/images/gitlab/gitlab-show-webtoken.png" />
    </Frame>
  </Step>

  <Step title="Select events">
    Selecione os eventos que você quer que acionem o webhook:

    * Eventos de push (Todas as branches)
    * Eventos de merge requests

    Quando terminar, deve parecer com isso:

    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken.png" />
    </Frame>
  </Step>

  <Step title="Test the Webhook">
    Depois de criar o Webhook, clique no menu suspenso "Testar" e selecione "Eventos de push" para enviar uma carga útil de exemplo para garantir que está configurado corretamente. Ele dirá "Hook executado com sucesso: HTTP 200" se estiver configurado corretamente.

    Isso ajudará você a verificar se tudo está funcionando corretamente e se sua documentação será sincronizada adequadamente com seu repositório GitLab.

    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken-test.png" />
    </Frame>
  </Step>
</Steps>
