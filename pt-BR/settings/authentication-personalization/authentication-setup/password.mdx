<Info>
  Esta é a documentação para o Handshake de Senha de**Autenticação**. O
  Handshake de Senha não está disponível para Personalização.
</Info>

Se você não tem requisitos de segurança rigorosos, ou não quer gerenciar um
banco de dados de leitores de documentação, você pode usar um conjunto de senhas compartilhadas para
proteger seus documentos.

<Note>
  A autenticação está disponível apenas mediante solicitação. Por favor{" "}
  <a href="mailto:sales@mintlify.com">entre em contato com nossa equipe de vendas</a> para acesso.
  Depois que tivermos habilitado o acesso, você pode seguir as instruções de implementação.
</Note>

## Implementação

<Steps>
  <Step title="Adicionar uma senha">
    Vá para o seu [dashboard
    configurações](https://dashboard.mintlify.com/products/authentication) e crie
    uma senha.
  </Step>

  <Step title="Compartilhe sua senha">
    Compartilhe a senha de forma segura com os leitores da sua documentação. É isso!
  </Step>
</Steps>

## Exemplo

Quero configurar a autenticação para meus documentos hospedados em `docs.foo.com`. Não quero
ter que ficar controlando quem pode e quem não pode acessar os documentos. Meu principal caso de uso para
autenticação é impedir que concorrentes bisbilhotem - o compartilhamento de senha é seguro
o suficiente para minha equipe.

Para configurar a autenticação com o Mintlify, vou ao meu dashboard do Mintlify e adiciono pelo
menos uma senha. Em seguida, compartilho essa senha, junto com a URL dos documentos privados,
com potenciais clientes.
