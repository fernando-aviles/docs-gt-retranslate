## Introdução

O editor web é uma interface visual para criar, editar e revisar documentação diretamente no seu navegador.

O editor web oferece uma experiência **What-You-See-Is-What-You-Get (WYSIWYG)** enquanto mantém a sincronização com seu repositório Git, o que permite que você veja atualizações em tempo real e colabore com sua equipe nas mudanças da documentação.

### Editor web vs. CLI

O editor web permite que você escreva e edite sua documentação no seu navegador sem precisar de ferramentas de desenvolvimento local ou usar a linha de comando. Você deve usar o editor web se quiser manter sua documentação em um só lugar com uma única ferramenta.

O CLI é uma ferramenta de linha de comando que permite criar e gerenciar sua documentação localmente usando o IDE de sua escolha. Você deve usar o CLI se quiser integrar a documentação ao seu fluxo de trabalho de desenvolvimento existente.

Tanto o editor web quanto o CLI são totalmente integrados com seu repositório Git, então você pode usá-los de forma intercambiável e diferentes membros da sua equipe podem usar qualquer ferramenta com base em suas preferências.

## Modos do Editor

O editor web oferece dois modos para acomodar diferentes preferências e necessidades de edição.

Você pode alternar entre os modos naturais a qualquer momento usando o botão no canto superior direito da barra de ferramentas do editor.

O modo visual fornece uma experiência WYSIWYG onde as alterações que você faz no editor são as alterações que serão publicadas no seu site de documentação. Este modo é ideal para quando você quer ver como suas alterações ficarão em tempo real.

## Fazendo Alterações

1. **Navegar pelos arquivos**: Use o explorador de arquivos na barra lateral para navegar pela sua documentação.
2. **Abrir um arquivo**: Clique no arquivo que você deseja editar para abri-lo no editor.
3. **Fazer alterações**: Edite o conteúdo usando o modo visual ou Markdown. As alterações são automaticamente salvas como rascunhos.
4. **Visualizar alterações**: Veja como suas alterações aparecerão no modo visual.

## Publicação

<Info>

## Branches

Branches permitem que você trabalhe em alterações na documentação sem atualizar a versão principal até que você esteja pronto para que suas alterações entrem em vigor.

<Tip>
  Use branches para atualizações significativas, novas seções ou quando vários membros da equipe estão trabalhando em diferentes partes da documentação simultaneamente.
</Tip>

### Criando um Branch

1. Selecione o nome do branch na barra de ferramentas do editor (geralmente `main` por padrão).
2. Selecione **Novo Branch**.
3. Digite um nome descritivo para o seu branch.
4. Selecione **Criar Branch**.

### Alternando Branches

1. Selecione o nome do branch atual na barra de ferramentas do editor.
2. Selecione o branch para o qual você deseja mudar no menu suspenso.

### Salvando Alterações em um Branch

Para salvar suas alterações em um branch, selecione o botão **Salvar Alterações**botão no canto superior direito do editor.

Quando você está trabalhando em um branch, suas alterações não são salvas automaticamente.

## Pull Requests

Pull requests (ou PRs) permitem que você e outras pessoas revisem as alterações que você fez em um branch e depois mesclem essas alterações em sua documentação principal.

### Criando um Pull Request em um Branch

1. Faça suas alterações em um branch.
2. Clique no**Publish Pull Request**botão no canto superior direito do editor.
3. Adicione um Título e Descrição do Pull Request para o seu pull request. Um bom título e descrição ajudarão os revisores a entender as alterações que você fez.
4. Clique em**Publish Pull Request**.

### Revisando Pull Requests

Você pode revisar pull requests na sua plataforma Git (GitHub, GitLab).

Depois de criar um pull request, você pode ver uma implantação de preview das alterações.

Depois que um revisor aprovar um pull request, você pode mesclar o pull request em seu branch principal e as alterações serão implantadas em seu site de documentação ao vivo.

## Sincronização Git

O editor web se integra ao seu repositório Git, garantindo que todas as alterações sejam devidamente versionadas e rastreadas.

### Como funciona a sincronização Git

* **Autenticação**: O editor web se conecta ao seu repositório Git através do nosso[GitHub App](/pt-BR/settings/github)ou[GitLab integration](/pt-BR/settings/gitlab).

* **Busca automática**: Quando você abre o editor, ele busca automaticamente o conteúdo mais recente do branch principal do seu repositório.

* **Rastreamento de alterações**: Conforme você faz edições, o editor web rastreia as alterações e pode commitá-las em seu repositório.

* **Branching**: Você pode fazer alterações diretamente no seu branch principal ou em um branch separado, dependendo das suas preferências de fluxo de trabalho.

* **Pull requests**: Para fluxos de trabalho colaborativos, você pode criar pull requests a partir do editor web.

## Terminologia Git

Entender os seguintes termos pode ajudá-lo a trabalhar de forma mais eficaz com o editor web e o fluxo de trabalho Git.

<AccordionGroup>
  <Accordion title="Repositório">
    Um repositório (ou repo) é onde seus arquivos de documentação são armazenados, junto com seu histórico de revisão. O editor web se conecta ao seu repositório Git para buscar e armazenar o conteúdo da documentação.
  </Accordion>

  <Accordion title="Commit">
    Um commit é um instantâneo das alterações em sua documentação em um ponto específico no tempo. Quando você publica alterações no editor web, você está criando um commit em seu repositório Git.
  </Accordion>

  <Accordion title="Branch">
    Um branch é uma versão paralela da sua documentação que permite que você trabalhe em alterações sem afetar a versão principal. O editor web permite que você crie e alterne entre branches.
  </Accordion>

  <Accordion title="Pull Request">
    Um pull request (ou PR) é uma proposta para mesclar alterações de um branch para outro, tipicamente de um branch de feature para o branch principal. PRs facilitam a revisão e discussão antes que as alterações sejam incorporadas.
  </Accordion>

  <Accordion title="Diff">
    Um diff (ou diferença) mostra as alterações específicas entre duas versões de um arquivo. Ao revisar pull requests, diffs destacam o que foi adicionado, removido ou modificado.
  </Accordion>
</AccordionGroup>

## Solução de problemas

Aqui estão soluções para problemas comuns que você pode encontrar com o editor web.

<AccordionGroup>
  <Accordion title="Alterações não aparecem após a publicação">
    **Possíveis causas:**

    * A implantação ainda está em andamento
    * Problemas de cache no seu navegador

    **Solutions:**

    1. Verifique o status da implantação no seu Painel Mintlify.
    2. Tente atualizar forçadamente seu navegador (Ctrl+F5 ou Cmd+Shift+R).
    3. Limpe o cache do seu navegador.
  </Accordion>

  <Accordion title="Erros de permissão ao publicar">
    **Possíveis causas:**

    * Permissões insuficientes para o repositório Git
    * Problemas de autenticação com seu provedor Git

    **Solutions:**

    1. Verifique se você tem o acesso correto ao repositório.
    2. Verifique se sua integração Git está configurada corretamente.
    3. Revise a documentação de[Editor Permissions](/pt-BR/advanced/dashboard/permissions).
  </Accordion>

  <Accordion title="Problemas de carregamento do editor">
    **Possíveis causas:**

    * Problemas de conectividade de rede
    * Repositórios de documentação grandes

    **Solutions:**

    1. Verifique sua conexão com a internet.
    2. Atualize a página e tente novamente.
    3. Entre em contato com o suporte se o problema persistir.
  </Accordion>
</AccordionGroup>
