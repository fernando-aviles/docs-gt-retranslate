Existem dois tipos de campos: Campos de Parâmetro e Campos de Resposta.

## Campo de Parâmetro

Um componente`ParamField`é usado para definir os parâmetros para suas APIs ou SDKs. Adicionar um`ParamField`adicionará automaticamente um[API Playground](/pt-BR/api-playground/overview).

<ParamField path="param" type="string" required>
  Um exemplo de um campo de parâmetro
</ParamField>

<RequestExample>
  ```jsx Path Example
  <ParamField path="param" type="string">
    An example of a parameter field
  </ParamField>
  ```

  ```jsx Body Example
  <ParamField body="user_age" type="integer" default="0" required>
    The age of the user. Cannot be less than 0
  </ParamField>
  ```

  ```jsx Response Example
  <ResponseField name="response" type="string" required>
    A response field example
  </ResponseField>
  ```
</RequestExample>

### Props

<ParamField body="query, path, body, or header" type="string">
  Se é um parâmetro de consulta, caminho, corpo ou cabeçalho seguido pelo nome
</ParamField>

<ParamField body="type" type="string">
  Tipo esperado do valor do parâmetro

  Suporta`number`,`string`,`bool`,`object`.

  Arrays podem ser definidos usando o sufixo`[]`. Por exemplo`string[]`.
</ParamField>

<ParamField body="required" type="boolean">
  Indica se o parâmetro é obrigatório
</ParamField>

<ParamField body="deprecated" type="boolean">
  Indica se o parâmetro está obsoleto
</ParamField>

<ParamField body="default" type="string">
  Valor padrão usado pelo servidor se a solicitação não fornecer um valor
</ParamField>

<ParamField body="initialValue" type="any">
  Valor que será usado para inicializar o playground
</ParamField>

<ParamField body="placeholder" type="string">
  Texto de espaço reservado para a entrada no playground
</ParamField>

<ParamField body="children" type="string">
  Descrição do parâmetro (markdown habilitado)
</ParamField>

## Campo de Resposta

O componente`<ResponseField>`é projetado para definir os valores de retorno de uma API. Muitas documentações também usam`<ResponseField>`em páginas quando você precisa listar os tipos de algo.

<ResponseField name="response" type="string" required>
  Um exemplo de campo de resposta
</ResponseField>

```jsx
<ResponseField name="response" type="string" required>
  A response field example
</ResponseField>
```

### Props

<ResponseField name="name" type="string" required>
  O nome do valor de resposta.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipo esperado do valor de resposta - isso pode ser qualquer string arbitrária.
</ResponseField>

<ResponseField name="default" type="string">
  O valor padrão.
</ResponseField>

<ResponseField name="required" type="boolean">
  Mostrar "required" ao lado do nome do campo.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Se um campo está obsoleto ou não.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Rótulos que são mostrados antes do nome do campo
</ResponseField>

<ResponseField name="post" type="string[]">
  Rótulos que são mostrados após o nome do campo
</ResponseField>
