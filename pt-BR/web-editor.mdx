O Web Editor é a forma preferida de editar documentação diretamente sem precisar abrir seu IDE ou executar`mintlify dev`.

O editor inclui alguns recursos principais para integrar diretamente ao seu fluxo de trabalho git existente,
como criar branches, pull requests, commits e diffs para suas alterações atuais.

Também inclui uma experiência totalmente editável para alterar e adicionar conteúdo diretamente,
mesmo com componentes personalizados.

Se você já entende os fluxos de trabalho git e nossas integrações, você pode pular para[aqui](#editing-content).

## Fluxos de trabalho Git e atualizações

### Noções básicas do Git

Embora o Web Editor signifique que você não precisa ir ao GitHub ou sua linha de comando para fazer
alterações, ainda é útil conhecer o básico do git.

Terminologia do Git:

* **Repositório**: A pasta onde seu código reside. Pode ser local (no seu computador) ou remoto (como GitHub).

* **Commit**: Um instantâneo das alterações feitas nos arquivos do repositório.

* **Branch**: Uma linha separada de desenvolvimento. É uma cópia de trabalho do código que permite que você trabalhe em alterações sem afetar a versão principal.

* **Pull request:** Uma solicitação para mesclar alterações de um branch de trabalho no branch principal. Isso é usado para revisar conteúdo antes de tornar as alterações ativas.

## Modos do editor

Para oferecer a maior flexibilidade possível, o editor possui três modos:

### Editor Visual

O **"Editor Visual"** é o modo WYSIWYG típico que você veria em algo como o Notion. Ele oferece uma visualização dos seus documentos de uma forma totalmente editável que reflete como a página final ficaria no seu site principal de documentação.

### Editor de Código-Fonte

O **"Editor de Código-Fonte"** oferece uma maneira de editar seus arquivos MDX em código, da mesma forma que você faria em seu IDE. Isso oferece menos flexibilidade, já que nossos componentes não estão disponíveis para autocompletar, mas tem duas vantagens únicas.

1. Permite que você edite propriedades dos componentes (veja nossas [limitações abaixo](#current-limitations)) que atualmente não está disponível no modo **"Editor Visual"** ainda.

2. Permite que você corrija erros de sintaxe que possam ter aparecido em seu repositório que podem não ser compatíveis com o modo **"Editor Visual"** até que você os corrija.

### Visualização de Diferenças

A **"Visualização de Diferenças"** é uma maneira de visualizar as alterações feitas em um documento específico antes de enviá-lo para seu repositório.

Isso ajudará você a ver as alterações que fez junto com as alterações de formatação feitas pelo editor.

## Limitações atuais

Temos algumas limitações atuais no editor que estamos trabalhando para resolver.

<AccordionGroup>
  <Accordion title="WYSIWYG navigation tree">
    Atualmente, você não pode atualizar sua navegação em tempo real com base em arquivos adicionados/editados. Você ainda precisa editar manualmente os arquivos renomeados, adicionados e excluídos em seu`docs.json`
  </Accordion>

  <Accordion title="Snippet previews">
    Atualmente, não mostramos nenhuma prévia para snippets personalizados. Isso está em nosso roteiro para suportar como componentes totalmente editáveis.
  </Accordion>

  <Accordion title="OpenAPI previews">
    Atualmente, não mostramos nenhuma prévia para especificações OpenAPI. Isso está em nosso roteiro para suportar como uma prévia somente leitura.
  </Accordion>
</AccordionGroup>
