Web Editorは、IDEを開いたり実行したりすることなく、直接文書を編集する推奨される方法です`mintlify dev`。

エディターには、ブランチの作成、プルリクエスト、コミット、現在の変更の差分など、既存のgitワークフローに直接統合するための主要な機能が含まれています。

また、カスタムコンポーネントでも、コンテンツを直接変更および追加できる完全な編集機能も含まれています。

gitワークフローと私たちの統合についてすでに理解している場合は、[こちら](#editing-content)にスキップできます。

## Gitと更新ワークフロー

### Gitの基本

Web Editorを使用すれば変更を加えるためにGitHubやコマンドラインに行く必要はありませんが、gitの基本を知っておくと役立ちます。

Git用語：

* **リポジトリ**：コードが存在するフォルダです。ローカル（あなたのコンピュータ上）またはリモート（GitHubなど）にあります。

* **コミット**：リポジトリ内のファイルに加えられた変更のスナップショットです。

* **ブランチ**：開発の独立した流れです。メインバージョンに影響を与えることなく変更作業ができるコードの作業コピーです。

* **プルリクエスト：**&#x4F5C;業ブランチからメインブランチへの変更のマージを要求するものです。これは変更を本番環境に反映する前にコンテンツをレビューするために使用されます。

文書を更新するために、gitワークフローと統合するための特定のボタンをいくつか用意しています。

</Note>

## エディターモード

最大限の柔軟性を提供するため、エディターには3つのモードがあります：

<Frame caption="View mode toggle">
  <img className="block dark:hidden" src="/images/editor/mode-toggle-light.png" />

  <img className="hidden dark:block" src="/images/editor/mode-toggle-dark.png" />
</Frame>

### ビジュアルエディター

&#xNAN;**「ビジュアルエディター」**&#x306F;、Notionのような典型的なWYSIWYGモードです。メインのドキュメントサイトでの最終的なページの見た目を反映した、完全に編集可能な形式でドキュメントを表示します。

### ソースエディター

&#xNAN;**「ソースエディター」**&#x3067;は、IDEと同じようにコードでMDXファイルを編集できます。コンポーネントのオートコンプリートは利用できませんが、2つの独自の利点があります。

1. コンポーネントのプロパティを編集できます（[以下の制限事項を参照](#current-limitations)）。これは現&#x5728;**「ビジュアルエディター」**&#x30E2;ードではまだ利用できません。

2. リポジトリに発生した可能性のある、**「ビジュアルエディター」**&#x30E2;ードと互換性のない構文エラーを修正できます。

### 差分表示

&#xNAN;**「差分表示」**&#x306F;、リポジトリにコミットする前に特定のドキュメントに加えられた変更を確認する方法です。

これにより、エディターによって行われた書式の変更とともに、あなたが加えた変更を確認できます。

## 現在の制限事項

エディターには現在、解決に取り組んでいるいくつかの制限があります。

<AccordionGroup>
  <Accordion title="WYSIWYG navigation tree">
    現在、追加/編集されたファイルに基づいてナビゲーションをリアルタイムで更新することはできません。名前が変更された、追加された、削除されたファイルは、まだ手動で編集する必要があります。`docs.json`
  </Accordion>

  <Accordion title="Snippet previews">
    現在、カスタムスニペットのプレビューは表示されません。これは完全に編集可能なコンポーネントとしてサポートするロードマップに含まれています。
  </Accordion>

  <Accordion title="OpenAPI previews">
    現在、OpenAPI仕様のプレビューは表示されません。これは読み取り専用のプレビューとしてサポートするロードマップに含まれています。
  </Accordion>
</AccordionGroup>
