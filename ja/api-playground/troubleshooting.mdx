APIページは複雑です。そのため、多くの問題が発生する可能性があります。
ここでは、お客様がよく遭遇する一般的な問題をリストアップしています：

<AccordionGroup>
  <Accordion title="All of my OpenAPI pages are completely blank">
    このシナリオでは、MintlifyがOpenAPIドキュメントを見つけられないか、
    OpenAPIドキュメントが無効である可能性があります。

    ローカルで`mintlify dev`を実行すると、これらの問題が明らかになるはずです。

    OpenAPIドキュメントが検証に合格することを確認するには：

    1. こちらの[バリデーター](https://editor.swagger.io/)
    2. 「テキストを検証」タブに切り替え
    3. OpenAPIドキュメントを貼り付け
    4. 「検証する！」をクリック

    下に表示されるテキストボックスが緑色の枠で囲まれている場合、ドキュメントは検証に合格しています。これはMintlifyがOpenAPIドキュメントを検証するために使用している正確な検証パッケージなので、ここで検証に合格すれば、問題は他の箇所にある可能性が高いです。

    また、MintlifyはOpenAPI 2.0をサポートしていません。ドキュメントがこのバージョンの仕様を使用している場合、この問題が発生する可能性があります。以下のサイトでドキュメントを変換できます：[editor.swagger.io](https://editor.swagger.io/) (Edit > Convert to OpenAPI 3から):

    <Frame>
      ![](/images/convert-oas-3.png)
    </Frame>
  </Accordion>

  <Accordion title="One of my OpenAPI pages is completely blank">
    これは通常、ページメタデータの`openapi`フィールドのスペルミスが原因です。HTTPメソッドとパスがOpenAPIドキュメントのHTTPメソッドとパスと完全に一致していることを確認してください。

    問題が発生する可能性のある例を示します：

    ```md get-user.mdx
    ---
    openapi: "GET /users/{id}/"
    ---
    ```

    ```yaml openapi.yaml
    paths:
      "/users/{id}":
        get: ...
    ```

    注意してください。`openapi`フィールドのパスには末尾にスラッシュがありますが、OpenAPIドキュメントにはありません。

    もう一つの一般的な問題は、ファイル名のスペルミスです。`openapi`フィールドで特定のOpenAPIドキュメントを指定する場合は、ファイル名が正しいことを確認してください。例えば、2つのOpenAPIドキュメントがある場合`openapi/v1.json`と`openapi/v2.json`、メタデータは次のようになります：

    ```md api-reference/v1/users/get-user.mdx
    ---
    openapi: "v1 GET /users/{id}"
    ---
    ```
  </Accordion>

  <Accordion title="Requests from the API Playground don't work">
    カスタムドメインを設定している場合、これはリバースプロキシの問題である可能性があります。デフォルトでは、API Playgroundを通じて行われるリクエストは`POST`リクエストから始まります
    `/api/request`パスでドキュメントサイトに。リバースプロキシが`GET`
    リクエストのみを許可するように設定されている場合、これらのリクエストはすべて失敗します。これを修正するには、リバースプロキシを設定して`POST`リクエストを`/api/request`パスに許可してください。

    あるいは、リバースプロキシが`POST`リクエストを受け付けない場合、`api.playground.proxy`設定を`docs.json`で、[ここ](/settings/global#api-configurations)に説明されているように設定することで、Mintlifyがリクエストを直接バックエンドに送信するように設定できます。これらのリクエストはユーザーのブラウザから直接送信されるため、サーバーでCORSを設定する必要がある可能性が高いです。
  </Accordion>
</AccordionGroup>
