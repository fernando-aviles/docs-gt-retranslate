---
title: MDX Setup
description: MDXを使用してAPIエンドポイントのドキュメントページを生成する
---

Mintlifyでは、`docs.json`設定、MDXメタデータフィールド、および`<ParamFields />`コンポーネントを組み合わせてAPIエンドポイントを定義できます。定義されたエンドポイントから、APIプレイグラウンド、リクエスト例、レスポンス例を生成します。

<Steps>
  <Step title="Configure your API">
    あなたの`docs.json`ファイルで、ベースURLと認証方法を定義します：

    ```json
     "api": {
      "mdx": {
        "server": "https://mintlify.com/api", // string array for multiple base URLs
        "auth": {
          "method": "key",
          "name": "x-api-key" // options: bearer, basic, key.
        }
      }
    }
    ```

    APIプレイグラウンドを表示したくない場合は、認証タイプを含める必要はありません。以下のフィールドでプレイグラウンドを非表示にできます：

    ```json
    "api": {
      "playground": {
        "display": "none"
      }
    }
    ```

    API設定の完全なリストは[こちら](/settings/global#param-api)をご覧ください。
  </Step>

  <Step title="Create your endpoint pages">
    各APIエンドポイントページには、対応するMDXファイルが必要です。各ファイルの先頭で以下を定義します：

    ```md
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    ---
    ```

    パラメータ名を`{}`で囲んでパスに追加することで、パスパラメータを指定できます：

    ```bash
    https://api.example.com/v1/endpoint/{userId}
    ```

    <Note>
      もし`server`が[docs.json](/settings/global)で設定されている場合、`/v1/endpoint`のような相対パスを使用できます。
    </Note>

    また、MDXファイルの先頭に`playground`を追加することで、ページごとにAPIプレイグラウンドのグローバルに定義された表示モードを上書きすることもできます：

    ```md
    ---
    title: 'Create new user'
    api: 'POST https://api.mintlify.com/user'
    playground: 'none'
    ```
  </Step>

  <Step title="Add your endpoints to your docs">
    あなたの`navigation`フィールドに`docs.json`パスを追加して、エンドポイントページをサイドバーに追加します。ドキュメントの構造化について[こちら](/settings/navigation)で詳しく学べます。
  </Step>
</Steps>
