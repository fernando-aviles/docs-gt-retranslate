## APIの説明

OpenAPIドキュメントの学習と構築のための優れたオンラインツールが多数あります。以下は私たちのお気に入りです：

* [Swagger's OpenAPIガイド](https://swagger.io/docs/specification/about/)：OpenAPI構文に慣れるため
* [OpenAPI v3.1.0仕様](https://github.com/OAI/OpenAPI-Specification/blob/main/versions/3.1.0.md)：最新のOpenAPI仕様に関するすべての詳細
* [Swagger & OpenAPIバリデーター](https://editor.swagger.io/)：OpenAPIドキュメントのデバッグ用
* [Swaggerのエディター](https://editor.swagger.io/)：実際の例を確認するため

<Note>
  SwaggerのOpenAPIガイドはOpenAPI v3.0向けですが、ほぼすべての情報がv3.1にも適用されます。v3.0とv3.1の違いについての詳細は、[OpenAPIのブログ投稿](https://www.openapis.org/blog/2021/02/16/migrating-from-openapi-3-0-to-3-1-0)をご覧ください。
</Note>

## APIのURLの指定

OpenAPIドキュメントでは、異なるAPIエンドポイントはそのパスで指定されます。例えば`/users/{id}`、あるいは単に`/`。これらのパスを追加するべきベースURLを指定するために、OpenAPIは`servers`フィールドを提供します。このフィールドは、API Playgroundなどのいくつかのmintlifyの機能を使用するために必要です。`servers`フィールドの設定方法については[Swaggerのドキュメント](https://swagger.io/docs/specification/api-host-and-base-path/)をご覧ください。

API PlaygroundはこれらのサーバーURLを使用してリクエストの送信先を決定します。複数のサーバーが指定されている場合、サーバー間を切り替えるためのドロップダウンが表示されます。サーバーが指定されていない場合、リクエストを送信する方法がないため、API Playgroundはシンプルモードを使用します。

APIの異なるエンドポイントが異なるURLに存在する場合、[特定のパスや操作に対してサーバーフィールドを上書き](https://swagger.io/docs/specification/api-host-and-base-path/#:~:text=%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%C2%A0%20%2D%20southeastasia-,Overriding%20Servers,-The%20global%20servers)することができます。

## 認証の指定

ほとんどすべてのAPIには何らかの認証方法が必要です。OpenAPIは`securitySchemes`フィールドを提供し、APIで使用される認証方法を定義します。最も一般的な認証タイプ -[Basic](https://swagger.io/docs/specification/authentication/basic-authentication/)、[Bearer](https://swagger.io/docs/specification/authentication/bearer-authentication/)、および[API Keys](https://swagger.io/docs/specification/authentication/api-keys/)に対してシンプルな設定を提供します。これらの認証方法をエンドポイントに適用するために、OpenAPIは`security`フィールドを使用します。認証の定義と適用の構文は少し直感的ではないので、必ず[Swaggerのドキュメントと例](https://swagger.io/docs/specification/authentication/)を確認してください。

APIの説明とAPI PlaygroundはOpenAPIドキュメントのセキュリティ設定に基づいて認証フィールドを追加します。

APIの異なるエンドポイントで異なる認証方法が必要な場合、[特定の操作に対してセキュリティフィールドを上書き](https://swagger.io/docs/specification/authentication/#:~:text=you%20can%20apply%20them%20to%20the%20whole%20API%20or%20individual%20operations%20by%20adding%20the%20security%20section%20on%20the%20root%20level%20or%20operation%20level%2C%20respectively.)することができます。
