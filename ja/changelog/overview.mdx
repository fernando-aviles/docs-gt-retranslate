<Update label="February 2025">
  ## 新しい設定スキーマ`docs.json`

  <Frame>
    ![docs.json screenshot](/images/changelog/docsjson.png)
  </Frame>

  新しい`docs.json`スキーマを導入し、`mint.json`に代わるものとして、より良いマルチレベルバージョニング、より簡単な視覚的理解、より一貫した用語をサポートします。変更点の詳細については、[ブログをチェックしてください](https://mintlify.com/blog/refactoring-mint-json-into-docs-json)。

  以下の手順で`mint.json`から`docs.json`にアップグレードしてください：

  1. CLIが最新バージョンであることを確認

  ```
  npm i mintlify@latest -g
  ```

  1. ドキュメントリポジトリで実行

  ```
  mintlify upgrade
  ```

  1. 古いmint.jsonファイルを削除して変更をプッシュ

  ## CIチェック

  リンク切れの検出、スペルや文法の問題の発見、独自のVale設定による文章スタイルの強制など、ドキュメントを自動的にリントします。詳細は[ドキュメント](settings/ci)をご覧ください。

  ## LLM向け.mdサポート

  すべてのドキュメントページが自動的にプレーンなMarkdownファイルとして利用可能になりました—URLに`.md`を追加するだけです。これによりLLMがドキュメントの個別ページを取り込みやすくなります。

  ## より多くのテーマ

  <Frame>
    ![graphic with text "Themes v2"](/images/changelog/themes.png)
  </Frame>

  新しい[事前構築されたテーマ](/themes)でドキュメントの見た目と雰囲気を変更できます。[docs.jsonファイル](/settings/global)で設定してください。

  現在利用可能：

  * Maple
  * Palm
  * Willow

  ## その他の改善点

  * [技術文書作成ガイド：](https://mintlify.com/guides/introduction)オーディエンス調査、コンテンツタイプ、執筆のヒントを含む、技術文書作成のベストプラクティス。
  * [ドロップダウンコンポーネント](/navigation/divisions#dropdowns)：タブやアンカーに加えて、ドロップダウンでナビゲーションを整理。
  * [AI構文修正機能](https://x.com/ricardonunez_io/status/1892334887644123192)：ウェブエディタが構文エラーを検出し、AIを使用して修正を提案します。

  ## コード行のハイライト

  言語識別子の後に特別なコメントを追加することで、重要な部分を強調して注目を集めるために、ドキュメント内のコード行をハイライトできるようになりました。中括弧`{}`を使用し、カンマで区切られた行番号または範囲を指定します。

  ```javascript Line Highlighting Example {1,3-5}
  const greeting = "Hello, World!";
  function sayHello() {
    console.log(greeting);
  }
  sayHello();
  ```

  ````md
  ```javascript Line Highlighting Example {1,3-5}
  const greeting = "Hello, World!";
  function sayHello() {
    console.log(greeting);
  }
  sayHello();
  ```
  ````

  ## ライトモードのコードブロック

  コードブロックにライトモードのバリアントが追加されました。以下を`mint.json`に追加することで有効にできます：

  ```json
  "codeBlock": {
    "mode": "auto"
  }
  ```
