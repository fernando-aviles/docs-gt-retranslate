## はじめに

ようこそ！以下の手順に従って、Mintlifyでドキュメントのデプロイ、更新、
機能強化の方法を学びましょう。

### リポジトリの作成

Mintlifyのドキュメントは、MDXファイルと設定から生成され、
[starter kit](https://github.com/mintlify/starter)で定義されています。GitHubを使用して
ドキュメントとコードを統合し、ソース管理を簡単にします。[dashboard](https://dashboard.mintlify.com)からオンボードするか、[starter kit](https://github.com/mintlify/starter)をクローンして始めましょう。

<AccordionGroup>
  <Accordion title="Install the GitHub app" icon="message-bot">
    変更をプッシュした際に自動的にデプロイされるようにするには、GitHubアプリをインストールしてください。インストールリンクは[dashboard](https://dashboard.mintlify.com/settings)の設定ページにあります。インストールが成功すると、リポジトリのコミットハッシュの横にチェックマークが表示されます。

  </Accordion>

### コンテンツの更新

Mintlifyを使用すると、ドキュメントのスタイル、構造、コンテンツを簡単にカスタマイズできます。

<AccordionGroup>
  <Accordion title="Editing locally (recommended for developers)" icon="rectangle-terminal">
    1. インストール[git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git)。
    2. gitがインストールされたら、`git clone <your-repo>`を使用してドキュメントリポジトリをクローンします。まだセットアップしていない場合は、[SSH keys](https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent)を使用してセットアップするのがよいでしょう。
    3. お好みのIDEでリポジトリを開きます。
    4. Mintlify CLIをインストールして変更をプレビューします`npm i -g mintlify`。

    詳しくは[local development guide](/development)をご覧ください。
  </Accordion>

  <Accordion title="Editing with the web editor" icon="pen-to-square">
    ウェブエディタの使用方法については、[guide](/web-editor)をご覧ください。
  </Accordion>

  <Accordion title="Customizing to your brand" icon="palette">
    色、ロゴ、ボタンなどの設定を`docs.json`ファイルで簡単にカスタマイズできます。以下の基本設定から始めましょう：

    ```json
    "name": "Your Company"
    "logo": {
      "light": "/logo/light.svg",
      "dark": "/logo/dark.svg",
      "href": "https://yourcompany.com"
    },
    "favicon": "/favicon.svg",
    "colors": {
      "primary": "#2AB673",
      "light": "#55D799",
      "dark": "#117866",
    },
    ```

    サポートされている設定の完全なリストは[global settings](/settings/global)で確認できます。
  </Accordion>

  <Accordion title="Adding content" icon="pencil">
    シンプルなMDXファイルでコンテンツを追加します。以下のテンプレートでページを開始します：

    ```md
    ---
    title: "Page Title"
    sidebarTitle: "Sidebar title (optional - if different from page title)"
    description: "Subtitle (optional)"
    ---
    ```

    画像、表、リストなどの追加については、[MDX syntax](/text)で詳しく説明しています。また、[wide array of components](/content/components)も提供しています。
  </Accordion>

  <Accordion title="Pushing changes" icon="rocket">
    準備ができたら、変更をコミットしてプッシュし、ドキュメントサイトを更新します。[guide](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)でその方法を説明しています。GitHubアプリが変更を正常にデプロイできない場合は、[dashboard](https://dashboard.mintlify.com)から手動でドキュメントを更新できます。

    <Frame>
      ![Manual
      Update](https://mintlify.s3-us-west-1.amazonaws.com/mintlify/images/manual-update.png)
    </Frame>
  </Accordion>

  <Accordion title="Generating API references" icon="code-branch">
    OpenAPIドキュメントを使用してAPIリファレンスを簡単にセットアップできます。

    1. 追加する`openapi.yaml`または`openapi.json`ファイルをドキュメントリポジトリに入れるか、`openapi`フィールドを`docs.json`でURLとして定義します。

    ```json
    "openapi": "link-to-your-openapi-file"
    ```

    2. 私たちの[scraper](/api-playground/openapi/setup#autogenerate-files-recommended)を使用してOpenAPIエンドポイントファイルを自動生成します：

    ```bash
    npx @mintlify/scraping@latest openapi-file <path-or-url-to-openapi-file>
    ```

    3. 最後に、生成されたエンドポイントMDXファイルを`docs.json`の下に`navigation`含めます。

    MintlifyとOpenAPIの使用に関する完全なガイドは[this guide](/api-playground/openapi/setup)を[This guide](/api-playground/openapi/writing-openapi)では、APIの認証方法の設定方法を説明しています。手動でAPIリファレンスを定義する場合は、[our syntax](/api-playground/overview)をご覧ください。
  </Accordion>

  <Accordion title="Adding analytics" icon="chart-simple">
    私たちの社内アナリティクスは、ページビュー、検索分析、セッション記録などの洞察を提供します。これらは[dashboard](https://dashboard.mintlify.com/analytics)でアクセスできます。

    また、様々なアナリティクスプロバイダーとの統合もサポートしています。プロバイダーのリストは[here](/integrations/analytics/overview)で確認できます。
  </Accordion>
</AccordionGroup>
