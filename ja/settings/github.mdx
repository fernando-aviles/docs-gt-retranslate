MintlifyはGitHub APIと統合され、ドキュメントとGitHubリポジトリ間の同期を可能にします。この統合は[GitHub Apps](https://docs.github.com/en/developers/apps/getting-started-with-apps/about-apps#about-github-apps)を通じて実現されています。

## インストール

まず、ドキュメントリポジトリが存在するGitHubアカウントにMintlify GitHub Appをインストールする必要があります。GitHub Appのインストールには、組織の所有権またはリポジトリの管理者権限が必要です。必要な権限がない場合は、リポジトリの所有者が承認する必要があります。インストールページには[Mintlifyダッシュボード](https://dashboard.mintlify.com)からアクセスできます。

GitHub Appsでは、単一のリポジトリにのみ権限を付与することができます。ドキュメントがホストされているリポジトリへのアクセスのみが必要なため、そのようにすることを強く推奨します。

## 権限

GitHub appのインストール時に、特定の権限を付与するよう求められます：

* 読み取りと書き込みアクセス：`checks`、`contents`、`deployments`、`pull requests`、および`workflows`

これらの権限は、ドキュメントを管理する際にシームレスな体験を提供するために活用されます。

ドキュメントのデプロイメントブランチとして設定したブランチにコミットを行うと、変更されたファイルの内容を取得してドキュメントを更新します。これを実現するために`contents`への読み取りアクセスが必要です。

プルリクエストが作成されると、チェックとプレビューデプロイメントを作成するため、`checks`と`deployments`への書き込みアクセスが必要です。

Mintlifyウェブエディタ内では、Mintlifyがあなたに代わってブランチとプルリクエストを作成するため、`pull requests`と`workflows`への書き込みアクセスが必要です。

<Info>
  書き込み権限について懸念がある場合、GitHub Appはアクセスを許可したリポジトリにのみアクセスできます*また*ブランチ保護が設定されている場合*できません*ブランチに直接プッシュすることは - ブランチ保護ルールに従います。
</Info>

## リポジトリのインストール

GitHub appをインストールする際、すべてのリポジトリまたはその一部を選択するよう求められます。この選択は[GitHub appの設定](https://github.com/apps/mintlify/installations/new)でいつでも変更できます。
