  この機能は有料プランのお客様とGitHubでのみご利用いただけます。他のプラットフォームへのサポートは近日公開予定です。
</Warning>

CIチェックを使用してドキュメントのエラーをリントし、デプロイ前に警告を表示します。

## インストール

開始するには、[GitHub](/settings/github)ページの手順に従う必要があります。

GitHub Appsの場合、単一のリポジトリにのみ権限を付与することができます。
ドキュメントがホストされているリポジトリへのアクセスのみが必要なため、
そのようにすることを強くお勧めします。

## 設定

Mintlifyダッシュボードの「Add-Ons」タブでデプロイメントのCIチェックを設定できます。そこで実行したいチェックを有効または無効にすることができます。

チェックを有効にする際、`Warning`または`Blocking`レベルで実行するかを選択できます。

<Note>
  `Blocking`レベルのチェックは、パスしない場合や変更が提案される場合に失敗ステータスを提供します。<br />

  `Warning`レベルのチェックは、エラーや提案があっても決して失敗ステータスを提供しません。
</Note>

## 実行のタイミング

CIチェックは、設定されたデプロイメントブランチへのコミット時、またはそのブランチに対するプルリクエスト時に実行されるように設定されています。

## 利用可能なCIチェック

### リンク切れ

ローカルマシンでの[CLIリンクチェッカー](/settings/broken-links#broken-links)と同様に、ドキュメントのリンク切れを自動的にチェックします。
このチェックの結果を確認するには、特定のコミットに対するGitHubのチェック結果ページにアクセスできます。

### Vale

[Vale](https://vale.sh/)は、MarkdownやMDXを含む様々な文書タイプをサポートするオープンソースのルールベースの文章リンターです。

MintlifyはValeをCIチェックで自動的に実行し、結果をチェックステータスとして表示することをサポートしています。

#### 設定

デプロイメントのコンテンツディレクトリのルートに`.vale.ini`ファイルがある場合、その設定ファイルを自動的に使用します。
指定された`stylesPath`内の設定ファイルも自動的に使用します。

<Tip>Vale設定がない、または始め方がわからない場合でも心配いりません。Mintlifyには、設定が提供されていない場合に自動的に使用されるデフォルトの設定があります。</Tip>

<Warning>
  セキュリティ上の理由により、絶対`stylesPath`、または`stylesPath`に含まれる`..`値はサポートできません。相対パスを使用し、`stylesPath`をリポジトリに含めてください。
</Warning>

#### パッケージ

Valeは様々な[パッケージ](https://vale.sh/docs/keys/packages)をサポートしており、スペルやスタイルのエラーをチェックするために使用できます。
正しい`stylesPath`の下にリポジトリに含まれるパッケージは、自動的にインストールされVale設定で使用されます。

リポジトリに含まれていないパッケージについては、[Valeパッケージレジストリ](https://vale.sh/explorer)からパッケージを指定することができ、自動的にダウンロードされVale設定で使用されます。

<Warning>
  セキュリティ上の理由により、[Valeパッケージレジストリ](https://vale.sh/explorer)以外からのパッケージの自動ダウンロードはサポートできません。
</Warning>

#### ValeとMDX

ValeはネイティブではMDXをサポートしていませんが、Valeの作者が[カスタム拡張機能](https://github.com/errata-ai/MDX)を提供しています。

この拡張機能を使用したくない場合は、`.vale.ini`ファイルに以下の行を追加することをお勧めします：

```ini
[formats]
mdx = md

[*.mdx]
CommentDelimiters = {/*, */}

TokenIgnores = (?sm)((?:import|export) .+?$), \
(?<!`)(<\w+ ?.+ ?\/>)(?!`), \
(<[A-Z]\w+>.+?<\/[A-Z]\w+>)

BlockIgnores = (?sm)^(<\w+\n .*\s\/>)$, \
(?sm)^({.+.*})
```

Valeのドキュメント内コメントを使用するには、MDXスタイルのコメントを使用してください`{/* ... */}`。
もし`CommentDelimiters = {/*, */}`[設定](https://vale.sh/docs/keys/commentdelimiters)を設定ファイルで使用している場合、Valeはリンティング中にこれらのコメントを自動的に解釈します。
これにより、行やセクションのスキップなど、Valeの組み込み機能を簡単に使用できます。

```mdx
{/* vale off */}

This text will be ignored by Vale

{/* vale on */}
```

を使用しない場合`CommentDelimiters`、しかしValeのコメントを使用することを選択する場合は、ValeコメントをMDXコメントでラップする必要があります`{/* ... */}`。例えば：

```mdx
{/* <!-- vale off --> */}

This text will be ignored by Vale

{/* <!-- vale on --> */}
```

これらのコメントタグはMintlifyコンポーネント内ではサポートされていませんが、ドキュメントのベースレベルであればどこでも使用できます。
