この`navigation` プロパティはドキュメントの階層構造を制御します。

## ページ

階層構造が不要な場合は、`navigation` フィールド内にページを定義するだけです。

配列の各エントリは`pages`リポジトリ内に存在するファイルへのパスである必要があります。
ファイルパスに`.mdx`を付ける必要はありません。

```json
{
  "navigation": {
    "pages": [
      "overview",
      "quickstart",
      "advanced/components",
      "advanced/integrations"
    ]
  }
}
```

## グループ

グループを使用してページをグループ化できます。グループは互いにネスト化することもできます。

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting Started",
        "pages": [
          "quickstart",
          {
            "group": "Editing",
            "icon": "pen-paintbrush",
            "pages": ["development", "web-editor"]
          }
        ]
      },
      {
        "group": "Writing Content",
        "pages": ["writing-content/page", "writing-content/text"]
      }
    ]
  }
}
```

## タブ

タブは、ドキュメントの異なるトピックやセクションを区別するのに役立ちます。

<Frame>
  <img className="block dark:hidden rounded-md" src="/images/tabs-light.png" />

  <img className="hidden dark:block rounded-md" src="/images/tabs-dark.png" />
</Frame>

```json docs.json
"navigation": {
  "tabs": [
    {
      "tab": "API References",
      "pages": [
        "api-reference/get",
        "api-reference/post",
        "api-reference/delete"
      ]
    },
    {
      "tab": "SDKs",
      "pages": [
        "sdk/fetch",
        "sdk/create",
        "sdk/delete",
      ]
    },
    {
      "tab": "Blog",
      "href": "https://external-link.com/blog"
    }
  ]
}
```

## アンカー

アンカーはコンテンツをセクション分けする別の方法です。

<Frame>
  <img className="block dark:hidden rounded-md" src="/images/anchor-light.png" />

  <img className="hidden dark:block rounded-md" src="/images/anchor-dark.png" />
</Frame>

設定はタブの設定と非常によく似ています。

```json docs.json
"navigation": {
  "anchors": [
    {
      "anchor": "API References",
      "pages": [
        "api-reference/get",
        "api-reference/post",
        "api-reference/delete"
      ]
    },
    {
      "anchor": "SDKs",
      "pages": [
        "sdk/fetch",
        "sdk/create",
        "sdk/delete",
      ]
    },
    {
      "anchor": "Blog",
      "href": "https://external-link.com/blog"
    }
  ]
}
```

## グローバルリンク

外部リンクをリストするためだけにタブやアンカーを使用したい場合は、`navigation.global`プロパティを使用します。

```json
"navigation": {
  "global": {
    "anchors": [
      {
        "anchor": "Slack",
        "icon": "slack",
        "href": "https://slack.com
      },
      {
        "anchor": "API Reference",
        "icon": "code",
        "href": "https://api-reference.com"
      }
    ]
  }
}
```

### 非表示ページ

に含まれていないMDXファイルは`docs.json`サイドバーには表示されませんが、直接リンクすることでアクセスできます。

非表示ページは、デフォルトではドキュメント内の検索用にインデックス化されません。この動作をオーバーライドしたい場合は、
[`seo.indexing`](/settings/global#param-indexing)属性を`docs.json`に`navigable`を設定できます。

## ナビゲーションバー

### リンク

トップバーにリンクを追加するには、`links`フィールドを`docs.json`ファイルで使用します。

<Frame>
  <img className="block rounded-md" src="/images/topbar-links.png" />
</Frame>

この`links`フィールドは以下のフィールドをサポートしています：`label`、`href`。

```json
"navbar": {
  "links": [
    {
      "label": "Community",
      "href": "https://mintlify.com/community"
    }
  ]
}
```

### プライマリボタン

トップバーのプライマリボタンをカスタマイズするには、`primary`
フィールドを使用します。

<Frame>
  <img className="block rounded-md" src="/images/topbar-cta-get-started.png" />
</Frame>

この`primary`フィールドは以下のフィールドをサポートしています：`label`、`href`、`type`。これらのフィールドのオプションについての詳細は、[docs.jsonスキーマ](/settings/global#param-navbar)を参照してください。

```json
"navbar": {
  "primary": {
    "type": "button",
    "label": "Get Started",
    "href": "https://mintlify.com/get-started"
  }
}
```

#### GitHub

CTAボタンをGitHubリポジトリに直接リンクするように設定することもできます。`primary`フィールドを`type`を`github`に設定して使用します。

<Frame>
  <img className="block rounded-md" src="/images/topbar-cta-github.png" />
</Frame>

```json
"navbar": {
  "primary": {
    "type": "github",
    "label": "GitHub",
    "href": "https://github.com/mintlify/docs"
  }
}
```
