## プロパティ

### カスタマイズ

<ResponseField name="theme" type="&#x22;mint&#x22; | &#x22;maple&#x22; | &#x22;palm&#x22;" required>
  プロジェクトのレイアウトテーマ。例：
  [Maple](https://maple.mintlify.app/)、[Palm](https://palm.mintlify.app/)、
</ResponseField>

<ResponseField name="name" type="string" required>
  プロジェクト、組織、または製品の名前 最小長：1
</ResponseField>

<ResponseField name="description" type="string">

</ResponseField>

### スタイリング

<ResponseField name="colors" type="object" required>
  ドキュメントで使用する色。最低でも、プライマリカラーを定義する必要があります。例：

  ```json
  {
    "colors": {
      "primary": "#ff0000"
    }
  }
  ```

  <Expandable title="Colors">
    <ResponseField name="primary" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$" required>

      パターンに一致する必要があります：^#(\[a-fA-F0-9]{6}|\[a-fA-F0-9]{3})\$
    </ResponseField>

    <ResponseField name="light" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">

    <ResponseField name="dark" type="string matching ^#([a-fA-F0-9]{6}|[a-fA-F0-9]{3})$">

    </ResponseField>
  </Expandable>
</ResponseField>

## ベストプラクティス

設定ファイル`docs.json`を設定する際は、以下のベストプラクティスを考慮してください：

1. 関連する設定をグループ化して設定を整理する
2. ナビゲーション構造のグループとページに意味のある名前を使用する
3. すべてのアセット（ロゴ、ファビコンなど）の完全なパスを提供する
4. ライトモードとダークモードの両方で設定をテストする
5. すべての外部リンクとインテグレーションが正しく設定されていることを確認する
6. アクセシビリティのための適切なカラーコントラストを使用する
7. 検索エンジンの可視性を向上させるためのSEO設定を構成する

## バリデーション

この`docs.json`ファイルは、適切な設定を確保するためにJSONスキーマに対して検証されます。以下を含めることでスキーマを参照できます：

```json
{
  "$schema": "https://mintlify.com/docs.json"
}
```

            詳細は[プレイグラウンドガイド](/api-playground)
          </ResponseField>

          <ResponseField name="disableProxy" type="boolean" default={false}>
            デフォルトでは、APIプレイグラウンドのリクエストはMintlifyによってプロキシされます。この設定でこの動作を無効にすることができます。

            ファイルアップロードなど、特定のリクエストタイプに必要です。
          </ResponseField>
        </Expandable>
      </ResponseField>

      <ResponseField name="request" type="object">
        APIリクエストの設定

        <Expandable title="Request">
          <ResponseField name="example" type="object">
            自動生成されるAPIリクエスト例の設定

            <Expandable>
              <ResponseField name="languages" type="string[]">
                自動生成されるリクエスト例の言語の順序を決定する文字列の配列。[x-codeSamples](/api-playground/openapi/advanced-features#x-codesamples)を使用してカスタム言語を定義するか、デフォルトの言語を使用できます。デフォルトの言語には`bash`、`python`、`javascript`、`php`、`go`、`java`
              </ResponseField>
            </Expandable>
          </ResponseField>
        </Expandable>
      </ResponseField>

      <ResponseField name="paramFields" type="ApiParamFields">
        APIプレイグラウンドのパラメータフィールドの設定

        <Expandable title="ApiParamFields">
          <ResponseField name="expanded" default="none" type="&#x22;all&#x22; | &#x22;topLevel&#x22; | &#x22;topLevelOneOfs&#x22; | &#x22;none&#x22;">
            APIプレイグラウンドの展開可能なオプションのデフォルトの展開状態。

            `"all"` - すべての展開可能なコンポーネントが展開される

            `"topLevel"` - すべてのトップレベルの展開可能なコンポーネントが展開される

            `"topLevelOneOfs"` - すべてのトップレベルの`oneOf`タイプが展開される

            `"none"` - すべての展開可能なコンポーネントが閉じている（デフォルトの動作）
          </ResponseField>
        </Expandable>
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="openapi" type="string | string[]">
    OpenAPIファイルを指すURLまたは相対パスの文字列、または文字列の配列。

    例：

    <CodeGroup>
      ```json Absolute
      "openapi": "https://example.com/openapi.json"
      ```

      ```json Relative
      "openapi": "/openapi.json"
      ```

      ```json Multiple
      "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
      ```
    </CodeGroup>
  </ResponseField>

  ### インテグレーション

  <ResponseField name="integrations" type="Integrations">
    サードパーティのインテグレーションを追加するための設定（アナリティクスインテグレーションを除く）

    <Expandable title="Integrations">
      <ResponseField name="intercom" type="string">
        ドキュメントサイトでIntercomウィジェットを有効にします。値はIntercom App IDを設定してください。
      </ResponseField>

      <ResponseField name="frontchat" type="string">
        ドキュメントサイトでFrontchatウィジェットを有効にします。値はFrontchat App IDを設定してください。
      </ResponseField>
    </Expandable>
  </ResponseField>

  <ResponseField name="analytics" type="Analytics">
    サードパーティのアナリティクスインテグレーションを追加するための設定。サポートされているアナリティクスの完全なリストは[こちら](/integrations/analytics/overview)をご覧ください。
  </ResponseField>

  ### リダイレクト

  <ResponseField name="redirects" type="Redirect[]">
    別のパスに永続的にリダイレクトするように設定したいパスの配列

    例：

    ```json
    "redirects": [
      {
        "source": "/source/path",
        "destination": "/destination/path"
      }
    ]
    ```

    <Expandable title="Redirect">
      <ResponseField name="source" type="string">
        リダイレクト元のパス。

        例：`/source`
      </ResponseField>

      <ResponseField name="destination" type="string">
        リダイレクト先のパス。

        例：`/destination`
      </ResponseField>
    </Expandable>
  </ResponseField>

  </ResponseField>
