docsフォルダ内のファイルのパスを変更すると、そのページのURLのパスも変更されます。これは、ドキュメントの再構成やサイドバータイトルの変更時に発生する可能性があります。

## 壊れたリンク

CLIで壊れたリンクを検出します。単に[CLIをインストール](/development)して、次のコマンドを実行します：

```bash
mintlify broken-links
```

CLIは、ドキュメント内の存在しない相対リンクを特定します。

## リダイレクト

301リダイレクトを設定するには、`redirects`フィールドを`docs.json`ファイルに追加します。

```json
"redirects": [
  {
    "source": "/source/path",
    "destination": "/destination/path"
  }
]
```

これにより`/source/path`から`/destination/path`に永続的にリダイレクトされ、元のページのSEOが失われることはありません。

ワイルドカードパスにマッチさせるには、`*`をパラメータの後に使用します。この例では、`/beta/:slug*`は`/beta/introduction`にマッチし、`/v2/introduction`にリダイレクトします。

```json
"redirects": [
  {
    "source": "/beta/:slug*",
    "destination": "/v2/:slug*"
  }
]
```
