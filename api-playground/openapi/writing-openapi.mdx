## Describing your API

Key resources for OpenAPI:
- [Swagger's OpenAPI Guide](https://swagger.io/docs/specification/about/) - Learn OpenAPI syntax
- [OpenAPI v3.1.0 Specification](https://github.com/OAI/OpenAPI-Specification/blob/main/versions/3.1.0.md) - Latest specification details
- [Swagger & OpenAPI Validator](https://apitools.dev/swagger-parser/online/) - Debug OpenAPI documents
- [Swagger's Editor](https://editor.swagger.io/) - See examples in action

<Note>
  Most Swagger guide content applies to OpenAPI v3.1. See differences in [OpenAPI's blog post](https://www.openapis.org/blog/2021/02/16/migrating-from-openapi-3-0-to-3-1-0).
</Note>

## URL Configuration

Use the `servers` field to specify your API's base URL. This enables Mintlify features like API Playground. Multiple servers can be configured with server overrides available for specific endpoints. [Learn more](https://swagger.io/docs/specification/api-host-and-base-path/).

## Authentication

Configure authentication using `securitySchemes` for Basic, Bearer, and API Key methods. Apply these using the `security` field. Supports both global and endpoint-specific authentication settings. [See documentation](https://swagger.io/docs/specification/authentication/) for implementation details.
