Learn how to use OpenAPI 3's advanced features with Mintlify.

## Schema Combinations: `oneOf`, `anyOf`, `allOf`

OpenAPI provides keywords to combine schemas:
- `oneOf`: exclusive-or operator
- `anyOf`: or operator
- `allOf`: and operator

<Warning>Mintlify treats `oneOf` and `anyOf` identically for practical purposes.</Warning>

<Info>The `not` keyword is not currently supported.</Info>

### Using `allOf`

Mintlify combines properties of object schemas when using `allOf`. Example using OpenAPI components:

```yaml
org_with_users:
  allOf:
    - $ref: '#/components/schemas/Org'
    - type: object
      properties:
        users:
          type: array
          description: An array containing all users in the organization

components:
  schemas:
    Org:
      type: object
      properties:
        id:
          type: string
          description: The ID of the organization
```

### Using `oneOf` and `anyOf`

Use the `title` field to name your options. Mintlify displays them in tabs:

```yaml
delivery_address:
  oneOf:
    - title: StreetAddress
      type: object
      properties:
        address_line_1:
          type: string
          description: The street address
    - title: POBox
      type: object
      properties:
        box_number:
          type: string
          description: The PO Box number
```

## `x-codeSamples`

Add SDK examples to your API documentation using the `x-codeSamples` extension:

<ParamField body="lang" type="string" required>
  Code sample language
</ParamField>

<ParamField body="label" type="string">
  Sample label (for multiple examples)
</ParamField>

<ParamField body="source" type="string" required>
  Source code
</ParamField>

Example:
```yaml
paths:
  /plants:
    get:
      x-codeSamples:
        - lang: bash
          source: |
            planter list -u
        - lang: javascript
          source: |
            const planter = require('planter');
            planter.list({ unwatered: true });
```
