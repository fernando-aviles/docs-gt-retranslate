## Add an OpenAPI specification file

Add a valid OpenAPI document (JSON or YAML format) following the [OpenAPI specification](https://swagger.io/specification/) version 3.0+.

## Integration Options

### 1. Auto-populate API pages

Add an `openapi` field to your `mint.json` under a tab or anchor. This can be either a local file path or URL:

```json
{
  "anchors": [
    {
      "name": "API Reference",
      "openapi": "/path/to/openapi.json",
      "url": "api-reference",
      "icon": "square-terminal"
    }
  ]
}
```

<Tip>Validate your OpenAPI spec with: `mintlify openapi-check <openapiFilenameOrUrl>`</Tip>

### 2. Custom MDX Files

For more control, create individual MDX files for each API operation. You can:
- Generate files automatically using our scraper:
  ```bash
  npx @mintlify/scraping@latest openapi-file <path-to-openapi-file> -o api-reference
  ```
- Create files manually using the `openapi` field:
  ```md
  ---
  title: "Get users"
  openapi: "/path/to/openapi.json GET /users"
  ---
  ```

### 3. OpenAPI Schemas

Create pages for individual schemas using:
```md
---
openapi-schema: "schema-key"
---
```

<Note>
  Ensure methods and paths match exactly with your OpenAPI document.
</Note>
