## Add an OpenAPI Specification

Start with a valid OpenAPI 3.0+ document in JSON or YAML format that follows the [OpenAPI specification](https://swagger.io/specification/).

## Quick Setup with Auto-Population

The simplest way to start is by adding an `openapi` field to your `mint.json`. You can either:
- Use a local path to your OpenAPI document
- Link to a hosted OpenAPI URL

Mintlify will automatically generate pages for all API operations.

**Using Anchors:**
```json {5}
{
  "anchors": [
    {
      "name": "API Reference",
      "openapi": "/path/to/openapi.json",
      "url": "api-reference",
      "icon": "square-terminal"
    }
  ]
}
```

**Using Tabs:**
```json {6}
{
  "tabs": [
    {
      "name": "API Reference",
      "url": "api-reference",
      "openapi": "https://petstore3.swagger.io/api/v3/openapi.json"
    }
  ]
}
```

<Tip>Validate your OpenAPI spec with: `mintlify openapi-check <openapiFilenameOrUrl>`</Tip>

Auto-generated pages will use:
- `title`: OpenAPI operation summary or generated from method/endpoint
- `description`: OpenAPI operation description
- `version`: Version from anchor/tab if available

## Custom MDX Pages

For more control over your API documentation, create individual MDX pages for each operation. This allows you to:
- Customize page metadata
- Add extra content
- Select specific operations
- Control page ordering

### Generate MDX Files Automatically

For large APIs, use our scraper to generate MDX files:

```bash
npx @mintlify/scraping@latest openapi-file <path-to-openapi-file> -o api-reference
```

### Manual MDX Creation

Create individual MDX pages by referencing OpenAPI operations in the frontmatter:

```md
---
title: "Get users"
openapi: "/path/to/openapi.json GET /users"
---
```

### OpenAPI Schema Pages

Create pages for schema definitions using:

```md
---
openapi-schema: SchemaName
---
```
