Web Editor ist die bevorzugte Methode, um Dokumente direkt zu bearbeiten, ohne dass Sie Ihre IDE öffnen oder ausführen müssen`mintlify dev`.

Der Editor enthält einige wichtige Funktionen zur direkten Integration in Ihren bestehenden Git-Workflow, wie das Erstellen von Branches, Pull Requests, Commits und Diffs für Ihre aktuellen Änderungen.

Er bietet auch eine vollständig bearbeitbare Umgebung zum Ändern und Hinzufügen von Inhalten direkt, sogar mit benutzerdefinierten Komponenten.

Wenn Sie Git-Workflows und unsere Integrationen bereits verstehen, können Sie direkt[hierher](#editing-content).

## Git und Update-Workflows

### Git-Grundlagen

Auch wenn der Web Editor bedeutet, dass Sie nicht zu GitHub oder Ihrer Kommandozeile gehen müssen, um Änderungen vorzunehmen, ist es dennoch hilfreich, die Grundlagen von Git zu kennen.

Git-Terminologie:

* **Repository**: Der Ordner, in dem Ihr Code liegt. Er kann lokal (auf Ihrem Computer) oder remote (wie GitHub) sein.

* **Commit**: Eine Momentaufnahme der Änderungen, die an Dateien im Repository vorgenommen wurden.

* **Branch**: Eine separate Entwicklungslinie. Es ist eine Arbeitskopie des Codes, die es Ihnen ermöglicht, an Änderungen zu arbeiten, ohne die Hauptversion zu beeinflussen.

* **Pull Request:** Eine Anfrage, Änderungen von einem Arbeitsbranch in den Hauptbranch zu übernehmen. Dies wird verwendet, um Inhalte zu überprüfen, bevor Änderungen live geschaltet werden.

## Editor-Modi

Um maximale Flexibilität zu bieten, hat der Editor drei Modi:

<Frame caption="View mode toggle">
  <img className="block dark:hidden" src="/images/editor/mode-toggle-light.png" />

  <img className="hidden dark:block" src="/images/editor/mode-toggle-dark.png" />
</Frame>

### Visueller Editor

Der **"Visuelle Editor"** ist der typische WYSIWYG-Modus, den Sie von Programmen wie Notion kennen. Er bietet Ihnen eine vollständig bearbeitbare Ansicht Ihrer Dokumente, die dem endgültigen Aussehen auf Ihrer Haupt-Dokumentationsseite entspricht.

### Quelltext-Editor

Der **"Quelltext-Editor"** bietet Ihnen die Möglichkeit, Ihre MDX-Dateien im Code zu bearbeiten, genau wie in Ihrer IDE. Dies bietet weniger Flexibilität, da unsere Komponenten nicht für die automatische Vervollständigung verfügbar sind, hat aber zwei einzigartige Vorteile.

1. Er ermöglicht Ihnen die Bearbeitung von Komponenten-Props (siehe unsere [Einschränkungen unten](#current-limitations)), was derzeit nicht verfügbar ist im **"Visuellen Editor"**-Modus.

2. Er ermöglicht Ihnen die Korrektur von Syntaxfehlern, die möglicherweise in Ihrem Repository aufgetreten sind und die nicht kompatibel sind mit dem **"Visuellen Editor"**-Modus, bis Sie sie behoben haben.

### Diff-Ansicht

Die **"Diff-Ansicht"** ist eine Möglichkeit, die Änderungen an einem bestimmten Dokument anzuzeigen, bevor Sie es in Ihr Repository übertragen.

Dies hilft Ihnen, sowohl Ihre eigenen Änderungen als auch die vom Editor vorgenommenen Formatierungsänderungen zu sehen.

## Aktuelle Einschränkungen

Wir haben derzeit einige Einschränkungen im Editor, an deren Behebung wir arbeiten.

<AccordionGroup>
  <Accordion title="WYSIWYG navigation tree">
    Sie können derzeit die Navigation nicht live aktualisieren basierend auf hinzugefügten/bearbeiteten Dateien. Sie müssen umbenannte, hinzugefügte und gelöschte Dateien weiterhin manuell in Ihrer`docs.json`
  </Accordion>

  <Accordion title="Snippet previews">
    Wir zeigen derzeit keine Vorschauen für benutzerdefinierte Snippets an. Dies steht auf unserer Roadmap zur Unterstützung als vollständig bearbeitbare Komponenten.
  </Accordion>

  <Accordion title="OpenAPI previews">

  </Accordion>
</AccordionGroup>
