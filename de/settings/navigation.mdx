Die `navigation` Eigenschaft steuert die Hierarchie Ihrer Dokumentation.

## Seiten

Wenn Sie keine Hierarchie wünschen, können Sie einfach Seiten in Ihrem `navigation` Feld definieren.

Jeder Eintrag im `pages` Array muss ein Pfad zu einer Datei sein, die in Ihrem Repository existiert.
Beachten Sie, dass Sie nicht `.mdx` an die Dateipfade anhängen müssen.

```json
{
  "navigation": {
    "pages": [
      "overview",
      "quickstart",
      "advanced/components",
      "advanced/integrations"
    ]
  }
}
```

## Gruppen

Gruppen ermöglichen es Ihnen, Ihre Seiten zu gruppieren. Gruppen können auch ineinander verschachtelt werden.

```json
{
  "navigation": {
    "groups": [
      {
        "group": "Getting Started",
        "pages": [
          "quickstart",
          {
            "group": "Editing",
            "icon": "pen-paintbrush",
            "pages": ["development", "web-editor"]
          }
        ]
      },
      {
        "group": "Writing Content",
        "pages": ["writing-content/page", "writing-content/text"]
      }
    ]
  }
}
```

## Tabs

Tabs helfen dabei, zwischen verschiedenen Themen oder Abschnitten Ihrer
Dokumentation zu unterscheiden.

<Frame>
  <img className="block dark:hidden rounded-md" src="/images/tabs-light.png" />

  <img className="hidden dark:block rounded-md" src="/images/tabs-dark.png" />
</Frame>

```json docs.json
"navigation": {
  "tabs": [
    {
      "tab": "API References",
      "pages": [
        "api-reference/get",
        "api-reference/post",
        "api-reference/delete"
      ]
    },
    {
      "tab": "SDKs",
      "pages": [
        "sdk/fetch",
        "sdk/create",
        "sdk/delete",
      ]
    },
    {
      "tab": "Blog",
      "href": "https://external-link.com/blog"
    }
  ]
}
```

## Anker

Anker sind eine weitere Möglichkeit, Ihren Inhalt zu gliedern.

<Frame>
  <img className="block dark:hidden rounded-md" src="/images/anchor-light.png" />

  <img className="hidden dark:block rounded-md" src="/images/anchor-dark.png" />
</Frame>

Die Konfiguration ist der Tab-Konfiguration sehr ähnlich.

```json docs.json
"navigation": {
  "anchors": [
    {
      "anchor": "API References",
      "pages": [
        "api-reference/get",
        "api-reference/post",
        "api-reference/delete"
      ]
    },
    {
      "anchor": "SDKs",
      "pages": [
        "sdk/fetch",
        "sdk/create",
        "sdk/delete",
      ]
    },
    {
      "anchor": "Blog",
      "href": "https://external-link.com/blog"
    }
  ]
}
```

## Verschachtelte Hierarchie

Sie können sowohl Anker als auch Tabs zusammen verwenden - beide können beliebig ineinander verschachtelt werden.

## Globale Links

Wenn Sie Tabs oder Anker ausschließlich zum Auflisten externer Links verwenden möchten, nutzen Sie die `navigation.global` Eigenschaft.

```json
"navigation": {
  "global": {
    "anchors": [
      {
        "anchor": "Slack",
        "icon": "slack",
        "href": "https://slack.com
      },
      {
        "anchor": "API Reference",
        "icon": "code",
        "href": "https://api-reference.com"
      }
    ]
  }
}
```

### Versteckte Seiten

MDX-Dateien, die nicht in `docs.json` enthalten sind, werden nicht in der Seitenleiste angezeigt, sind aber
über direkte Links zugänglich.

Versteckte Seiten werden standardmäßig nicht für die Suche in Ihrer Dokumentation indexiert. Wenn Sie
dieses Verhalten überschreiben möchten, können Sie das [`seo.indexing`](/settings/global#param-indexing) Attribut
in Ihrer `docs.json` auf `navigable` setzen.

## Navigationsleiste

### Links

Fügen Sie Links zur oberen Leiste mit dem `links` Feld in der `docs.json` Datei hinzu.

<Frame>
  <img className="block rounded-md" src="/images/topbar-links.png" />
</Frame>

Das `links` Feld unterstützt die folgenden Felder: `label`, `href`.

```json
"navbar": {
  "links": [
    {
      "label": "Community",
      "href": "https://mintlify.com/community"
    }
  ]
}
```

### Primärer Button

Passen Sie den primären Button in der oberen Leiste mit dem `primary`
Feld an.

<Frame>
  <img className="block rounded-md" src="/images/topbar-cta-get-started.png" />
</Frame>

Das `primary` Feld unterstützt die folgenden Felder: `label`, `href`, `type`. Weitere Informationen zu den Optionen für diese Felder finden Sie im [docs.json Schema](/settings/global#param-navbar).

```json
"navbar": {
  "primary": {
    "type": "button",
    "label": "Get Started",
    "href": "https://mintlify.com/get-started"
  }
}
```

#### GitHub

Sie können den CTA-Button auch so konfigurieren, dass er direkt zu Ihrem GitHub
Repository verlinkt. Verwenden Sie das `primary` Feld mit dem `type` auf `github` gesetzt.

<Frame>
  <img className="block rounded-md" src="/images/topbar-cta-github.png" />
</Frame>

```json
"navbar": {
  "primary": {
    "type": "github",
    "label": "GitHub",
    "href": "https://github.com/mintlify/docs"
  }
}
```
