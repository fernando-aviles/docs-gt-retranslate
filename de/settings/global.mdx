Jede Dokumentationsseite benötigt eine`docs.json` Datei, die die grundlegenden Konfigurationseinstellungen enthält. Diese Datei steuert alles von Styling und Navigation bis hin zu Integrationen und Analysen.

<ConfigUpgrade />

## Eigenschaften

### Anpassung

<ResponseField name="theme" type="&#x22;mint&#x22; | &#x22;maple&#x22; | &#x22;palm&#x22;" required>
  Das Layout-Theme des Projekts. Beispiele:
  [Maple](https://maple.mintlify.app/), [Palm](https://palm.mintlify.app/),
</ResponseField>

<ResponseField name="name" type="string" required>
  Der Name des Projekts, der Organisation oder des Produkts. Mindestlänge: 1
</ResponseField>

<ResponseField name="description" type="string">
  Optionale Beschreibung für SEO und LLM-Indexierung
</ResponseField>

### Styling

<ResponseField name="colors" type="object" required>
  Die Farben, die in Ihrer Dokumentation verwendet werden sollen. Sie müssen mindestens die Primärfarbe definieren. Zum Beispiel:

  ```json
  {
    "colors": {
      "primary": "#ff0000"
    }
  }
  ```

  <Expandable title="Colors">

</ResponseField>

## Beste Praktiken

Bei der Konfiguration Ihrer `docs.json` Datei beachten Sie diese bewährten Praktiken:

1. Halten Sie die Konfiguration übersichtlich, indem Sie zusammengehörige Einstellungen gruppieren
2. Verwenden Sie aussagekräftige Namen für Gruppen und Seiten in Ihrer Navigationsstruktur
3. Geben Sie vollständige Pfade für alle Assets an (Logos, Favicons, etc.)
4. Testen Sie Ihre Konfiguration sowohl im hellen als auch im dunklen Modus
5. Überprüfen Sie, ob alle externen Links und Integrationen korrekt konfiguriert sind
6. Verwenden Sie angemessene Farbkontraste für die Barrierefreiheit
7. Konfigurieren Sie SEO-Einstellungen für bessere Suchmaschinenoptimierung

## Validierung

Die `docs.json` Datei wird gegen ein JSON-Schema validiert, um die korrekte Konfiguration sicherzustellen. Sie können auf das Schema verweisen, indem Sie Folgendes einfügen:

```json
{
  "$schema": "https://mintlify.com/docs.json"
}
```

  <ResponseField name="redirects" type="Redirect[]">
    Ein Array von Pfaden, die Sie permanent zu einem anderen Pfad weiterleiten möchten

    Beispiel:

    ```json
    "redirects": [
      {
        "source": "/source/path",
        "destination": "/destination/path"
      }
    ]
    ```

    <Expandable title="Redirect">
      <ResponseField name="source" type="string">
        Der Pfad, von dem aus Sie weiterleiten möchten.

        Beispiel: `/source`
      </ResponseField>

      <ResponseField name="destination" type="string">
        Der Pfad, zu dem Sie weiterleiten möchten.

        Beispiel: `/destination`
      </ResponseField>
    </Expandable>
  </ResponseField>

  ### Suchmaschinenoptimierung

  <ResponseField name="seo" type="SEO">
    Einstellungen für Suchmaschinenoptimierung.

    Beispiel:

    ```json
    "seo": {
      "indexHiddenPages": true
    }
    ```

    <Expandable title="SEO">
      <ResponseField name="indexHiddenPages" type="boolean" default="false">
        Aktiviert die Indexierung von Seiten, die nicht in `navigation`.
      </ResponseField>
    </Expandable>
  </ResponseField>
