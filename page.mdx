## Page Configuration

Every MDX file requires metadata between `---` delimiters:

<CodeGroup>

```md Basic Example
---
title: "Page"
description: "Your description"
---
```

```md Advanced Example
---
title: "Page Titles and Metadata"
description: "Setting the title and metadata for navigation and SEO"
sidebarTitle: "Short title" # Optional shorter nav title
icon: "rectangle-code" # Font Awesome icon
iconType: "solid" # Optional icon type
mode: "wide" # Optional layout mode
url: "https://external-link.com" # Optional external URL
---
```

</CodeGroup>

<Tip>
  Files without a title will use the filename by default.
</Tip>

## API Pages

To create API playgrounds, add the `api` or `openapi` property:

```md
api: "GET /users"
# or
openapi: "GET /users"
```

Learn more in [our API playground demo](/api-playground).

## Page Modes

- **Default**: Standard layout with table of contents
- **Wide**: Hides table of contents (`mode: "wide"`)
- **Custom**: Minimal layout with only top bar (`mode: "custom"`)

## SEO Optimization

Add meta tags for social sharing and SEO:

```md
---
"twitter:image": "/images/your-photo.jpg"
# Other meta tags supported
---
```

See [SEO settings](/settings/seo) for all options.
