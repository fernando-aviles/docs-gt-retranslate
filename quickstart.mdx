## Get your site deployed

<Steps>
<Step title="Create your account and project">
  [Sign up](https://mintlify.com/start) and complete the onboarding flow. Choose your project name carefully - it becomes your initial URL.

  <Check>
  After onboarding, your site is immediately available at `https://yourproject.mintlify.app`
  </Check>
</Step>

<Step title="Connect your GitHub account">
  Link Mintlify to your GitHub for automatic deployments:
  
  1. In your [dashboard](https://dashboard.mintlify.com/), go to **Settings** → **GitHub App**
  2. Click **Install GitHub App** 
  3. Select your organization and repositories
  4. Return to **Settings** → **My Profile** and click **Authorize GitHub account**

  <Check>
  Green checkmarks appear next to "GitHub App" and "GitHub Authorization" in your dashboard
  </Check>
</Step>

<Step title="Find your live site">
  Your documentation is already deployed! Find your URL in the dashboard overview.

  <Frame>
    <img src="/images/quickstart/mintlify-domain-light.png" alt="Dashboard showing live documentation URL" className="block dark:hidden" />
    <img src="/images/quickstart/mintlify-domain-dark.png" alt="Dashboard showing live documentation URL" className="hidden dark:block" />
  </Frame>

  <Tip>
  Share this URL with your team immediately - they can start viewing and providing feedback while you customize.
  </Tip>
</Step>
</Steps>

## Choose your editing workflow

Pick the workflow that matches how you prefer to work:

<Tabs>
<Tab title="Code Editor" icon="terminal">
  **Perfect for developers** who want to manage docs alongside code, use their favorite editor, and leverage Git workflows.

  ### Quick setup

  <Steps>
  <Step title="Install the CLI">
    Install the Mintlify CLI on your machine:

    <CodeGroup>
    ```bash npm
    npm i -g mint
    ```
    ```bash pnpm  
    pnpm add -g mint
    ```
    ```bash yarn
    yarn global add mint
    ```
    </CodeGroup>

    <Info>
    Requires Node.js v19+. Check your version with `node --version`.
    </Info>
  </Step>

  <Step title="Clone and edit locally">
    1. Clone the repository created during onboarding
    2. Open `index.mdx` in your editor
    3. Change the title from "Introduction" to "Welcome to our docs"

    ```mdx index.mdx {2}
    ---
    title: "Welcome to our docs"
    description: "This is the introduction to the documentation"
    ---
    ```
  </Step>

  <Step title="Preview your changes">
    Run the development server:

    ```bash
    mint dev
    ```

    Visit `localhost:3000` to see your changes instantly.

    <Check>
    Your title change appears in the local preview
    </Check>
  </Step>

  <Step title="Deploy your changes">
    Push to your repository:

    ```bash
    git add .
    git commit -m "Update homepage title"
    git push
    ```

    <Check>
    Changes automatically deploy to your live site within 2-3 minutes
    </Check>
  </Step>
  </Steps>

</Tab>

<Tab title="Web Editor" icon="browser">
  **Perfect for content creators** who prefer visual editing without installing tools or managing Git directly.

  ### Quick setup

  <Steps>
  <Step title="Open the web editor">
    1. Go to your [dashboard](https://dashboard.mintlify.com/)
    2. Click **Editor** in the sidebar

    <Frame>
      <img alt="Mintlify web editor interface" src="/images/quickstart/web-editor-light.png" className="block dark:hidden" />
      <img alt="Mintlify web editor interface" src="/images/quickstart/web-editor-dark.png" className="hidden dark:block" />
    </Frame>
  </Step>

  <Step title="Edit your content">
    1. Select `index.mdx` from the file explorer
    2. Update the title field to "Welcome to our docs"
    3. Use the rich formatting toolbar or type `/` for component shortcuts

    <Tip>
    The editor shows a live preview as you type. No need to save manually!
    </Tip>
  </Step>

  <Step title="Publish your changes">
    Click **Publish** in the top-right corner.

    <Check>
    Your changes are immediately live on your documentation site
    </Check>
  </Step>
  </Steps>

</Tab>

## Add your custom domain

Transform your site from `yourproject.mintlify.app` to `docs.yourcompany.com`:

<Steps>
<Step title="Configure your domain">
  1. Go to [Domain Setup](https://dashboard.mintlify.com/settings/deployment/custom-domain) in your dashboard
  2. Enter your desired domain (like `docs.yourcompany.com`)

  <Frame>
    <img src="/images/quickstart/custom-domain-light.png" alt="Custom domain configuration interface" className="block dark:hidden" />
    <img src="/images/quickstart/custom-domain-dark.png" alt="Custom domain configuration interface" className="hidden dark:block" />
  </Frame>
</Step>

<Step title="Update your DNS settings">
  Add this CNAME record with your domain provider:

  | Record Type | Name | Value | TTL |
  |-------------|------|-------|-----|
  | CNAME | docs | cname.mintlify.app | 3600 |

  <Info>
  DNS changes typically take 5-30 minutes but can take up to 48 hours in rare cases.
  </Info>
</Step>

<Step title="Verify your domain">
  <Check>
  Your documentation loads at your custom domain with a valid SSL certificate
  </Check>
</Step>
</Steps>

## Troubleshooting

<AccordionGroup>
<Accordion title="Local preview isn't working">
**Solution**: Ensure you have Node.js v19+ and run `mint dev` from the directory containing `docs.json`.

**Check your setup**: 
- Run `node --version` (should be v19+)
- Run `ls` and confirm you see `docs.json`
- Try `npm install` if you see dependency errors
</Accordion>

<Accordion title="Changes aren't appearing on my live site">
**For Git workflow**: Check your repository's Actions tab for build errors. Deployments take 2-5 minutes.

**For web editor**: Changes should appear immediately. Try refreshing your browser and clearing cache.

**DNS issues**: If using a custom domain, verify your CNAME record with [DNS Checker](https://dnschecker.org/).
</Accordion>

<Accordion title="GitHub connection failing">
**Solution**: 
1. Ensure you have admin access to the repository
2. Reinstall the GitHub App from **Settings** → **GitHub App**  
3. Re-authorize your account from **Settings** → **My Profile**
4. Contact your GitHub organization admin if you see permission errors
</Accordion>
</AccordionGroup>
