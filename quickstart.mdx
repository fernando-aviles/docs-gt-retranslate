## Overview

Welcome to Mintlify! This guide will walk you through the essential steps to create, deploy, and customize your documentation. Follow these instructions to get your docs up and running in minutes.

## Initial Setup

### Setting Up Your Repository

<Steps>
  <Step title="Choose Your Setup Method">
    You have two options to get started:
    - Use our [dashboard](https://dashboard.mintlify.com) for guided setup
    - Clone our [starter kit](https://github.com/mintlify/starter) directly
  </Step>

  <Step title="Install GitHub Integration">
    <Note>
      The GitHub app ensures automatic deployment of your documentation updates.
    </Note>

    1. Go to [dashboard settings](https://dashboard.mintlify.com/settings)
    2. Find the GitHub app installation link
    3. Complete the installation process
    4. Verify successful integration (look for the checkmark next to your commit hash)

    <Frame>
      ![GitHub Bot Verification](https://mintlify.s3-us-west-1.amazonaws.com/mintlify/images/github-bot-verification.png)
    </Frame>
  </Step>
</Steps>

### Monorepo Configuration

<Info>
  If you prefer keeping your docs alongside your code, Mintlify supports monorepo setups.
</Info>

<Steps>
  <Step title="Move Documentation">
    Transfer your docs content into your monorepo structure
  </Step>
  
  <Step title="Update Configuration">
    Specify your `docs.json` path in the [dashboard settings](https://dashboard.mintlify.com/settings/deployment/git-settings)
  </Step>
</Steps>

## Content Management

### Local Development Setup

<Steps>
  <Step title="Install Prerequisites">
    1. Install [git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git)
    2. Set up [SSH keys](https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent) for GitHub
  </Step>

  <Step title="Clone and Configure">
    1. Clone your repository: `git clone <your-repo>`
    2. Open the repository in your preferred IDE
    3. Install Mintlify CLI: `npm i -g mintlify`
  </Step>
</Steps>

<Tip>
  For detailed instructions on local development, check our [development guide](/development).
</Tip>

### Web Editor

For those who prefer a browser-based editing experience, we offer a powerful web editor. Learn more in our [web editor guide](/web-editor).

### Brand Customization

Customize your documentation's appearance through the `docs.json` configuration file:

```json
{
  "name": "Your Company",
  "logo": {
    "light": "/logo/light.svg",
    "dark": "/logo/dark.svg",
    "href": "https://yourcompany.com"
  },
  "favicon": "/favicon.svg",
  "colors": {
    "primary": "#2AB673",
    "light": "#55D799",
    "dark": "#117866"
  }
}
```

<Info>
  Find all available configuration options in our [global settings](/settings/global) guide.
</Info>

### Creating Content

Add new pages using MDX files with this basic structure:

```md
---
title: "Page Title"
sidebarTitle: "Sidebar title (optional)"
description: "Subtitle (optional)"
---
```

<Tip>
  Explore our guides on [MDX syntax](/text) and [components](/content/components) to enhance your content.
</Tip>

## API Documentation

### Setting Up API References

<Steps>
  <Step title="Prepare OpenAPI Document">
    Add your OpenAPI specification file (`openapi.yaml` or `openapi.json`) to your repository, or specify a URL in `docs.json`:

    ```json
    {
      "openapi": "link-to-your-openapi-file"
    }
    ```
  </Step>

  <Step title="Generate Documentation">
    Use our scraper to create endpoint files:
    ```bash
    npx @mintlify/scraping@latest openapi-file <path-or-url-to-openapi-file>
    ```
  </Step>

  <Step title="Update Navigation">
    Include the generated MDX files in your `docs.json` navigation structure
  </Step>
</Steps>

<Info>
  For detailed API documentation setup, see our guides on [OpenAPI setup](/api-playground/openapi/setup) and [authentication](/api-playground/openapi/writing-openapi).
</Info>
