Welcome to Mintlify! This guide will walk you through deploying your documentation and making it shine. We'll cover initial setup, content updates, and deployment steps.

## Repository Setup

### Create Your Documentation Repository 

Mintlify renders documentation from MDX files using configurations in our [starter kit](https://github.com/mintlify/starter). You can begin by either:

1. Onboarding through the [dashboard](https://dashboard.mintlify.com)
2. Cloning our [starter kit](https://github.com/mintlify/starter) repository

### Connect with GitHub

<Accordion icon="message-bot" title="Install the GitHub App">
  Follow these steps to enable automatic deployments:

  1. Visit your [dashboard settings](https://dashboard.mintlify.com/settings)
  2. Find and click the GitHub app installation link
  3. Complete the installation process
  4. Verify success - you'll see a check mark next to your repository's commit hash

  <Frame>
    ![GitHub Bot Verification](https://mintlify.s3-us-west-1.amazonaws.com/mintlify/images/github-bot-verification.png)
  </Frame>
</Accordion>

### Repository Structure Options

<Accordion icon="code-branch" title="Monorepo Setup">
  To include documentation within your codebase:

  1. Move your docs content into your monorepo
  2. Update the path to your `mint.json` in the [dashboard settings](https://dashboard.mintlify.com/settings/deployment/git-settings)

## Content Management

You have multiple options for updating your documentation content. Choose the method that best fits your workflow.

### Development Environments

<Accordion icon="rectangle-terminal" title="Local Development (Developer Recommended)">
  Set up your local environment:

  1. Install [git](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git)
  2. Clone your repository: `git clone <your-repo>`
  3. Open the repository in your preferred IDE
  4. Install our preview tool: `npm i -g mintlify`

  For detailed instructions, see our [local development guide](/development).
</Accordion>

<Accordion icon="pen-to-square" title="Web Editor">
  For a more visual editing experience, try our web editor. Learn how to use it in our [web editor guide](/web-editor).
</Accordion>

### Customization

<Accordion icon="palette" title="Brand Customization">
  Update your `mint.json` file with your brand elements:

  ```json Basic Configuration
  {
    "name": "Your Company",
    "logo": {
      "light": "/logo/light.svg",
      "dark": "/logo/dark.svg",
      "href": "https://yourcompany.com"
    },
    "favicon": "/favicon.svg",
    "colors": {
      "primary": "#2AB673",
      "light": "#55D799",
      "dark": "#117866"
    }
  }
  ```

  Find all available options in our [global settings](/settings/global) guide.
</Accordion>

### Content Creation

<Accordion icon="pencil" title="Adding Pages">
  Create new pages using this basic template:

  ```md Page Template
  ---
  title: "Page Title"
  sidebarTitle: "Sidebar title (optional)"
  description: "Brief description"
  ---
  ```

  Learn more about:
  - [MDX syntax](/text) for formatting
  - [Component library](/content/components) for enhanced features
</Accordion>

### Deployment Process

<Accordion icon="rocket" title="Publishing Changes">
  Deploy your changes by:

  1. Committing your updates
  2. Pushing to your repository

  If the GitHub app deployment fails, you can manually update through the [dashboard](https://dashboard.mintlify.com).

### API Documentation

<Accordion icon="code-branch" title="API Reference Setup">
  Configure API documentation using OpenAPI:

  1. Add your OpenAPI specification:
  ```json
  "openapi": "link-to-your-openapi-file"
  ```

  2. Generate endpoint files:
  ```bash
  npx @mintlify/scraping@latest openapi-file <path-or-url-to-openapi-file>
  ```

  3. Add generated files to `mint.json` navigation

  Resources:
  - [OpenAPI setup guide](/api-playground/openapi/setup)
  - [Authentication configuration](/api-playground/openapi/writing-openapi)
  - [Manual API documentation](/api-playground/overview)
</Accordion>
