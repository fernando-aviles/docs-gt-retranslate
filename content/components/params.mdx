The `ParamField` component helps you define parameters for your APIs or SDKs. When you add a ParamField to your documentation, it automatically creates an [API Playground](/api-playground/overview) where users can test the parameters.

<ParamField path="param" type="string" required>
  An example of a parameter field
</ParamField>

<RequestExample>

```jsx Path Example
<ParamField path="param" type="string">
  An example of a parameter field
</ParamField>
```

```jsx Query Example
<ParamField query="filter" type="string" default="none" required>
  The filtering command that helps sort through users
</ParamField>
```

```jsx Body Example
<ParamField body="user_age" type="integer" default="0" required>
  The age of the user. Must be 0 or greater
</ParamField>
```

</RequestExample>

## Available Properties

Here are all the properties you can use with ParamField:

<ParamField body="query, path, body, or header" type="string">
  Choose the parameter type (query, path, body, or header) and add its name
</ParamField>

<ParamField body="type" type="string">
  The kind of value you expect for this parameter (like string, number, etc.)
</ParamField>

<ParamField body="required" type="boolean">
  Sets whether users must provide this parameter or not
</ParamField>

<ParamField body="deprecated" type="boolean">
  Shows if this parameter is no longer recommended for use
</ParamField>

<ParamField body="default" type="string">
  The value that will be used if no other value is provided
</ParamField>

<ParamField body="initialValue" type="any">
  The starting value that will appear in the playground
</ParamField>

<ParamField body="placeholder" type="string">
  Helper text that appears in the input field of the playground
</ParamField>

<ParamField body="children" type="string">
  The description of what this parameter does (supports markdown formatting)
</ParamField>
