Mintlify supports three methods for authenticating users to access personalized content:

## Authentication Options

<Tabs>
  <Tab title="Shared Session">
    Use your dashboard's existing session token for authentication.

    **Requirements:**
    - Dashboard hosted at your domain with cookie-based sessions
    - API endpoint at same origin/subdomain as dashboard
    - Docs hosted at same domain as dashboard

    **Best For:** Teams with existing session-based authentication wanting seamless integration.
  </Tab>
  <Tab title="JWT">
    Use your login flow to pass user info via JWT in the URL.

    **Requirements:**
    - Existing login flow
    - Ability to create and redirect with JWT

    **Best For:** Applications needing simple setup with minimal CORS configuration.
  </Tab>
  <Tab title="OAuth 2.0">
    Integrate with your OAuth server using PKCE flow.

    **Requirements:**
    - OAuth server supporting PKCE flow
    - API endpoint accessible via OAuth token

    **Best For:** Applications requiring high security standards.
  </Tab>
</Tabs>

## Method Comparison

<Tabs>
  <Tab title="Shared Session">
    ✅ Automatic login with dashboard
    ✅ Persistent sessions
    ✅ Minimal setup
    ❌ Requires backend requests
    ❌ CORS configuration needed
  </Tab>
  <Tab title="JWT">
    ✅ Reduced API endpoint abuse
    ✅ No CORS configuration
    ✅ Flexible API URLs
    ❌ Separate login from dashboard
    ❌ Requires regular re-login for data updates
  </Tab>
  <Tab title="OAuth 2.0">
    ✅ Highest security standard
    ✅ Flexible API URLs
    ❌ Complex initial setup
    ❌ Separate login from dashboard
    ❌ May be excessive for simple needs
  </Tab>
</Tabs>
