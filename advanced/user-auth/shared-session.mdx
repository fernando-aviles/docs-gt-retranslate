---
title: 'Shared Session Auth'
description: 'Seamlessly share user sessions between your dashboard and your docs'
---

Share existing session authentication from your user's browser with your documentation.

## Implementation

<Steps>
  <Step title="Create your Info API">
    Create an API endpoint that uses session authentication and returns a JSON payload following the [UserInfo](./sending-data) format.

    For different domains:
    - Add docs domain to `Access-Control-Allow-Origin` header (not `*`)
    - Set `Access-Control-Allow-Credentials` to `true`
    <Warning>
      Enable these CORS settings only on the user info endpoint.
    </Warning>
  </Step>
  <Step title="Configure Settings">
    Add your API URL and Login URL in your [Mintlify dashboard settings](https://dashboard.mintlify.com/mintlify/mintlify/settings/deployment/user-authentication).
  </Step>
</Steps>

## Example Configurations

1. **Dashboard and docs on subdomains** (`dash.foo.com` and `docs.foo.com`)
   - Create endpoint: `dash.foo.com/api/docs/user-info`
   - Add CORS for `https://docs.foo.com`

2. **Dashboard on subdomain, docs at root** (`dash.foo.com` and `foo.com/docs`)
   - Create endpoint: `dash.foo.com/api/docs/user-info`
   - Add CORS for `https://foo.com`

3. **Both at root** (`foo.com/dashboard` and `foo.com/docs`)
   - Create endpoint: `foo.com/api/docs/user-info`
   - No CORS needed
