## Create Cloudfront Distribution

1. Go to [Cloudfront](https://aws.amazon.com/cloudfront) and click `Create distribution`
2. Set Origin domain to `[SUBDOMAIN].mintlify.dev`
3. Select `Caching Optimized` for Cache key and origin requests
4. Enable Web Application Firewall (WAF) security protections
5. Create the distribution with default remaining settings

## Configure Origins and Behaviors

### Add Default Origin
1. Navigate to `Origins` tab
2. Add your landing page staging URL (e.g., `.webflow.io` or `.vercel.app` domain)
3. You should have two origins: `[SUBDOMAIN].mintlify.dev` and your staging URL

### Set Behaviors
Configure three behaviors to control routing:

1. **`/docs/*` Behavior**
   - Origin: `[SUBDOMAIN].mintlify.dev`
   - Cache policy: CachingOptimized

2. **`/docs` Behavior**
   - Same settings as `/docs/*`

3. **Default (*) Behavior**
   - Origin: Your staging URL

## Test and Deploy

1. Test using the Distribution domain name URL in the `General` tab
2. Connect to Route53:
   - Go to Route53 hosted zone for your domain
   - Create record with Alias to CloudFront distribution
   - Remove existing A record if present

Your documentation should now be accessible at `yourdomain.com/docs`.
