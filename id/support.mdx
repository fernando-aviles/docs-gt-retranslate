## Pertanyaan yang Sering Diajukan

<AccordionGroup>
  <Accordion title="Masalah aplikasi GitHub" icon="merge">
    Aplikasi GitHub bisa bermasalah dan mereset koneksi adalah cara yang bagus untuk menyelesaikan masalah.

    <Steps>
      <Step title="Copot pemasangan aplikasi Mintlify melalui GitHub">
        1. Di GitHub, pergi ke [installations](https://github.com/settings/installations) dan pilih **Configure** di sebelah aplikasi Mintlify. Gulir ke bawah dan pilih **Uninstall**.
        2. Pergi ke [Authorized GitHub Apps](https://github.com/settings/apps/authorizations) dan pilih **Revoke** di sebelah aplikasi Mintlify.
      </Step>

      <Step title="Pasang ulang aplikasi Mintlify">
        1. Di dasbor Mintlify Anda, pergi ke [Git Settings](https://dashboard.mintlify.com/settings/deployment/git-settings) dan pasang aplikasi GitHub.
        2. Otorisasi akun Anda di bagian [My Profile](https://dashboard.mintlify.com/settings/account) di dasbor Anda.
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="Masalah halaman API" icon="book-open">
    Halaman API itu rumit. Akibatnya, ada banyak hal yang bisa salah. Berikut adalah daftar masalah umum yang sering dihadapi pelanggan:

    <AccordionGroup>
      <Accordion title="Semua halaman OpenAPI saya benar-benar kosong">
        Dalam skenario ini, kemungkinan besar Mintlify tidak dapat menemukan dokumen OpenAPI Anda atau dokumen OpenAPI Anda tidak valid.

        Menjalankan `mint dev` secara lokal seharusnya mengungkapkan beberapa masalah ini.

        Untuk memverifikasi bahwa dokumen OpenAPI Anda akan lulus validasi:

        1. Jalankan `mint openapi-check <openapiFilenameOrUrl>` di CLI Anda.
        2. Validasi spesifikasi OpenAPI Anda dengan [Swagger Editor](https://editor.swagger.io/).

        Selain itu, Mintlify tidak mendukung OpenAPI 2.0. Jika dokumen Anda menggunakan OpenAPI 2.0, Anda mungkin mengalami masalah ini. Anda dapat mengkonversi dokumen Anda dengan [Swagger Editor](https://editor.swagger.io/).

        <Frame>
          !["Convert to OpenAPI 3" menu item highlighted in the Swagger Editor](/images/convert-oas-3.png)
        </Frame>
      </Accordion>

      <Accordion title="Salah satu halaman OpenAPI saya benar-benar kosong">
        Ini biasanya disebabkan oleh kesalahan ejaan pada bidang `openapi` di metadata halaman. Pastikan bahwa metode HTTP dan path cocok persis dengan metode HTTP dan path dalam dokumen OpenAPI.

        Berikut adalah contoh bagaimana hal-hal bisa salah:

        ```md get-user.mdx
        ---
        openapi: "GET /users/{id}/"
        ---
        ```

        ```yaml openapi.yaml
        paths:
          "/users/{id}":
            get: ...
        ```

        Perhatikan bahwa path di bidang `openapi` memiliki garis miring di akhir, sedangkan path di dokumen OpenAPI tidak.

        Masalah umum lainnya adalah kesalahan ejaan nama file. Jika Anda menentukan dokumen OpenAPI tertentu di bidang `openapi`, pastikan nama filenya benar. Misalnya, jika Anda memiliki dua dokumen OpenAPI `openapi/v1.json` dan `openapi/v2.json`, metadata Anda mungkin terlihat seperti ini:

        ```md api-reference/v1/users/get-user.mdx
        ---
        openapi: "v1 GET /users/{id}"
        ---
        ```
      </Accordion>

      <Accordion title="Permintaan dari API Playground tidak berfungsi">
        Jika Anda memiliki domain kustom yang dikonfigurasi, ini bisa menjadi masalah dengan reverse proxy Anda. Secara default, permintaan yang dibuat melalui API Playground dimulai dengan permintaan `POST` ke path `/api/request` pada situs dokumen. Jika reverse proxy Anda dikonfigurasi untuk hanya mengizinkan permintaan `GET`, maka semua permintaan ini akan gagal. Untuk memperbaikinya, konfigurasikan reverse proxy Anda untuk mengizinkan permintaan `POST` ke path `/api/request`.

        Atau, jika reverse proxy Anda mencegah Anda menerima permintaan `POST`, Anda dapat mengonfigurasi Mintlify untuk mengirim permintaan langsung ke backend Anda dengan pengaturan `api.playground.proxy` di `docs.json`. Lihat [configurations for the API playground](/id/settings#param-proxy) untuk informasi lebih lanjut. Ini kemungkinan akan mengharuskan Anda untuk mengonfigurasi CORS di server Anda, karena permintaan ini sekarang akan datang langsung dari browser pengguna Anda.
      </Accordion>
    </AccordionGroup>
  </Accordion>

  <Accordion title="Halaman dokumen tidak dapat dimuat (error 404)" icon="ban" iconType="regular">
    Periksa apakah Anda memiliki `/api` di URL. Ini adalah folder yang dicadangkan di lingkungan produksi, yang menghasilkan error 404.
  </Accordion>

  <Accordion title="Masalah menambahkan SVG ke kartu" icon="image">
    Kami mendukung jenis SVG tertentu, yang disebut JSX, dan karenanya gambar perlu dikonversi.

    1. Gunakan [SVGR converter](https://react-svgr.com/playground/) untuk menghasilkan kode SVG yang kompatibel dengan JSX.
    2. Salin kode di dalam tag `<svg/>`.

    ![Output from the SVGR converter with the code inside  and  tags highlighted.](/images/svg-jsx.png)

    3. Tempel kode ke dalam kartu Anda. Pastikan Anda hanya menyalin dan menempel kode di dalam tag `<svg/>`.
    4. Anda mungkin perlu mengurangi tinggi dan lebar untuk membuat gambar pas.
  </Accordion>
</AccordionGroup>
