<Info>
  **Prasyarat**: Harap instal [Node.js](https://nodejs.org/en) (versi 19 atau lebih tinggi) sebelum melanjutkan.
</Info>

**Langkah 1**: Instal [CLI](https://www.npmjs.com/package/mint):

<CodeGroup>
  ```bash npm
  npm i -g mint
  ```

  ```bash yarn
  yarn global add mint
  ```

  ```bash pnpm
  pnpm add -g mint
  ```
</CodeGroup>

**Langkah 2**: Navigasikan ke direktori docs (di mana file `docs.json` berada) dan jalankan perintah berikut:

```bash
mint dev
```

Atau, jika Anda tidak ingin menginstal CLI secara global, Anda dapat menggunakan skrip satu kali:

<CodeGroup>
  ```bash npm
  npx mint dev
  ```

  ```bash yarn
  yarn dlx mint dev
  ```

  ```bash pnpm
  pnpm dlx mint dev
  ```
</CodeGroup>

Pratinjau lokal dokumentasi Anda akan tersedia di `http://localhost:3000`.

## Pembaruan

Setiap rilis CLI terkait dengan versi tertentu dari platform Mintlify. Jika pratinjau lokal Anda tidak sesuai dengan versi produksi, harap perbarui CLI:

<CodeGroup>
  ```bash npm
  npm i -g mint@latest
  ```

  ```bash yarn
  yarn global upgrade mint
  ```

  ```bash pnpm
  pnpm up --global mint
  ```
</CodeGroup>

## Port Kustom

Secara default, CLI menggunakan port 3000. Anda dapat menyesuaikan port menggunakan flag `--port`. Untuk menjalankan CLI di port 3333, misalnya, gunakan perintah ini:

```bash
mint dev --port 3333
```

Jika Anda mencoba menjalankan pada port yang sudah digunakan, CLI akan menggunakan port berikutnya yang tersedia:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Perintah Tambahan

Meskipun `mint dev` adalah perintah yang paling sering digunakan, ada perintah lain yang dapat Anda gunakan untuk mengelola dokumentasi Anda.

### Temukan Tautan Rusak

CLI dapat membantu memvalidasi tautan referensi yang dibuat dalam dokumentasi Anda. Untuk mengidentifikasi tautan yang rusak, gunakan perintah berikut:

```bash
mint broken-links
```

### Periksa Spesifikasi OpenAPI

Anda dapat menggunakan CLI untuk memeriksa file OpenAPI Anda untuk kesalahan menggunakan perintah berikut:

```bash
mint openapi-check <openapiFilenameOrUrl>
```

Anda dapat memasukkan nama file (misalnya `./openapi.yaml`) atau URL (misalnya `https://petstore3.swagger.io/api/v3/openapi.json`).

### Mengganti Nama File

Anda dapat mengganti nama file menggunakan perintah berikut:

```bash
mint rename <oldFilename> <newFilename>
```

Ini adalah peningkatan dibandingkan dengan memperbarui nama file secara normal karena juga akan memperbarui semua referensi ke file tersebut, memastikan tidak ada tautan yang rusak.

## Pemformatan

Saat mengembangkan secara lokal, kami merekomendasikan penggunaan ekstensi pada IDE Anda untuk mengenali dan memformat MDX.

Jika Anda menggunakan Cursor, Windsurf, atau VSCode, kami merekomendasikan [ekstensi MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) untuk penyorotan sintaks, dan [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) untuk pemformatan kode.

Jika Anda menggunakan JetBrains, kami merekomendasikan [plugin MDX IntelliJ IDEA](https://plugins.jetbrains.com/plugin/14944-mdx) untuk penyorotan sintaks, dan pengaturan [Prettier](https://prettier.io/docs/webstorm) untuk pemformatan kode.

## Pemecahan Masalah

<AccordionGroup>
  <Accordion title="Error: Could not load the &#x22;sharp&#x22; module using the darwin-arm64 runtime">
    Ini mungkin disebabkan oleh versi node yang sudah usang. Coba langkah-langkah berikut:

    1. Hapus versi CLI mint yang saat ini terinstal: `npm remove -g mint`
    2. Upgrade ke Node v19 atau lebih tinggi.
    3. Instal ulang CLI mint: `npm install -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
    Solution: Go to the root of your device and delete the \~/.mintlify folder. Afterwards, run `mint dev` lagi.
  </Accordion>
</AccordionGroup>
