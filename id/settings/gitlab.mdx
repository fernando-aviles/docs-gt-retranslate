Kami menggunakan kombinasi Access token dan Webhook untuk mengautentikasi dan menyinkronkan perubahan antara GitLab dan Mintlify.

* Kami menggunakan Access token untuk mengambil informasi dari GitLab.
* Kami menggunakan Webhook agar GitLab dapat memberi tahu Mintlify ketika ada perubahan.
  * Ini memungkinkan Mintlify untuk membuat preview deployment ketika MR dibuat.

## Mengatur koneksi

<Steps>
  <Step title="Find your project ID">
    Di dalam proyek GitLab Anda, navigasikan ke `Settings` > `General` dan temukan `Project ID`.

    <Frame>
      <img src="/images/gitlab/gitlab-project-id.png" />
    </Frame>
  </Step>

  <Step title="Generate an access token">
    a. Navigasikan ke `Settings` > `Access Tokens`.

    b. Pilih `Add new token`.

    1. Beri nama token "Mintlify".
    2. Jika Anda memiliki repo private, Anda harus mengatur peran sebagai `Maintainer`.
    3. Pilih `api` dan `read_api` untuk scopenya.

    c. Terakhir klik `Create project access token` dan salin tokennya.

    <Frame>
      <img src="/images/gitlab/gitlab-project-access-token.png" />
    </Frame>
  </Step>

  <Step title="Set up the connection">
    Di dalam [dashboard Mintlify](https://dashboard.mintlify.com/settings/deployment/git-settings), tambahkan ID proyek dan access token dari langkah-langkah sebelumnya bersama dengan konfigurasi lainnya. Klik "Save Changes" ketika Anda selesai.

    <Frame>
      <img src="/images/gitlab/gitlab-config.png" />
    </Frame>
  </Step>
</Steps>

## Membuat webhook

Webhook memungkinkan kami menerima event ketika ada perubahan sehingga kami dapat memicu deployment secara otomatis.

<Steps>
  <Step title="Navigate to Settings > Webhooks and click 'Add new Webhook'">
    <Frame>
      <img src="/images/gitlab/gitlab-webhook.png" />
    </Frame>
  </Step>

  <Step title="Set up URL and webhook">
    Di kolom "URL", masukkan endpoint `https://leaves.mintlify.com/gitlab-webhook` dan beri nama webhook "Mintlify".
  </Step>

  <Step title="Paste token">
    Tempel token Webhook yang dihasilkan setelah mengatur koneksi.

    <Frame>
      <img src="/images/gitlab/gitlab-show-webtoken.png" />
    </Frame>
  </Step>

  <Step title="Select events">
    Pilih event yang ingin Anda picu webhook:

    * Push events (Semua branch)
    * Merge requests events

    Ketika Anda selesai, hasilnya akan terlihat seperti ini:

    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken.png" />
    </Frame>
  </Step>

  <Step title="Test the Webhook">
    Setelah membuat Webhook, klik dropdown "Test" dan pilih "Push events" untuk mengirim contoh payload untuk memastikan konfigurasi sudah benar. Akan muncul pesan "Hook executed successfully: HTTP 200" jika dikonfigurasi dengan benar.

    Ini akan membantu Anda memverifikasi bahwa semuanya berfungsi dengan benar dan bahwa dokumentasi Anda akan tersinkronisasi dengan baik dengan repositori GitLab Anda.

    <Frame>
      <img src="/images/gitlab/gitlab-project-webtoken-test.png" />
    </Frame>
  </Step>
</Steps>
