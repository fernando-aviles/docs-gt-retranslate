<Info>
  Ini adalah dokumentasi untuk Handshake**Autentikasi** Kata Sandi. Handshake Kata Sandi tidak tersedia untuk Personalisasi.
</Info>

Jika Anda tidak memiliki persyaratan keamanan yang ketat, atau Anda tidak ingin mengelola database pembaca dokumentasi, Anda dapat menggunakan serangkaian kata sandi bersama untuk melindungi dokumentasi Anda.

<Note>
  Autentikasi hanya tersedia berdasarkan permintaan. Silakan{" "}
  <a href="mailto:sales@mintlify.com">hubungi tim sales kami</a> untuk mendapatkan akses. Setelah kami mengaktifkan akses, Anda dapat mengikuti instruksi implementasi.
</Note>

## Implementasi

<Steps>
  <Step title="Tambahkan kata sandi">
    Pergi ke [dashboard
    pengaturan](https://dashboard.mintlify.com/products/authentication) dan buat
    sebuah kata sandi.
  </Step>

  <Step title="Bagikan kata sandi Anda">
    Bagikan kata sandi tersebut dengan aman kepada pembaca dokumentasi Anda. Itu saja!
  </Step>
</Steps>

## Contoh

Saya ingin mengatur autentikasi untuk dokumentasi saya yang dihosting di `docs.foo.com`. Saya tidak ingin
harus melacak siapa yang dapat dan tidak dapat mengakses dokumentasi. Kasus penggunaan utama saya untuk
autentikasi adalah untuk mencegah pesaing mengintip - berbagi kata sandi cukup
aman untuk tim saya.

Untuk mengatur autentikasi dengan Mintlify, saya pergi ke dashboard Mintlify saya dan menambahkan
setidaknya satu kata sandi. Kemudian saya membagikan kata sandi tersebut, bersama dengan URL dokumentasi pribadi,
kepada calon pelanggan.
