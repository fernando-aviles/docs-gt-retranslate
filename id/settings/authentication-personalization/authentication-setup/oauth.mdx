<Info>
  Ini adalah dokumentasi untuk OAuth **Authentication** Handshake. Langkah-langkah untuk mengatur [OAuth **Personalization** Handshake](/id/settings/authentication-personalization/personalization-setup/oauth) sedikit berbeda.
</Info>

Jika Anda memiliki server OAuth yang sudah ada, Anda dapat mengintegrasikannya dengan Mintlify untuk pengalaman login yang mulus.

## Implementasi

<Steps>
  <Step title="Konfigurasi pengaturan Authentication Anda">
    Pergi ke [pengaturan authentication Mintlify](https://dashboard.mintlify.com/products/authentication), pilih opsi OAuth, dan isi bidang-bidang yang diperlukan:

    * **Authorization URL**: URL dasar untuk permintaan otorisasi, yang akan kami tambahkan parameter query yang sesuai.
    * **Client ID**: ID untuk klien OAuth 2.0 yang akan digunakan.
    * **Scopes**: Array scope yang akan diminta.
    * **Token URL**: URL dasar untuk permintaan pertukaran token.
    * **Info API URL** (opsional): Endpoint yang akan diakses untuk mengambil info pengguna. Jika dihilangkan, alur OAuth hanya akan digunakan untuk memverifikasi identitas, dan info pengguna akan kosong.
  </Step>

  <Step title="Konfigurasi klien OAuth Anda">
    Salin Redirect URL yang tercantum di [pengaturan authentication Mintlify](https://dashboard.mintlify.com/products/authentication) dan tambahkan sebagai URL redirect yang diizinkan untuk server OAuth Anda.
  </Step>

  <Step title="Buat Info API Anda (Opsional)">
    Jika Anda ingin memanfaatkan fitur kustomisasi autentikasi, Anda perlu membuat endpoint untuk mengambil info tentang pengguna Anda.
    Buat endpoint API yang dapat diakses dengan token akses OAuth, dan merespons dengan payload JSON mengikuti format [User](../sending-data).

    Kembali ke [pengaturan authentication Mintlify](https://dashboard.mintlify.com/products/authentication) Anda dan tambahkan URL Info API
    ke konfigurasi OAuth Anda.
  </Step>
</Steps>

## Contoh

Saya memiliki server OAuth yang sudah ada yang mendukung alur Authorization Code. Saya ingin mengatur autentikasi untuk docs saya yang di-host di `foo.com/docs`.

Untuk mengatur autentikasi dengan Mintlify, saya membuat endpoint `api.foo.com/docs/user-info` yang memerlukan token akses OAuth dengan scope `docs-user-info`, dan merespons dengan data kustom pengguna sesuai dengan spesifikasi Mintlify.

Kemudian saya pergi ke pengaturan dashboard, navigasi ke pengaturan Autentikasi, pilih OAuth, dan masukkan nilai-nilai yang relevan untuk alur OAuth dan endpoint Info API:

* **Authorization URL**: `https://auth.foo.com/authorization`
* **Client ID**: `ydybo4SD8PR73vzWWd6S0ObH`
* **Scopes**: `['docs-user-info']`
* **Token URL**: `https://auth.foo.com/exchange`
* **Info API URL**: `https://api.foo.com/docs/user-info`

Terakhir, saya menyalin Redirect URL yang ditampilkan di pengaturan dashboard dan menambahkannya sebagai URL redirect yang diizinkan dalam pengaturan konfigurasi klien OAuth saya.
