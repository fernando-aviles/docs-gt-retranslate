Ada dua jenis field: Field Parameter dan Field Respons.

## Field Parameter

Komponen`ParamField`digunakan untuk mendefinisikan parameter untuk API atau SDK Anda. Menambahkan`ParamField`akan secara otomatis menambahkan[API Playground](/id/api-playground/overview).

<ParamField path="param" type="string" required>
  Contoh field parameter
</ParamField>

<RequestExample>
  ```jsx Path Example
  <ParamField path="param" type="string">
    An example of a parameter field
  </ParamField>
  ```

  ```jsx Body Example
  <ParamField body="user_age" type="integer" default="0" required>
    The age of the user. Cannot be less than 0
  </ParamField>
  ```

  ```jsx Response Example
  <ResponseField name="response" type="string" required>
    A response field example
  </ResponseField>
  ```
</RequestExample>

### Props

<ParamField body="query, path, body, or header" type="string">
  Apakah itu parameter query, path, body, atau header diikuti dengan nama
</ParamField>

<ParamField body="type" type="string">
  Tipe yang diharapkan dari nilai parameter

  Mendukung`number`,`string`,`bool`,`object`.

  Array dapat didefinisikan menggunakan akhiran`[]`. Misalnya`string[]`.
</ParamField>

<ParamField body="required" type="boolean">
  Menunjukkan apakah parameter diperlukan
</ParamField>

<ParamField body="deprecated" type="boolean">
  Menunjukkan apakah parameter sudah usang
</ParamField>

<ParamField body="default" type="string">
  Nilai default yang digunakan oleh server jika permintaan tidak menyediakan nilai
</ParamField>

<ParamField body="initialValue" type="any">
  Nilai yang akan digunakan untuk menginisialisasi playground
</ParamField>

<ParamField body="placeholder" type="string">
  Teks placeholder untuk input di playground
</ParamField>

<ParamField body="children" type="string">
  Deskripsi parameter (markdown diaktifkan)
</ParamField>

## Field Respons

Komponen`<ResponseField>`dirancang untuk mendefinisikan nilai kembali dari API. Banyak dokumentasi juga menggunakan`<ResponseField>`pada halaman ketika Anda perlu mendaftar tipe sesuatu.

<ResponseField name="response" type="string" required>
  Contoh field respons
</ResponseField>

```jsx
<ResponseField name="response" type="string" required>
  A response field example
</ResponseField>
```

### Props

<ResponseField name="name" type="string" required>
  Nama nilai respons.
</ResponseField>

<ResponseField name="type" type="string" required>
  Tipe yang diharapkan dari nilai respons - ini bisa berupa string arbitrer apa pun.
</ResponseField>

<ResponseField name="default" type="string">
  Nilai default.
</ResponseField>

<ResponseField name="required" type="boolean">
  Tampilkan "required" di samping nama field.
</ResponseField>

<ResponseField name="deprecated" type="boolean">
  Apakah suatu field sudah usang atau tidak.
</ResponseField>

<ResponseField name="pre" type="string[]">
  Label yang ditampilkan sebelum nama field
</ResponseField>

<ResponseField name="post" type="string[]">
  Label yang ditampilkan setelah nama field
</ResponseField>
