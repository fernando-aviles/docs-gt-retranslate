Web Editor adalah cara yang disukai untuk mengedit dokumentasi secara langsung tanpa harus membuka IDE Anda atau menjalankan`mintlify dev`.

Editor ini mencakup beberapa fitur kunci untuk berintegrasi langsung ke dalam alur kerja git Anda yang ada,
seperti membuat cabang, pull request, commit, dan diff untuk perubahan Anda saat ini.

Ini juga mencakup pengalaman yang sepenuhnya dapat diedit untuk mengubah dan menambahkan konten secara langsung,
bahkan dengan komponen kustom.

Jika Anda memahami alur kerja git dan integrasi kami, Anda dapat melompat ke[sini](#editing-content).

## Alur kerja Git dan pembaruan

### Dasar-dasar Git

Meskipun Web Editor berarti Anda tidak perlu ke GitHub atau command line untuk membuat
perubahan, tetap berguna untuk mengetahui dasar-dasar git.

Terminologi Git:

* **Repository**: Folder tempat kode Anda berada. Bisa lokal (di komputer Anda) atau remote (seperti GitHub).

* **Commit**: Snapshot perubahan yang dibuat pada file dalam repository.

* **Branch**: Jalur pengembangan terpisah. Ini adalah salinan kerja dari kode yang memungkinkan Anda bekerja pada perubahan tanpa mempengaruhi versi utama.

* **Pull request:** Permintaan untuk menggabungkan perubahan dari cabang kerja ke cabang utama. Ini digunakan untuk meninjau konten sebelum membuat perubahan langsung.

## Mode editor

Untuk memberikan fleksibilitas maksimal, editor memiliki tiga mode:

<Frame caption="View mode toggle">
  <img className="block dark:hidden" src="/images/editor/mode-toggle-light.png" />

  <img className="hidden dark:block" src="/images/editor/mode-toggle-dark.png" />
</Frame>

### Editor Visual

Mode **"Editor Visual"** adalah mode WYSIWYG tipikal yang bisa Anda lihat di aplikasi seperti Notion. Mode ini menawarkan tampilan dokumen Anda yang dapat diedit sepenuhnya dan mencerminkan tampilan akhir halaman di situs dokumentasi utama Anda.

### Editor Sumber

Mode **"Editor Sumber"** menawarkan cara untuk mengedit file MDX Anda dalam kode, sama seperti yang Anda lakukan di IDE. Ini menawarkan fleksibilitas yang lebih sedikit, karena komponen kami tidak tersedia untuk pelengkapan otomatis, tetapi memiliki dua keunggulan unik.

1. Ini memungkinkan Anda mengedit properti komponen (lihat [batasan di bawah](#current-limitations)) yang saat ini tidak tersedia di mode **"Editor Visual"** untuk saat ini.

2. Ini memungkinkan Anda memperbaiki kesalahan sintaks yang mungkin muncul di repositori Anda yang mungkin tidak kompatibel dengan mode **"Editor Visual"** sampai Anda memperbaikinya.

### Tampilan Diff

Mode **"Tampilan Diff"** adalah cara untuk melihat perubahan yang dibuat pada dokumen tertentu sebelum melakukan commit ke repositori Anda.

Ini akan membantu Anda melihat perubahan yang telah Anda buat beserta perubahan format yang dilakukan oleh editor.
