The Web Editor provides an intuitive interface to edit your docs directly through the dashboard without needing to open your IDE or run the development server. Let's explore its key features and how to use them effectively.

## Core Features

The editor comes with several powerful capabilities:

- Live preview as you edit content
- Git workflow integration for branches and PRs 
- Visual and source code editing modes
- Direct file creation and editing
- Image upload and management
- Component insertion via slash commands

If you're already familiar with git workflows and our integrations, feel free to skip to the [Editing Content](#editing-content) section.

## Understanding Git Workflows

### Key Git Concepts

Before diving into the editor features, here are some essential git terms to know:

- **Repository**: Your code's storage location, either local (on your computer) or remote (like GitHub)
- **Branch**: A separate development line for working on changes safely
- **Commit**: A snapshot of your file changes
- **Pull Request**: A request to merge your branch's changes into the main branch

### Setting Up GitHub Integration

<Note>
  Before making updates, install the Mintlify GitHub app. You can do this through the [GitHub App settings](https://dashboard.mintlify.com/settings/organization/github-app) or follow our [deployment documentation](#1-deploying-your-docs-repository).
</Note>

## Working with Branches

### Creating New Branches

When making changes, it's best practice to work in a separate branch rather than directly in main. Here's how to create and manage branches:

1. Click the branch selector in the top left of the editor
2. Choose "New Branch" from the modal

### Branch Creation Modal

The branch creation modal lets you:

<Frame>

<Tip>
  Always create a new branch for changes to enable team review via pull requests. Some repositories may restrict direct main branch changes.
</Tip>

### Branch Switching

After creating a branch:
- Your editor automatically switches to the new branch
- Changes are saved to this branch until you switch or reload
- Reloading defaults back to the main branch

## Editor Modes

The editor offers three distinct modes for different needs:

<Frame caption="View mode toggle">
  <img src="/images/editor/mode-toggle-light.png" className="block dark:hidden" />
  <img src="/images/editor/mode-toggle-dark.png" className="hidden dark:block" />
</Frame>

### Visual Editor Mode

The Visual Editor provides a WYSIWYG interface similar to Notion:
- Real-time preview of content
- Visual component editing
- Slash command support
- Drag-and-drop functionality

### Source Editor Mode

The Source Editor lets you work directly with MDX code:
- Direct access to component props
- Manual syntax error correction
- Code-level content editing
- Raw markdown editing

### Diff View Mode

The Diff View helps track changes:
- Side-by-side change comparison
- Highlighted additions/deletions
- Format change visibility
- Pre-commit review capability
